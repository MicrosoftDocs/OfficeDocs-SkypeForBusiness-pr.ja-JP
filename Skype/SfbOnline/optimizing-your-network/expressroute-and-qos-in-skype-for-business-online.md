<span data-ttu-id="1bf62-749">転送プロセスを簡素化する以外に、MPLS が提供するもう 1 つの利点は、ネットワーク管理システムが、ネットワーク内のすべてのリンクでどの接続が行われるかを知るという利点です。</span><span class="sxs-lookup"><span data-stu-id="1bf62-749">Beside simplifying the forwarding process, the other advantage MPLS provides is that the network management system will know what connections are being carried on every link in the network.</span></span> <span data-ttu-id="1bf62-750">ネットワーク経由のトラフィックのルーティング方法を制御することで、オペレーターは各パスが提供する QoS を保証できます。</span><span class="sxs-lookup"><span data-stu-id="1bf62-750">By controlling the way traffic is routed through the network, the operator can guarantee the QoS each path will provide.</span></span> <span data-ttu-id="1bf62-751">そのため、従来の IP または基本 IP のベスト イベスト パフォーマンスとは異なり、MPLS 演算子は予測可能なパフォーマンスを持つ IP サービスを提供できます。</span><span class="sxs-lookup"><span data-stu-id="1bf62-751">So unlike the best effort performance of traditional or basic IP, MPLS operators can provide an IP service with predictable performance.</span></span> <span data-ttu-id="1bf62-752">また、LSP により、MPLS は従来のインターネット サービスよりも本質的にセキュリティが高くなっています。</span><span class="sxs-lookup"><span data-stu-id="1bf62-752">That LSP also makes MPLS inherently more secure than traditional Internet services.</span></span> <span data-ttu-id="1bf62-753">そのため、基本的な IP サービスを使用すると、ネットワークが高品質の音声を提供するのに十分なパフォーマンスを得て、FEC や回復性の高い音声コーディングのような手法を使用して確率を向上できると期待できますが、MPLS を使用することで確実に実現できます。</span><span class="sxs-lookup"><span data-stu-id="1bf62-753">So with basic IP service we can hope that the network will perform well enough to provide good quality voice and use techniques like FEC and more resilient voice coding to improve the odds, but by using MPLS, we can be sure of it.</span></span>
