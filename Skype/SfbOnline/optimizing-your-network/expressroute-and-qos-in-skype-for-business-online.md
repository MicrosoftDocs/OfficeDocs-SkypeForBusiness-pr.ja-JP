<span data-ttu-id="df28e-743">マルチプロトコル ラベル の切り替え (MPLS) は、ネットワーク サービス プロバイダーが遅延、ジッター、パケット損失のパフォーマンスを保証する IP サービスを提供するための手段として開発されました。</span><span class="sxs-lookup"><span data-stu-id="df28e-743">Multi-Protocol Label Switching (MPLS) was developed as a means for network service providers to offer an IP service with performance guarantees for delay, jitter and packet loss.</span></span> <span data-ttu-id="df28e-744">これらのパフォーマンスの保証を実現するために、MPLS は予測不可能性の一部を従来の IP から取り出します。</span><span class="sxs-lookup"><span data-stu-id="df28e-744">To deliver on those performance guarantees, MPLS takes some of the unpredictability out of traditional IP.</span></span> <span data-ttu-id="df28e-745">1 つ目は、各パケットが宛先へのルーター間の経路を見つけ出すのではなく (その結果、各パケットがソースから宛先への異なるルートを受け取る可能性がある)、MPLS はラベル切り替えパス (LSP) と呼ばれる固定ルートを持つ "仮想回線" 接続上のすべてのパケットをルーティングします。</span><span class="sxs-lookup"><span data-stu-id="df28e-745">First, rather than having each packet find its way router-to-router to its destination (the result of which could be that each packet takes a different route from the source to the destination), MPLS routes all of the packets on a "virtual circuit" connection with a fixed route called a Label Switched Path (LSP).</span></span> <span data-ttu-id="df28e-746">そのパス内のリンクの 1 つが失敗した場合、そのリンクを使用しているすべての LSP がすぐに再ルーティングされます。</span><span class="sxs-lookup"><span data-stu-id="df28e-746">If one of the links in that path fails, all of the LSPs using that link are quickly rerouted.</span></span>
  
<span data-ttu-id="df28e-747">パケットが MPLS ネットワークに送信される場合、ネットワーク サービス プロバイダーのエッジ ルーターは、適切な LSP を使用してパケットを転送するために使用されるラベルを含む追加のヘッダーをパケットに追加します。</span><span class="sxs-lookup"><span data-stu-id="df28e-747">When a packet is sent into the MPLS network the network service provider's edge router appends an additional header to the packet that includes a label that is used to forward it over the appropriate LSP.</span></span> <span data-ttu-id="df28e-748">ラベルは、MPLS ネットワークのもう一方の端にあるエッジ ルーターによって取り除かされます。</span><span class="sxs-lookup"><span data-stu-id="df28e-748">The label is stripped off by the edge router at the other end of the MPLS network.</span></span>
  
<span data-ttu-id="df28e-749">転送プロセスの簡素化に加え、MPLS が提供するもう 1 つの利点は、ネットワーク管理システムがネットワーク内のすべてのリンクでどの接続が行われるかを知るという利点です。</span><span class="sxs-lookup"><span data-stu-id="df28e-749">Beside simplifying the forwarding process, the other advantage MPLS provides is that the network management system will know what connections are being carried on every link in the network.</span></span> <span data-ttu-id="df28e-750">トラフィックがネットワーク経由でルーティングされる方法を制御することで、オペレーターは各パスで提供される QoS を保証できます。</span><span class="sxs-lookup"><span data-stu-id="df28e-750">By controlling the way traffic is routed through the network, the operator can guarantee the QoS each path will provide.</span></span> <span data-ttu-id="df28e-751">そのため、従来の IP または基本的な IP のベスト 努力のパフォーマンスとは異なり、MPLS オペレーターは予測可能なパフォーマンスで IP サービスを提供できます。</span><span class="sxs-lookup"><span data-stu-id="df28e-751">So unlike the best effort performance of traditional or basic IP, MPLS operators can provide an IP service with predictable performance.</span></span> <span data-ttu-id="df28e-752">また、この LSP により、MPLS は従来のインターネット サービスよりも本質的にセキュリティで保護されます。</span><span class="sxs-lookup"><span data-stu-id="df28e-752">That LSP also makes MPLS inherently more secure than traditional Internet services.</span></span> <span data-ttu-id="df28e-753">そのため、基本的な IP サービスを使用すると、ネットワークが高品質の音声を提供し、FEC や回復力のある音声コーディングのような手法を使用して確率を向上させるのに十分なパフォーマンスを得る必要がありますが、MPLS を使用することで確実に実現できます。</span><span class="sxs-lookup"><span data-stu-id="df28e-753">So with basic IP service we can hope that the network will perform well enough to provide good quality voice and use techniques like FEC and more resilient voice coding to improve the odds, but by using MPLS, we can be sure of it.</span></span>
  
<span data-ttu-id="df28e-754">MPLS プロバイダーは、サービス グラデーションのいくつかのクラスを提供します。残念ながら、異なる用語を使用してそれらを識別します。</span><span class="sxs-lookup"><span data-stu-id="df28e-754">MPLS providers offer several class of service gradients that unfortunately uses different terms to identify them.</span></span> <span data-ttu-id="df28e-755">[Lync 2010 および 2013 Bandwidth Calculator](https://go.microsoft.com/fwlink/?LinkID=690282)からの出力と、さまざまな Microsoft 365 または Office 365 Real Time Workloads アプリケーションに推奨されるオプションを理解するには、プロバイダーと密接に作業する必要があります。</span><span class="sxs-lookup"><span data-stu-id="df28e-755">You will have to work closely with your provider to ensure they understand the outputs from the [Lync 2010 and 2013 Bandwidth Calculator](https://go.microsoft.com/fwlink/?LinkID=690282) and the recommended options for different Microsoft 365 or Office 365 Real Time Workloads applications.</span></span>
