# <a name="session-details-report"></a>セッションの詳細レポート

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

[**レポート]** ダッシュボードには、組織内の製品のMicrosoft 365またはOffice 365アクティビティの概要が表示されます。 このダッシュボードでは、個々の製品レベルのレポートまで掘り下げ、各製品内のアクティビティについてより細かい洞察を得ることができます。 たとえば、[セッションの詳細] **Skype for Businessを** 使用して、個々のユーザーの通話エクスペリエンスに関する詳細を確認できます。
  
利用可能な [その他のレポート](https://support.office.com/article/0d6dfb17-8582-4172-a9a9-aed798150263) については、「レポートの概要」を参照してください。
  
このレポートと他のレポートSkype for Business、組織全体のセッションの詳細を含むアクティビティの詳細が表示されます。 これらの詳細は、組織に対して調査、計画、その他のビジネス上の決定を行う場合や、通信クレジット を設定する場合に [非常に役立ちます](/microsoftteams/what-are-communications-credits)。
  
> [!NOTE]
> 管理者として管理センターにログオンSkype for Business、すべてのレポートをMicrosoft 365できます。 
  
## <a name="how-to-get-to-the-skype-for-business-session-details-report"></a>セッションの詳細レポートをSkype for Businessする方法

1. 管理センターの [レポート] > **する**
    
2. 左側の **メニューから [** レポート] を選択し、[使用状況] を **クリックします**。
    
3. [レポートの選択 **] の一覧で、[** セッション **の詳細] Skype for Business クリックします**。
    
    > [!TIP]
    > このレポートが一覧表示しない場合は、管理センターの [**セッションの詳細** Skype for Business  >  **レポート] に**  >  **移動します**。 
  
    > [!IMPORTANT]
    > サブスクリプションのMicrosoft 365またはOffice 365によっては、ここに表示される製品とレポートの一部が表示されない場合があります。

表示される各列を見Skype for Business、ユーザーのセッションの詳細を確認できます。
  
レポートは、このように表示されます。

![ナンバー 1](../images/sfbcallout1.png)<br/>**エイリアスでユーザーを** 検索すると、1 人のユーザーを検索し、次の表にユーザーのすべてのセッションの詳細を表示できます。

![ナンバー 2](../images/sfbcallout2.png)<br/>**日付の時刻を入力** すると、開始日を入力できます。 予定表を使用して、日付を選択したり、日付を手動で入力したりすることができます。 このフィールドを設定する必要があります。

![ナンバー 3](../images/sfbcallout3.png)<br/>**「date time」と入力** すると、終了日を入力できます。 予定表を使用して、日付を選択したり、日付を手動で入力したりすることができます。 終了日が設定されている場合、既定値は開始日から 30 日です。
***
![数値 4](../images/sfbcallout4.png)<br/>次の表は、ユーザーごとのすべてのセッションの詳細の内訳を示しています。 これにより、ユーザーに割り当てられているSkype for Businessユーザーとそのセッション情報が表示されます。 表では、列を追加したり、削除したりすることができます。 <br/><br/>このテーブルには、セッションごとに次の列があります。
*    **ダイアログ ID** は、SIP セッションの一意識別子の ID です。
*    **メディアの種類の説明** では、セッションが電話会議か P2P セッションか、使用されるメディアの種類 (オーディオ/ビデオ/アプリケーション共有) が記述されます。
*    **開始時刻** は、セッションが開始された時刻です。
*    **終了時刻** は、セッションが終了した時刻です。
*    **FROM URI** は、セッションを開始したユーザーまたはサービスの URI です。 ユーザーが PSTN 電話からセッションを開始した場合、空白になる可能性があります。
*    **[URI** へ] は、セッション開始のターゲットだったユーザーまたはサービスの URI です。 会議の場合、これは開催者の URI です。 セッションのターゲットが PSTN 電話番号の場合は空白になる可能性があります。
*    **クライアント バージョンから** 、ユーザー エージェントと、セッションを開始したユーザーまたはサービスによって使用されるクライアントのバージョンが示されます。
*    **クライアントバージョンには** 、ユーザー エージェントと、セッション開始のターゲットだったユーザーまたはサービスによって使用されるクライアントのバージョンが示されます。
*    **[会議 URL]** は、会議の SIP URL です (セッションが電話会議の場合)。 同じ電話会議のすべてのユーザーが同じ電話会議 URL を持つ。 
*    **[電話番号から** ] は、該当する場合は、セッションのターゲットだった電話番号です。 ユーザーのプライバシーを保護するために、電話番号の最後の数字を "x" に置き換える場合があります。
*    **[電話番号へ** ] は、該当する場合は、セッションの対象だった電話番号です。 ユーザーのプライバシーを保護するために、電話番号の最後の数字を "x" に置き換える場合があります。
*    **エンドポイント ID は** 、From ユーザーによって使用されるエンドポイントの一意の GUID です。 ユーザーが同じエンドポイントから複数のセッションを通信するかどうかを識別するために使用します。 ユーザーが PSTN 電話を使用している場合、またはセッションがサービスから開始された場合は、空白になる可能性があります。
*    **To Endpoint Id は** 、To ユーザーによって使用されるエンドポイントの一意の GUID です。 ユーザーが同じエンドポイントから複数のセッションを通信するかどうかを識別するために使用します。 ユーザーが PSTN 電話を使用している場合、セッションがサービスから開始された場合、またはセッションが確立に失敗した場合は、空白になる可能性があります。
*    **Conf Instance** は、会議 URL を使用した会議のインスタンスの一意の GUID です。 定期的な会議の会議 URL は同じですが、会議のインスタンスごとに Conf Instance が異なっています。
*    **[Uri の代理** ] は、セッションの代理が確立されている側のデレガターの URI です。 <br/> **URI によって参照** されるのは、セッションの確立を参照したユーザーの URI です。
*    **応答コード** は、セッションが正常に確立されたかどうかを示すセッションを確立する SIP 応答コードです。

セッションごとに、シナリオに応じて異なるデータを含むサブ テーブルがあります。 次に、サブ テーブルの [From] と [To] のユーザーまたはサービスで使用できるタブを示します。
*    [セッション] タブには、マシンとオペレーティング システムに関するデータが表示されます。
*    [MEDIALINES] タブには、ネットワーク接続情報とデバイス情報が表示されます。
*    [AUDIOSTREAMS] タブには、セッションに関係するオーディオ ストリームに関するネットワーク パフォーマンス データが表示されます。
*    [AUDIOCLIENTEVENTS] タブには、オーディオ エクスペリエンスに影響を与えるクライアントが検出した問題に関するデータが表示されます。
*    [AUDIOSIGNALS] タブには、セッションのオーディオ信号処理に関するデータが表示されます。
*    [APPSHARINGSTREAMS] タブには、セッションに関係するアプリケーション共有ストリームまたはデスクトップ共有ストリームに関するネットワーク パフォーマンス データが表示されます。
*    VIDEOCLIENTEVENTS タブには、ビデオ エクスペリエンスに影響を与えるクライアントが検出した問題に関するデータが表示されます。
*    [VIDEOSTREAMS] タブには、セッションに関連するビデオ ストリームに関するネットワーク パフォーマンス データが表示されます。
*    [TRACEROUTES] タブには、セッション中に traceroute 経由で収集されたネットワーク ホップが表示されます。 セッションに使用される実際のメディア パスは異なる場合があります。このデータは、セッションにオーディオがある場合にのみ使用できます。
*    FEEDBACKREPORTS タブには、セッション内のユーザーによって提供された通話終了アンケート データが表示されます。
