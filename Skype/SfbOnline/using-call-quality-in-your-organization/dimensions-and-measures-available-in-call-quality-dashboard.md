# <a name="dimensions-and-measures-available-in-call-quality-dashboard"></a>ディメンションとメジャーが通話品質のダッシュ ボードで利用できます。

Microsoft チームの通話品質のダッシュ ボード (CQD) と Skype for Business Online を使用すると、Business services 用 Microsoft チームと Skype を使用して通話の品質をより深く洞察を取得できます。ここでは、ディメンションとメジャー CQD で表示されるに関する詳細情報を紹介します。CQD と有効にする方法についての詳細については、[有効にして Microsoft チームと Skype for Business Online の品質ダッシュ ボードの通話を使用する](turning-on-and-using-call-quality-dashboard.md)を参照してください。
  
## <a name="first-and-second-endpoint-classification"></a>最初と 2 番目のエンドポイントの分類

多くのディメンションとメジャー通話品質のダッシュ ボード (CQD) では、または 2 番目に分類されます。次のロジックを決定する端点がストリームに関連するまたは通話が最初にラベルが付いた。
  
- サーバーがストリーム/通話中に含まれている場合、サーバーのエンドポイント (AV MCU、仲介サーバーなど) に、必ず最初されます。
    
- 2 番目は必ずクライアント エンドポイントをストリームで 2 つのサーバーの端点の間でない限り。
    
- 場合、両方のエンドポイントは同じ種類の順序は、最初と 2 番目に基づいていますユーザー エージェントのカテゴリの内部順序。これにより、一貫性のある順序付けはします。

### <a name="notes-on-dimension-data-typeunits"></a>ディメンション データ型の単位の注意点:

#### <a name="range"></a>範囲

範囲またはグループの値として提供されているサイズは、次の形式は表示されます。
  
 _<sort order string>[<lower bound inclusive> -< 排他モードでの上限)_
  
たとえば、時間 (分) ディメンションは、値の範囲で報告された値を秒数の通話時間を表します。
  
|||

|064: [1-2)  <br/> |1 分 < = ストリーム期間 < 2 分  <br/> |
|065: [2-3)  <br/> |2 分 < = ストリーム期間 < 3 分  <br/> |
|066: [3-4)  <br/> |3 分 < = ストリーム期間 < 4 分  <br/> |
|…  <br/> |…  <br/> |
   
<sort order string>データを表示するために、並べ替え順序を制御するために使用し、フィルター処理するために使用することができます。たとえば、時間 (分) のフィルター <「065」は期間 (想定どおりに動作するフィルターの先頭に「0」必要です) より小さい 2 分ストリームを表示します。
  
> [!NOTE]
> 並べ替え順序の文字列の実際の値が大幅に表示されません。
