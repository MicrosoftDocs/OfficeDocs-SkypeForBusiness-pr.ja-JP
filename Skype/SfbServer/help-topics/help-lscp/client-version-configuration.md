# <a name="client-version-configuration"></a>クライアント バージョンの構成
 
環境内でサポートするクライアントのバージョンを指定するだけでなく、バージョン ポリシーを定義していないクライアントの既定のアクションを指定することも。 これにより、お客様の環境では、複数のクライアント バージョンのサポートに伴うコストを制御することができますで使用するクライアント バージョンを制限することができます。
  
## <a name="tasks-you-can-perform"></a>実行できるタスク

**クライアント バージョンの構成**] ページで、次のタスクを実行できます。
  
- (**グローバル**) の既定のクライアント バージョンの構成を編集します。
    
- 特定のサイトのクライアント バージョンの構成を作成します。
    
- 有効にして、既存のクライアント バージョンの構成を無効にします。
    
> [!NOTE]
> 匿名ユーザーがサイトに関連付けられていないため、グローバル レベルのポリシーだけでは匿名ユーザーが影響を受けます。 
  
## <a name="ui-reference"></a>UI リファレンス

以下は、ページ上のメニュー、コマンド、フィールド、およびプロパティについての説明です。
  
- **新しい**特定のサイトのクライアント バージョンの構成を作成することができます。
    
- **編集**クライアント バージョン ポリシーのいずれかのオプションを変更することができます。 このオプションを使用すると、以下を実行できます、します。
    
  - **詳細を表示します。** このオプションは、クライアント バージョンの構成のオプションを変更できるダイアログ ボックスを開きます。
    
  - **すべてを選択**このオプションでは、一覧のすべてのクライアント バージョンの構成を選択します。
    
  - **削除**このオプションは、選択したクライアントのバージョンのすべての構成を削除します。
    
- **更新**クライアント バージョンの構成] ボックスの一覧のすべてのクライアント バージョンの構成オプションの状態を確認するのにを更新することができます。
    
クライアントとクライアントのバージョン間での相互運用性に関する詳細については、計画ドキュメントに[Lync 2013 プレビューでのクライアントの相互運用性](http://technet.microsoft.com/library/0f126571-91a2-45d5-855c-1e4ddb45fc04.aspx)を参照してください。 クライアント バージョンの構成の操作に関する詳細については、操作マニュアルの[変更クライアントに明示的にサポートされていないまたは制限付きサイトの既定のアクション](http://technet.microsoft.com/library/548dd0f5-62fe-4c3f-8952-2b9fd4c5fff3.aspx)を参照してください。
