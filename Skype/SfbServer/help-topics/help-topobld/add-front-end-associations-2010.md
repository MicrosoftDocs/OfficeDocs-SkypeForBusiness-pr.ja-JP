# <a name="add-front-end-associations-2010"></a>フロントエンドの関連付けの追加 (2010)

この時点で、サーバーの役割をフロントエンドプールに関連付けて、他のサーバーの展開を必要とする特定の機能のサポートを有効にすることができます。 後でサーバーの役割をフロントエンド プールに関連付けることもできます。 フロントエンド プールに関連付けることができるサーバーの役割には、次のものがあります。

1. アーカイブサーバー。 アーカイブサポートの計画と実装の詳細については、「計画ドキュメントに[アーカイブのための組織の要件を定義](https://technet.microsoft.com/library/ce0fc0f6-7704-4b80-bf19-a1fa9818fc7a.aspx)する」および「[フロントエンドプールまたは Standard Edition サーバーを定義して構成](https://technet.microsoft.com/library/713fc263-23dd-414a-b001-82932e4fe966.aspx)する」および「展開ドキュメントに[アーカイブを展開](https://technet.microsoft.com/library/a89edd16-12d5-4602-ad2f-194b47d1188e.aspx)する」を参照してください。

2. サーバーを監視します。 監視サポートの実装の詳細については、「計画ドキュメントで[監視の計画を立てる](https://technet.microsoft.com/library/26cead5a-183c-42f1-a4b0-0e8d61c6159d.aspx)」および「[フロントエンドプールまたは Standard Edition サーバーを定義して構成](https://technet.microsoft.com/library/713fc263-23dd-414a-b001-82932e4fe966.aspx)する」を参照して、展開ドキュメントで[監視を展開](https://technet.microsoft.com/library/117f4a3e-0670-4388-a553-b9854921145f.aspx)してください。

3. A/V Edge サービス。 A/V Edge サービスの実装の詳細については、「[外部ユーザーアクセスの概要](https://technet.microsoft.com/library/97aded6c-5fa3-4225-95a6-9ad094d61654.aspx)」を参照してください。
