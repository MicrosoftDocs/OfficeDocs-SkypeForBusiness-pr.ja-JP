# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Lync Server 2010 用のフロント エンド全般設定エキスパンダー

次の属性を編集または構成して、フロントエンド サーバーまたはフロントエンド プールのプロパティを編集します。 構成ページは、次のセクションに分かれています。

 **全般**

- **FQDN**: フロントエンド サーバーまたはフロントエンド プールの完全修飾ドメイン名。

- [ **すべての構成済み IP アドレスを使用** する] を選択して、フロントエンド サーバーまたはフロントエンド プールで構成されたアドレスを使用します。

    > [!IMPORTANT]
    > 仲介サーバーをフロントエンド サーバーまたはフロントエンド プールに共に使用する場合は、このオプションを選択する必要はありません。 仲介サーバーとフロントエンド サーバーには、通信に使用する専用の IP アドレスが必要です。

- [ **サービスの使用を** 選択した IP アドレスに制限する] を選択し、展開の残りの部分と通信するフロントエンド サーバーまたはフロントエンド プールのプライマリ **IP** アドレスの IP アドレスを入力します。 PSTN IP **アドレスに、仲介** サーバーに関連付けられている IP アドレスを入力します。

    [**機能**]

- [**電話会議**]: 展開で電話会議機能を使用する場合、このチェック ボックスをオンにします。 電話会議には、音声、ビデオ、アプリケーション共有、デスクトップ共有、および Web 会議が含まれます。 Web 会議用 Web Apps サーバーを作成Office関連付ける必要があります (このプロパティ ページで後で定義します)。

- [電話会議] をオンにした場合、[**ダイヤルイン (PSTN) 会議**] をオンにすることができます。 ダイヤルイン会議の機能を有効にするには、このチェック ボックスをオンにします。

- Lync Server 2013 がボイス オーバー IP (VoIP) テクノロジを使用して電話音声システムとして機能する機能を展開する場合は、設計と要件に基づいて、仲介サーバー、PSTN ゲートウェイ、IP-PBX を組み合わせてまたは単独で使用して、ハンドセット電話、SIP トランク、または公衆交換電話網接続を展開するオプションを含む、機能を展開する場合は、チェック ボックス **エンタープライズ VoIP** を選択します。 Skype for Business Server 2015[エンタープライズ VoIPのエンタープライズ VoIP](https://technet.microsoft.com/library/c9da8099-6f4f-4346-ac67-f041bb96072c.aspx)とプランのエンタープライズ VoIP詳細については[、「Skype for Business Server 2015」](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md)を参照してください。

    **関連付け**

- **SQL Serverストア**: フロントエンド サーバーまたはフロントエンド SQL Server関連付けられているサーバーの FQDN (および必要に応じて名前付きインスタンス)。 リストから SQL Server ストアを選択するか、または、[**新規**] をクリックして新しい SQL Server ストアを作成します。

- **ファイル** ストア : Lync Server 2013 が作成し、レプリケーション、会議ディレクトリ、その他の目的で使用する共有ファイルのファイル ストアの場所として機能するサーバーと共有の FQDN (形式) を選択します。 `\\<FQDN of server>\<share name>` 一覧からファイル ストアを選択するか、[新規] をクリックして新しいファイル ストアを **作成します**。

- [アーカイブ **サーバーの関連付** け] チェック ボックスをオンにして、このフロントエンド サーバーまたはフロントエンド プールのアーカイブ サーバーを有効にします。 このチェック ボックスをオンにした後、リストから既存のアーカイブ サーバーを選択するか、[新規] をクリックして新しいアーカイブ サーバーの定義を作成します。

- [監視 **サーバーの関連付** け] チェック ボックスをオンにして、このフロントエンド サーバーまたはフロントエンド プールの監視サーバーを有効にします。 このチェック ボックスをオンにした後、リストから既存の監視サーバーを選択するか、[新規] をクリックして新しい監視サーバーの定義を作成します。

- このフロントエンド **サーバーまたはフロントエンド プール** でエッジ サーバーを有効にするには、[エッジ プールの関連付け] (メディア コンポーネントの場合) チェック ボックスをオンにします。 このチェック ボックスをオンにした後、一覧から既存のエッジ サーバーまたはプールを選択するか、[新規] をクリックして新しいエッジ サーバーまたはプールの定義を作成します。

  **復元性**

- [関連付けられたバックアップ レジストラー プール] チェック ボックスをオンにして、バックアップ レジストラーになるフロントエンド サーバーまたはフロントエンド プール (プライマリに障害が発生した場合にセカンダリ レジストラーとして指定されるフロントエンド サーバーまたはフロントエンド プール) を一覧から選択します。

- [関連付けられているバックアップ レジストラー プール] をオンにして、バックアップ レジストラーを選択した場合、[**音声に対する自動フェールオーバーとフェールバック**] のチェックボックスをオンにすることができます。 この場合、[**音声のエラー検出間隔 (秒)**] および [**音声のフェールバック間隔 (秒)**] の数値プロパティを定義できます。 詳細については、「[Planning for Enterprise Voice Resiliency](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx)」を参照してください。

  [**Web サービス**]

- [**内部 Web サービス**] を構成するには、**HTTP** および **HTTPS** 用の [**リッスン ポート**] を定義します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。また、**HTTP** および **HTTPS** 用の [**公開ポート**] を構成します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。内部 Web サービスの構成およびロード バランサー (ロード バランサー機器および DNS 負荷分散) の使用に基づいて、ポート値を調節して、リッスン ポートおよび公開ポートを定義します。

    > [!IMPORTANT]
    > 内部 Web サービスおよび定義したリッスン ポートと公開ポートは、内部クライアントおよび内部デバイス用です。外部クライアントおよび外部デバイスは、外部 Web サービスのリッスン ポートと公開ポートを、定義済み外部 Web サービスの完全修飾ドメイン名 (FQDN) とともに使用します。

- [**外部 Web サービス**] を構成するには、**HTTP** および **HTTPS** 用の [**リッスン ポート**] を定義します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。また、**HTTP** および **HTTPS** 用の [**公開ポート**] を構成します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。内部 Web サービスの構成およびロード バランサー (ロード バランサー機器および DNS 負荷分散) の使用に基づいて、ポート値を調節して、リッスン ポートおよび公開ポートを定義します。

    > [!IMPORTANT]
    > 外部 Web サービスと定義済みのリッスン ポートと公開ポートは、外部クライアントとデバイス用です。 外部クライアントとデバイスは、外部 Web サービスのリッスン ポートと公開ポートを使用します。通常は、リバース プロキシによって定義され、定義済みの外部 Web サービスの完全修飾ドメイン名 (FQDN) と共に定義されます。 外部 Web サービスの FQDN と簡易 URL の関係により、外部ユーザーとデバイスが利用できるサービスに外部クライアントがアクセスするために使用する URL (Uniform Resource Locator) アドレスが定義されます。 簡易 URL の詳細については、「簡易 URL の計画 [」を参照してください](https://technet.microsoft.com/library/20e4f4b6-b7ff-4297-b00d-d1211ee800ac.aspx)。

  **仲介サーバー**

- 仲介サーバー **の** プロパティを、同じ仲介サーバー (フロントエンド サーバーまたはフロントエンド プールに展開された仲介サーバー) に対して構成するには、[配置された仲介サーバーを有効にする] を選択 **します**。

- 同じ仲介 **サーバーの** リッスン ポートを定義するには、その仲介サーバーがリッスンしている **TLS** および **TCP** ポートの値を入力します。 既定では、TLS は TCP ポート 5067 と定義されています。

- 仲介サーバーの TCP ポート値を定義するには、[TCP ポートを有効にする] **チェック ボックスを** オンにします。 既定では、仲介サーバーは TCP プロトコル上のトランスポート層セキュリティ (TLS) を使用します。 TCP ポートは、[TCP ポートを有効にする] がオンになっている場合のみ利用できます。

    > [!NOTE]
    > これは必要に応じて設定します。この設定が必要かどうかを判断するには、ゲートウェイまたは PSTN の要件を参照する必要があります。既定では、TCP ポート値は 5068 です。

- 併置された仲介サーバーに関連付けられたトランクを定義します。すでにトランクを定義している場合、仲介サーバーに関連付けることができます。

    仲介サーバーに関連付けられたゲートウェイが複数ある場合は、既定にするゲートウェイを選択し、[既定にする] をクリックして、既定のゲートウェイを **指定できます**。 現在の既定のゲートウェイを削除する場合は、ゲートウェイを選択して、[**既定値の解除**] をクリックします。

> [!IMPORTANT]
> このダイアログでプロパティを変更する場合は、トポロジを公開し、影響を受けるすべてのサーバーで Skype for Business Server 展開ウィザードを実行する必要があります。 新しいトポロジを公開した後、Skype for Business Server 展開ウィザードを実行する必要がある影響を受けるサーバーの一覧が、正常なトポロジ公開の概要画面のリンクとして提供されます。 更新したトポロジの公開に関する詳細については、「[Publish the Topology](https://technet.microsoft.com/library/3b5a744b-b3a8-4538-a55e-e2e4f72dff47.aspx)」を参照してください。 Skype for Business Server 展開ウィザードの詳細については [、「Lync Server Administrative Tools」を参照してください](https://technet.microsoft.com/library/9b006f93-4f3d-461d-89b8-e80a34fdb3c5.aspx)。

トポロジ ドキュメントへの変更を保存して確定するには、[**OK**] をクリックします。

変更 **を破棄** し、フロントエンド サーバーまたはフロントエンド プールの **[** プロパティの編集] を閉じるには、[キャンセル] をクリックします。

このヘルプ トピックを読むには、[**ヘルプ**] をクリックします。
