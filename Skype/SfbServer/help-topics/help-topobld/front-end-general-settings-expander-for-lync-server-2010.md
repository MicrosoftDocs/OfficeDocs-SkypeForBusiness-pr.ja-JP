# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Lync Server 2010 用のフロントエンド全般設定の展開

フロントエンドサーバーまたはフロントエンドプールのプロパティを編集するには、次の属性を編集または構成します。 構成ページは、次のセクションに分かれています。

 **全般**

- **FQDN**: フロントエンドサーバーまたはフロントエンドプールの完全修飾ドメイン名。

- [ **すべての構成済み IP アドレスを使用** する] を選択して、フロントエンドサーバーまたはフロントエンドプールに構成されたすべてのアドレスを使用するようにします。

    > [!IMPORTANT]
    > フロントエンドサーバーまたはフロントエンドプールに仲介サーバーを併置する場合は、このオプションを選択しないでください。 仲介サーバーとフロントエンドサーバーには、通信する専用の IP アドレスが必要です。

- [ **選択した IP アドレスに対してサービスの利用を制限する** ] を選択し、フロントエンドサーバーまたはフロントエンドプールの他の展開との通信に使用する **プライマリ IP** アドレスの ip アドレスを入力します。 [ **PSTN IP アドレス** の入力仲介サーバーに関連付けられている ip アドレスを入力します。

    [**機能**]

- [**電話会議**]: 展開で電話会議機能を使用する場合、このチェック ボックスをオンにします。 電話会議には、音声、ビデオ、アプリケーション共有、デスクトップ共有、および Web 会議が含まれます。 Web 会議用の Office Web Apps サーバーを作成して関連付ける必要があります (このプロパティページで後で定義されます)。

- [電話会議] をオンにした場合、[**ダイヤルイン (PSTN) 会議**] をオンにすることができます。 ダイヤルイン会議の機能を有効にするには、このチェック ボックスをオンにします。

- 機能を展開する場合は、チェックボックス **エンタープライズ** voip を選択します。これにより、Lync server 2013 をボイスオーバー IP (VoIP) テクノロジを使用して電話音声システムとして動作させることができます。これには、仲介サーバー、PSTN ゲートウェイ、および ip-pbx を使用した、設計と要件に基づいた電話回線の接続があります エンタープライズ Voip の詳細については、「 [enterprise](https://technet.microsoft.com/library/c9da8099-6f4f-4346-ac67-f041bb96072c.aspx) voice」および「 [Plan for Enterprise voice In Skype for business Server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md) 」を参照してください。

    **連盟**

- **Sql server ストア**: フロントエンドサーバーまたはフロントエンドプールに関連付けられている sql SERVER の FQDN (および必要に応じて名前付きインスタンス)。 リストから SQL Server ストアを選択するか、または、[**新規**] をクリックして新しい SQL Server ストアを作成します。

- [**ファイルストア**]: `\\<FQDN of server>\<share name>` Lync server 2013 によって作成され、レプリケーション、会議ディレクトリ、その他の目的に使用される共有ファイルのファイルストアの場所として機能するサーバーおよび共有 (形式) の FQDN を選択します。 リストからファイルストアを選択するか、[ **新規**] をクリックして新しいファイルストアを作成します。

- [ **アーカイブサーバーの関連付け** ] チェックボックスをオンにして、このフロントエンドサーバーまたはフロントエンドプールのアーカイブサーバーを有効にします。 チェックボックスをオンにした後、一覧から既存のアーカイブサーバーを選択するか、[ **新規** ] をクリックして新しいアーカイブサーバーの定義を作成します。

- [ **監視サーバーの関連付け** ] チェックボックスをオンにして、このフロントエンドサーバーまたはフロントエンドプールの監視サーバーを有効にします。 チェックボックスをオンにした後、一覧から既存の監視サーバーを選択するか、[ **新規** ] をクリックして新しい監視サーバーの定義を作成します。

- [ **エッジプールの関連付け (メディアコンポーネント用** )] チェックボックスをオンにして、このフロントエンドサーバーまたはフロントエンドプールのエッジサーバーを有効にします。 チェックボックスをオンにした後、リストから既存のエッジサーバーまたはプールを選択するか、[ **新規** ] をクリックして新しいエッジサーバーまたはプールの定義を作成します。

  **性**

- [ **関連付けられたバックアップレジストラープール** ] チェックボックスをオンにして、バックアップレジストラーとなるフロントエンドサーバーまたはフロントエンドプール (プライマリが失敗した場合にセカンダリレジストラーとして指定されたフロントエンドサーバーまたはフロントエンドプール) を一覧から選択します。

- [関連付けられているバックアップ レジストラー プール] をオンにして、バックアップ レジストラーを選択した場合、[**音声に対する自動フェールオーバーとフェールバック**] のチェックボックスをオンにすることができます。 この場合、[**音声のエラー検出間隔 (秒)**] および [**音声のフェールバック間隔 (秒)**] の数値プロパティを定義できます。 詳細については、「[Planning for Enterprise Voice Resiliency](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx)」を参照してください。

  [**Web サービス**]

- [**内部 Web サービス**] を構成するには、**HTTP** および **HTTPS** 用の [**リッスン ポート**] を定義します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。また、**HTTP** および **HTTPS** 用の [**公開ポート**] を構成します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。内部 Web サービスの構成およびロード バランサー (ロード バランサー機器および DNS 負荷分散) の使用に基づいて、ポート値を調節して、リッスン ポートおよび公開ポートを定義します。

    > [!IMPORTANT]
    > 内部 Web サービスおよび定義したリッスン ポートと公開ポートは、内部クライアントおよび内部デバイス用です。外部クライアントおよび外部デバイスは、外部 Web サービスのリッスン ポートと公開ポートを、定義済み外部 Web サービスの完全修飾ドメイン名 (FQDN) とともに使用します。

- [**外部 Web サービス**] を構成するには、**HTTP** および **HTTPS** 用の [**リッスン ポート**] を定義します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。また、**HTTP** および **HTTPS** 用の [**公開ポート**] を構成します。既定では、それぞれ、TCP ポート 80 および TCP ポート 443 です。内部 Web サービスの構成およびロード バランサー (ロード バランサー機器および DNS 負荷分散) の使用に基づいて、ポート値を調節して、リッスン ポートおよび公開ポートを定義します。

    > [!IMPORTANT]
    > 外部 web サービス、および定義されているリッスンおよび公開ポートは、外部クライアントとデバイスに対して使用されます。 外部クライアントおよびデバイスでは、通常、リバースプロキシによって定義された、外部 web サービスによって定義された外部 web サービスの完全修飾ドメイン名 (FQDN) を使用します。 外部 web サービスの FQDN と簡易 Url の関係は、外部クライアントが外部ユーザーおよびデバイスで利用可能なサービスにアクセスするために使用する URL (uniform resource locator) アドレスを定義します。 簡易 Url の詳細については、「 [Planning For Simple urls](https://technet.microsoft.com/library/20e4f4b6-b7ff-4297-b00d-d1211ee800ac.aspx)」を参照してください。

  **仲介サーバー**

- 併置された仲介サーバー (つまり、フロントエンドサーバーまたはフロントエンドプールに展開された仲介サーバー) の **仲介サーバー** のプロパティを構成するには、[併置された **仲介サーバーが有効**] を選択します。

- 併置された仲介サーバーの **リッスンポート** を定義するには、併置された仲介サーバーがリッスンしている **TLS** および **TCP** ポートの値を入力します。 既定では、TLS は TCP ポート 5067 と定義されています。

- 仲介サーバーの TCP ポート値を定義するには、[ **tcp ポートを有効に** する] チェックボックスをオンにします。 既定では、仲介サーバーは TCP プロトコル経由のトランスポート層セキュリティ (TLS) を使用します。 TCP ポートは、[TCP ポートを有効にする] がオンになっている場合のみ利用できます。

    > [!NOTE]
    > これは必要に応じて設定します。この設定が必要かどうかを判断するには、ゲートウェイまたは PSTN の要件を参照する必要があります。既定では、TCP ポート値は 5068 です。

- 併置された仲介サーバーに関連付けられたトランクを定義します。すでにトランクを定義している場合、仲介サーバーに関連付けることができます。

    仲介サーバーに複数のゲートウェイが関連付けられている場合は、既定のゲートウェイを選択し、[ **既定**にする] をクリックすることで、既定のゲートウェイを指定できます。 現在の既定のゲートウェイを削除する場合は、ゲートウェイを選択して、[**既定値の解除**] をクリックします。

> [!IMPORTANT]
> このダイアログのプロパティを変更する場合は、トポロジを公開し、影響を受けるすべてのサーバーで Skype for Business Server 展開ウィザードを実行する必要があります。 新しいトポロジを公開した後、[トポロジの公開の概要を成功] 画面のリンクとして、Skype for Business Server 展開ウィザードを実行する必要がある影響を受けるサーバーの一覧が表示されます。 更新したトポロジの公開に関する詳細については、「[Publish the Topology](https://technet.microsoft.com/library/3b5a744b-b3a8-4538-a55e-e2e4f72dff47.aspx)」を参照してください。 Skype for Business Server の展開ウィザードの詳細については、「 [Lync Server 管理ツール](https://technet.microsoft.com/library/9b006f93-4f3d-461d-89b8-e80a34fdb3c5.aspx)」を参照してください。

トポロジ ドキュメントへの変更を保存して確定するには、[**OK**] をクリックします。

変更を破棄してフロントエンドサーバーまたはフロントエンドプールの**プロパティの編集**を閉じるには、[**キャンセル**] をクリックします。

このヘルプ トピックを読むには、[**ヘルプ**] をクリックします。
