# <a name="mobile-client-push-notification-configuration"></a>モバイル クライアント: プッシュ通知の構成
 
**Microsoft プッシュ通知**と**Apple プッシュ**通知を構成するには、必要なプッシュ通知の種類を定義するポリシーを作成する必要があります。
  
メイン構成画面で、[**更新**] をクリックして、ポリシーの一覧を更新して再設定することができます。 表示されるポリシーの一覧を絞り込むための検索ボックスが用意されています。 検索する名前を入力すると、ポリシーの一覧が自動的に縮小表示されます。
  
> [!IMPORTANT]
> あるポリシー レベルで適用されているポリシー設定が、他のポリシー レベルで適用されている設定によって無効になることがあります。ポリシーの優先順位は、ユーザー ポリシーが最も高く、サイト ポリシー、グローバル ポリシー (優先度が最も低い) と続きます。つまり、ポリシー設定が、そのポリシーの影響を受けるオブジェクトに近いほど、オブジェクトに及ぼす影響は大きくなります。 
  
ポリシーの作成と編集には、次の2つの選択肢があります。
  
1. [**新規**]: クリックして新しいポリシーを作成します。 ポリシーを適用するサイトを提供する必要があります。 次に、プッシュ通知の設定を構成します。 **プッシュ通知の構成**では、作成済みのサイトに対してのみポリシーを作成できます。
    
2. [**編集**]: ポリシーを選択し、[編集] をクリックして、ドロップダウンから操作を選択します。 編集できるのは、既に作成したサイトまたはグローバルポリシーを編集したサイトだけです。
    
   - [**詳細の表示]**: 現在選択されているポリシーに関する情報が表示されます。 既存のポリシーに変更を加えることができます。
    
   - **すべて選択**: 多数のポリシーがあり、すべてのポリシーを選択する必要がある場合は、[すべて選択] をクリックします。
    
   - [**削除**: 選択したポリシーを削除します。 **[すべて選択**して**削除**] を使用すると、すべてのポリシーが削除される
    
     > [!NOTE]
     > 既定の**グローバル**ポリシーは削除できません。 削除しようとすると、グローバルポリシーが既定値 (つまり、すべての設定がオフ) に戻されたことが通知されますが、ポリシーは削除できません。
  
新しいポリシーの作成または既存のポリシーの編集は、次の2つの操作に関連付けられます。
  
- **Commit**コミットアクションは、ポリシーを作成または更新し、変更内容を保存します。
    
- **キャンセル**キャンセルアクションは、最後のコミットアクション以降に行われたすべての変更を破棄します。 キャンセルすると、加えた変更はすべて失われます。
    
**プッシュ通知構成**では、2つの設定を行うことができます。 設定は、Microsoft および Apple のプッシュ通知サービスに関連付けられています。 いずれかのサービスのプッシュ通知を有効にするには、サービス名の横にあるチェックボックスをオンにします。 オフにするには、チェックボックスをオンにします。 選択が完了したら、コミットするかキャンセルします。 [コミット] をクリックすると、ポリシーの変更が保存されます。
