# <a name="deploy-edge-servers-in-skype-for-business-server"></a>ビジネス サーバーの Skype でエッジ トランスポート サーバーを展開します。

次のセクションには、後に[Skype ビジネス サーバー用にエッジ サーバーの展開を計画](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md)するビジネス サーバー ドキュメントの Skype が確認された後にするための手順が含まれています。 展開手順は次のとおりです。

## <a name="network-interfaces"></a>ネットワーク インターフェイス

計画で述べたように、かを構成するネットワーク インター フェースが DNS または DNS をホストしてエッジ サーバーを境界ネットワーク内境界ネットワーク内です。
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>境界ネットワークに DNS サーバーが存在する場合のインターフェイスの構成

1. 各エッジ サーバーは、内部に接続するインターフェイスでは、1 つは、外向きのインターフェイス用の 2 つのネットワーク アダプターをインストールします。
    
    > [!NOTE]
    > 内部サブネットと外部サブネット間のルーティングは許可しないでください。 
  
2. 外部インターフェイスに**1 つ**の次を構成します。
    
   a. 外部境界ネットワークのサブネットで 3 つの静的 IP アドレスを構成し、デフォルト ゲートウェイとして、外部ファイアウォールの内部インターフェイスを指すようにします。 アダプターの DNS 設定では、2 つの境界 DNS サーバーを指すように構成します。
    
   b. 外部境界ネットワークのサブネットで 1 つの静的 IP アドレスを構成し、デフォルト ゲートウェイとして、外部ファイアウォールの内部インターフェイスを指すようにします。 アダプターの DNS 設定は、2 つの境界 DNS サーバーを指すように構成します。 この構成は、以前は、 [Skype ビジネス サーバー用のエッジ ・ トポロジーを作成する](create-your-edge-topology.md)資料で説明するポートの割り当てを標準以外の値を持つトポロジを構成している場合に許容します。
    
3. インターフェイスでは、内部、内部境界ネットワークのサブネット上の 1 つの静的 IP を構成して、デフォルト ゲートウェイを設定しません。 アダプターの DNS 設定では、少なくとも 1 つの DNS サーバー (推奨は 2 つの境界 DNS サーバー) を指すように構成します。
    
4. Skype ビジネス サーバー、および Exchange ユニファイド メッセージング (UM) サーバー、クライアントが存在するすべての内部ネットワークへの内部インターフェイスでは、固定の静的ルートを作成します。
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>境界ネットワークに DNS サーバーがない場合のインターフェイスの構成

1. 各エッジ サーバーは、内部に接続するインターフェイスでは、1 つは、外向きのインターフェイス用の 2 つのネットワーク アダプターをインストールします。
    
    > [!NOTE]
    > 内部サブネットと外部サブネット間のルーティングは許可しないでください。 
  
2. 外部インターフェイスに**1 つ**の次を構成します。
    
   a. 外部の境界領域の 3 つの静的 IP アドレスは、サブネットをネットワークします。 必要もあります、外部インターフェイスのデフォルト ゲートウェイを構成するのには、デフォルト ゲートウェイとしてインターネットに接続するルーターまたは外部ファイアウォールを定義します。 理想的には、2 つの外部 DNS サーバーを外部 DNS サーバーを指すようにアダプターの DNS 設定を構成します。
    
   b. 外部の境界ネットワークのサブネットの静的 IP アドレスを 1 つ。 必要もあります、外部インターフェイスのデフォルト ゲートウェイを構成するのには、デフォルト ゲートウェイとしてインターネットに接続するルーターまたは外部ファイアウォールを定義します。 外部 DNS サーバー、または理想的には 2 つの外部 DNS サーバーを指すようにアダプターの DNS 設定を構成します。 この構成は、以前は、 [Skype ビジネス サーバー用のエッジ ・ トポロジーを作成する](create-your-edge-topology.md)資料で説明するポートの割り当てを標準以外の値を持つトポロジを構成している場合に許容します。
    
3. インターフェイスでは、内部、内部境界ネットワークのサブネット上の 1 つの静的 IP を構成して、デフォルト ゲートウェイを設定しません。 また、アダプターの DNS 設定は空白のままにします。
    
4. Skype ビジネス サーバー、および Exchange ユニファイド メッセージング (UM) サーバー、クライアントが存在するすべての内部ネットワークへの内部インターフェイスでは、固定の静的ルートを作成します。
    
5. 次ホップのサーバーのレコードを格納するためには、各エッジ サーバーまたは仮想 IP (VIP) 上のホスト ファイルを編集します。 このレコードは、ディレクター、Standard Edition サーバーまたはフロント エンド プール トポロジ ビルダーでエッジ サーバーの次ホップ アドレスとして構成するになります。 DNS 負荷分散を使用している場合は、次ホップ プールの各メンバーの行が含まれます。
