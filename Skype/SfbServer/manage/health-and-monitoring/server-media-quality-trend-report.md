# <a name="server-media-quality-trend-report-in-skype-for-business-server"></a>サーバー メディア品質傾向レポート (Skype for Business Server
 
**概要:** サーバー メディア品質傾向レポートの詳細については、「Skype for Business Server。
  
サーバー メディア品質傾向レポートでは、通話量、低通話率、パケット損失、ジッターなどの Quality of Experience メトリックで最大 5 つのサーバーをグラフィカルに比較できます。 これにより、パフォーマンスが低下しているサーバー、過小使用されているサーバー、または過剰使用されているサーバーの特定などを簡単に行えます。
  
## <a name="accessing-the-server-media-quality-trend-report"></a>サーバー メディア品質傾向レポートへのアクセス

サーバー メディア品質傾向レポートには、次のいずれかのレポートからアクセスできます。
  
- [[サーバー パフォーマンス レポート] Skype for Business Server](server-performance.md) ([傾向] メトリックをクリックして)
    
- [[詳細レポート] Skype for Business Server](call-detail-report.md) ([A/V エッジ サーバー] メトリックをクリックします)。 発信者または呼び出し先がサーバーの場合は、エンドポイント名をクリックしてサーバー品質メディア傾向レポートにアクセスすることもできます)。
    
## <a name="making-the-best-use-of-server-media-quality-trend-report"></a>サーバー メディア品質傾向レポートを最大限に活用するには

特定のサーバーの [サーバー パフォーマンス レポート] の [[](server-performance.md)傾向] Skype for Business Serverをクリックすると、サーバー メディア品質傾向レポートが開きます。 ただし、そのレポートの空白のインスタンスだけが表示されます。サーバー パフォーマンス レポートで選択したサーバーは、画面上に表示されません。 代わりに、[サーバー] ドロップダウンからそのサーバーを選択する必要があります。 [サーバー] ドロップダウンには [すべて選択] オプションが含まれています。 5 つ以上のサーバーがある場合、このオプションは機能しません。サーバー メディア品質傾向レポートでは、一度に最大 5 つのサーバーのデータしか表示できません。
  
サーバー メディア品質傾向レポートで表示されるグラフでは、[通話量] と [通話の低い通話率] というラベルの付いたポイントはホットリンクです。グラフ上のポイントをクリックすると、指定した期間の通話[の](call-list-report-0.md)合計 (または通話が不十分) を示す Skype for Business Server で通話リスト レポートのインスタンスが開きます。

## <a name="metrics"></a>指標

次の表に、サーバー メディア品質傾向レポートで提供される情報を示します。
  
**サーバー メディア品質傾向レポートの指標**

|**名前**|**このアイテムを並べ替えることはできますか?**|**説明**|
|:-----|:-----|:-----|
|[**通話ボリューム**] <br/> |不要  <br/> |通話の合計数。  <br/> |
|[**低下 (MOS)**] <br/> |不要  <br/> |通話中に発生した MOS (平均オプション スコア) の低下の平均量。 低下値の範囲は、0.0 から 5.0 の高値までです。0.5 以下の値は、許容可能な低下を表します。 従来、平均オピニオン値は、ユーザーが通話の品質を 1 から 5 の範囲で評価することによって計算されていました。 Skype for Business Server一連のアルゴリズムを使用して、ユーザーが通話を評価する方法を予測します。  <br/> この値が高い場合は、輻輳、帯域幅の不足、ワイヤレスの輻輳または干渉、メディア サーバーやエンドポイントの過負荷などの原因が考えられます。その結果、音声のひずみや欠落が生じます。  <br/> |
|[**低品質通話のパーセンテージ**] <br/> |不要  <br/> |低品質として分類される通話の総数。低品質通話とは、少なくとも 1 つの測定指標が許容値を超えている通話 (たとえば、過剰なジッターが発生した通話) のことです。  <br/> |
|[**往復 (ミリ秒)**] <br/> |不要  <br/> |リアルタイム転送プロトコル パケットが、あるエンドポイントとの間を往復するのに要する平均時間 (ミリ秒単位)。200 ミリ秒以下の往復時間が許容できる品質と見なされます。  <br/> この値が高い場合は、国際通話ルーティング、ルーティングの構成ミス、メディア サーバーの過負荷などの原因が考えられます。その結果、双方向のリアルタイムの音声会話が難しくなります。  <br/> |
|[**パケット損失**] <br/> |不要  <br/> |リアルタイム転送プロトコル (RTP) パケット損失の平均レート (パケット損失は、RTP パケット、つまりインターネット経由で音声とビデオを転送するために使われるプロトコルの一種で、パケットが宛先に到達できなかったときに発生します)。この値が高い場合は、輻輳、帯域幅の不足、ワイヤレスの輻輳または干渉、メディア サーバーの過負荷などの原因が考えられます。パケット損失が発生すると、通常、音声のひずみや欠落が生じます。  <br/> |
|[**ジッター (ミリ秒)**] <br/> |不要  <br/> |RTP パケットの着信間に検出された平均ジッター (ジッターとは、通話の "揺れ" の測定値です)。通常、この値が高い場合は、輻輳やメディア サーバーの過負荷の原因が考えられます。その結果、音声のひずみや欠落が生じます。  <br/> |
|[**ヒーラー隠し比率**] <br/> |不要  <br/> |サンプルの合計数に対する隠しオーディオ サンプルの平均比率 (隠しオーディオ サンプルとは、突然遷移を取り除くために使用される手法です。突然遷移は、通常、ネットワーク パケットの削除が原因で発生します)。この値が高い場合は、パケット損失やジッターのために高いレベルで損失の隠蔽が適用されています。その結果、音声のひずみや欠落が生じます。  <br/> |
|[**ヒーラー引き伸ばし比率**] <br/> |不要  <br/> |サンプルの合計数に対する引き伸ばされたオーディオ サンプルの平均比率 (引き伸ばされたオーディオとは、ネットワーク パケットの削除が検出されたときに通話の品質を維持できるように拡張されたオーディオのことです)。この値が高い場合は、ジッターのために高いレベルでサンプルの引き伸ばしが行われています。その結果、音声が機械のように聞こえたりひずんで聞こえたりします。  <br/> |
|[**ヒーラー圧縮比率**] <br/> |不要  <br/> |サンプルの合計数に対する圧縮オーディオ サンプルの平均比率 (圧縮オーディオとは、ネットワーク パケットの削除が検出されたときに通話の品質を維持できるように圧縮されたオーディオのことです)。この値が高い場合は、ジッターのために高いレベルでサンプルの圧縮が行われています。その結果、音声が早送りされたように聞こえたりひずんで聞こえたりします。  <br/> |
