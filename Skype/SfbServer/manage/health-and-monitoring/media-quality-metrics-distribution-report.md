# <a name="the-media-quality-metrics-distribution-report-in-skype-for-business-server"></a>[メディア品質指標] 配布レポート (Skype for Business Server 
 
**概要:** 詳細については、「メディア品質指標配布レポート」を参照Skype for Business Server。
  
メディア品質指標配布レポートを使用すると、ジッターやパケット損失などの Quality of Experience メトリックの分布値を示すグラフを表示できます。 たとえば、ユーザーが合計 10 件の電話を行ったとします。これらの 10 回の呼び出しでは、次のラウンドトリップ時間が報告されます。
  
|**電話番号**|**ラウンド トリップ時間 (ミリ秒)**|
|:-----|:-----|
|1  <br/> |50  <br/> |
|2  <br/> |50  <br/> |
|3  <br/> |50  <br/> |
|4  <br/> |50  <br/> |
|5  <br/> |50  <br/> |
|6   <br/> |50  <br/> |
|7   <br/> |50  <br/> |
|8   <br/> |4550  <br/> |
|9   <br/> |50  <br/> |
|10  <br/> |50  <br/> |
   
これらのラウンド トリップ時間の平均は 500 ミリ秒 (5000 を 10 で割った値) です。 500 ミリ秒は、非常に大きなラウンド トリップ時間です。その結果、ネットワークの輻輳に重大な問題が発生している可能性があります。 (通常、ラウンド トリップ時間が長い場合は、ネットワークの過負荷が発生します)。
  
実際には、もちろん、通話の 90% は優れた往復時間を持っています。単に、全体的な結果を歪め、1 つの悪い呼び出しがありました。 平均ラウンド トリップ時間のみを確認すると、非常に間違った結論に達する可能性があります。
  
Media Quality Metrics Distribution Report を使用すると、指定したメトリック (ラウンド トリップ時間など) のグラフィカルな分布を表示することで、間違った結論に飛び込むのを防ぐのに役立ちます。 これらのグラフは、9 つの非常に良い呼び出しと 1 つの非常に悪い呼び出しがあったことを明確にするのに役立ちます。 確かに、1 つの呼び出しをさらに調査する必要があります。ただし、10 回の呼び出しのうち 9 回が非常に良好だったという事実は、少なくとも現時点ではネットワークに大幅な変更を加える理由がないことを示唆しています。
