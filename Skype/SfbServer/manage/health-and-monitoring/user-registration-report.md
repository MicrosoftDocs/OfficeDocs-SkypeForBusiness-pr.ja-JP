# <a name="user-registration-report-in-skype-for-business-server"></a>ユーザー登録レポート (Skype for Business Server
 
**概要:**[ユーザー登録レポート] の詳細については、「Skype for Business Server。
  
ユーザー登録レポートには、ユーザー ログオン アクティビティの概要、特に、指定した期間中に Skype for Business Server にログオンしたユーザーの数 (時間単位、毎日、毎週、月単位) に関する情報が表示されます。 このレポートで示されるのは、ログオンしたユーザーの数のみです。 どのユーザーがログオンしたかは示されません。 監視レポートには、特定のユーザーが使用しているユーザー (および使用していないユーザー) に関Skype for Business Server情報は提供されない。 ただし、ユーザー アクティビティ レポートを使用してユーザー情報の概算は把握できます。
  
ユーザー登録レポートでは、ユーザー ログオンに関する情報を示す場合に 2 つの重要な分類が行われます。 まず、内部ログオンと外部ログオンという 2 つの大きなカテゴリにログオンが分類されます。 内部ログオンは、組織のファイアウォール内から (企業ネットワークに接続中に) ログオンしたユーザーを表します。 外部ログオンは、エッジ サーバーを介してファイアウォールの外部からログオンしたユーザーを表します (たとえば、インターネット カフェからログオンしたユーザーは、外部ログオンとしてカウントされます)。 ファイアウォールの外部からログオンしたユーザーの数が必要な場合、ユーザー登録レポートでその情報を確認できます。
  
また、ユーザー登録レポートは、指定した期間内に存在した "アクティブ" ユーザーの数も示します。 アクティブ ユーザーとは、インスタント メッセージング (IM) セッションに参加したユーザー、Skype for Business Server 会議に参加したユーザー、電話を受けたユーザー、その期間中に Skype for Business Server を使用したユーザーです。 ログオンしただけで実際にシステムを使用しなかったユーザーとは異なります。
  
## <a name="accessing-the-user-registration-report"></a>ユーザー登録レポートへのアクセス

ユーザー登録レポートには、監視レポートのホーム ページからアクセスできます。ユーザー登録レポートからは他のレポートへリンクしません。
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>ユーザー登録レポートの効果的な活用方法

一般的にSkype for Business Server質問の 1 つを展開した後:ユーザーが実際にこの新しいテクノロジを使用しているかどうかはどうすれば分かるのか。 いくつか制限はありますが、ユーザー登録レポートがこの疑問の解決に役立ちます。 ユーザーがユーザーを使用しているかどうかを判断するにはSkype for Business Server 2 つの操作を行う必要があります。 まず、ユーザー登録レポートで [一意のログオン ユーザー数] 指標の値を確認します。 この値は、ユーザーにログオンしている個人の数Skype for Business Server。
  
比較して、[ログオンの合計] メトリックは、ユーザーがユーザーにログオンした合計回数Skype for Business Server。 たとえば、Ken Myer が 1 日に 5 回Skype for Business Serverにログオンしたとします。 この場合、Ken Myer は、[ログオン合計数] 指標でログオン セッション回数 5 回として記録されるのに対し、[一意のログオン ユーザー数] 指標ではログオン ユーザー 1 人として記録されます。 また、1 人のユーザーが複数のデバイスまたは複数の場所からログオンすることも珍しくありません。 たとえば、ユーザーはデスクトップ コンピューター、ラップトップ コンピューターを使用してログオンし、ユーザーは自動的にコンピューターにログオンする IP 電話Skype for Business Server。 この場合、一意のユーザー数は 1 人、ログオン回数は 3 回になります。
  
ログオン合計数と一意のログオン ユーザー数の相違についてさらにわかりやすく説明にするために、ある期間内のログオンが次の表のようになっていた場合について検討します。
  
|**ユーザー**|**ログオン時刻**|
|:-----|:-----|
|Ken Myer  <br/> |2015/7/7 8:45 AM  <br/> |
|Ken Myer  <br/> |2015/7/7 8:46 AM  <br/> |
|Pilar Ackerman  <br/> |2015/7/7 9:17 AM  <br/> |
|Ken Myer  <br/> |2015/7/7 9:22 AM  <br/> |
|Pilar Ackerman  <br/> |2015/7/7 9:31 AM  <br/> |
   
この場合、ログオン合計数は 5 回ですが、ログオンしたユーザーは Ken Myer (ログオン回数 3 回) と Pilar Ackerman (ログオン回数 2 回) の 2 人だけです。ログオン合計数と一意のログオン ユーザー数はこのように異なります。
  
一意のログオンの数を知るだけでなく、ユーザーに対して有効になっているユーザーの総数をSkype for Business Server。 この値は、管理シェルを開き、Skype for Business Serverコマンドを実行することでWindows PowerShellできます。
  
```PowerShell
(Get-CsUser).Count
```

前のコマンドが 1,236 の値を返し、一意のログオン ユーザー メトリックが平均 667 を返す場合、Skype for Business で有効にしているユーザーの半分以上が実際にシステムに毎日ログオンしている (つまり、667 を 1,236 で割った値は約 54%) ことを示しています。
  
> [!CAUTION]
> ログオンの指標に記録されるのは、指定した期間内に実際にログオンしたユーザーの数である点に注意してください。 その前に既にシステムにログオンしていたユーザーは追跡されません。 たとえば、[一意のログオン ユーザー数] 指標が 667、ユーザー数が 1,236 の場合、約半分のユーザーがシステムにログオンしたことになります。 ただし、ログオン データのチェックを開始する前に既に 300 人のユーザーがシステムにログオンしていたとします。 つまり、実際には約 1,000 人のユーザーが Skype for Business Server にログオンしていたので、ユーザーの 80% 近くがログオンしていたことを意味します。 
  
また、[一意のログオン ユーザー数] の値と [一意のアクティブなユーザー数] 指標も比較する必要があります。 [一意のアクティブ ユーザー] メトリックは、Skype for Business Server で実際に使用した一意のユーザーの数を示します。電話をかかえたか、Skype for Business Server 会議に参加したか、IM セッションに参加しました。 この情報は、ユーザーがSkype for Business Serverを開始する度に自動的に開始するように構成Windows。 この理由により、毎日 Windows にログオンするときに Skype for Business に自動的にログオンするユーザーが多く、その期間中は実際には Skype for Business Server を使用しない場合があります。
  
また、[一意のアクティブ ユーザー] メトリックは、通常、ユーザーが一日の終わりにWindowsログオフしない組織内の、より意味のあるデータを提供します。 代わりに、コンピューターをロックし、コンピューターをWindowsしてSkype for Businessします。 このような状態では、ユーザーは何日もログオンしたままでログオフしないので、1 日あたりのログオン数は非常に低くなる可能性があります。 ただし、一意のアクティブ なユーザーは、ユーザーがアクティブにクライアントを使用Skype for BusinessクライアントSkype for Business Serverします。
  
## <a name="filters"></a>フィルター

フィルターは、細かく絞り込んだデータ セットを返したり、返されたデータをさまざま方法で表示したりする方法として利用できます。 たとえば、ユーザー登録レポートを使用すると、すべてのレジストラー プールとエッジ サーバーのデータを表示したり、個々のプールのデータを表示することができます。 また、データをグループ化する方法を選択することもできます。 この場合は、登録が、時間、日、週、または月を基準にグループ化されています。
  
次の表に、ユーザー登録レポートで使用できるフィルターを示します。
  
**ユーザー登録レポートのフィルター**

|**[名前]**|**説明**|
|:-----|:-----|
|**From** <br/> |時間範囲の開始日と開始時刻。データを時間単位で表示するには、次のように開始日と開始時刻の両方を入力します。  <br/> 2015 年 7 月 7 日 13:00  <br/> 開始時刻を入力しないと、レポートは自動的に指定日の午前 12:00 に開始します。データを日単位で表示するには、次のように日付のみを入力します。  <br/> 7/7/2015  <br/> 週単位または月単位で表示するには、表示する週または月の任意の日付を入力します (その週または月の最初の日である必要はありません)。  <br/> 7/3/2015  <br/> 週は、常に日曜日から土曜日までです。  <br/> |
|**To** <br/> |時間範囲の終了日と終了時刻。データを時間単位で表示するには、次のように終了日と終了時刻の両方を入力します。  <br/> 2015 年 7 月 7 日 13:00  <br/> 終了時刻を入力しないと、レポートは自動的に指定日の午前 12:00 に終了します。データを日単位で表示するには、次のように日付のみを入力します。  <br/> 7/7/2015  <br/> 週単位または月単位で表示するには、表示する週または月の任意の日付を入力します (その週または月の最初の日である必要はありません)。  <br/> 7/3/2015  <br/> 週は、常に日曜日から土曜日までです。  <br/> |
|**間隔** <br/> | 時間間隔です。次のいずれかを選択します。 <br/>  時間単位 (最大 25 時間の表示が可能) <br/>  日単位 (最大 31 日の表示が可能) <br/>  週単位 (最大 12 週の表示が可能) <br/>  月単位 (最大 12 か月の表示が可能) <br/>  開始日と終了日が、選択されている期間内で許容される値の最大数を超えると、(開始日から起算した) 値の最大数のみが表示されます。 たとえば、開始日が 7/7/2015 で終了日が 2015 年 2 月 28 日の日間隔を選択した場合、2015 年 8 月 7 日 12:00 AM ~ 9/7/2015 12:00 AM (合計 31 日分) のデータが表示されます。 <br/> |
|**Pool** <br/> |レジストラー プールまたはエッジ サーバーの完全修飾ドメイン名 (FQDN)。 個別のプールを選択するか、[**すべて**] をクリックしてすべてのプールのデータを表示できます。 このドロップダウン リストは、データベース内のレコードに基づいて自動的に設定されます。 <br/> |
