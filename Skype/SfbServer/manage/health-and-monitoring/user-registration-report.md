# <a name="user-registration-report-in-skype-for-business-server"></a>Skype for Business Server のユーザー登録レポート
 
**概要:** Skype for Business Server のユーザー登録レポートについて確認します。
  
ユーザー登録レポートには、ユーザー ログオン アクティビティの概要が示されます。特に、指定した期間 (時間単位、毎日、毎週、毎月) に Skype for Business Server にログオンしたユーザーの数に関する情報が表示されます。 このレポートで示されるのは、ログオンしたユーザーの数のみです。 どのユーザーがログオンしたかは示されません。 監視レポートでは、Skype for Business Server を使用している特定のユーザー (および使用していないユーザー) に関する情報は提供されません。 ただし、ユーザー アクティビティ レポートを使用してユーザー情報の概算は把握できます。
  
ユーザー登録レポートでは、ユーザー ログオンに関する情報を示す場合に 2 つの重要な分類が行われます。 まず、内部ログオンと外部ログオンという 2 つの大きなカテゴリにログオンが分類されます。 内部ログオンは、組織のファイアウォール内から (企業ネットワークに接続中に) ログオンしたユーザーを表します。 外部ログオンは、ファイアウォールの外側からエッジ サーバー経由でログオンしたユーザーを表します (たとえば、インターネット カフェからログオンしたユーザーは、外部ログオンとしてカウントされます)。 ファイアウォールの外部からログオンしたユーザーの数が必要な場合、ユーザー登録レポートでその情報を確認できます。
  
また、ユーザー登録レポートは、指定した期間内に存在した "アクティブ" ユーザーの数も示します。 アクティブ ユーザーとは、インスタント メッセージング (IM) セッションに参加したユーザー、Skype for Business Server 会議に参加したユーザー、通話を行ったまたは受信したユーザー、その期間中に Skype for Business Server を使用したユーザーです。 ログオンしただけで実際にシステムを使用しなかったユーザーとは異なります。
  
## <a name="accessing-the-user-registration-report"></a>ユーザー登録レポートへのアクセス

ユーザー登録レポートには、監視レポートのホーム ページからアクセスできます。ユーザー登録レポートからは他のレポートへリンクしません。
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>ユーザー登録レポートの効果的な活用方法

Skype for Business Server を展開した後、よく尋ねらわれる質問の 1 つは、ユーザーが実際にこの新しいテクノロジを使用しているかどうかの確認方法です。 いくつか制限はありますが、ユーザー登録レポートがこの疑問の解決に役立ちます。 ユーザーが Skype for Business Server を使用するかどうかを判断するには、2 つの操作を行う必要があります。 まず、ユーザー登録レポートで [一意のログオン ユーザー数] 指標の値を確認します。 この値は、Skype for Business Server にログオンした個別のユーザーの数を示します。
  
一方、[合計ログオン数] 指標は、ユーザーが Skype for Business Server にログオンした合計回数を示します。 たとえば、Ken Myer が 1 日に 5 回 Skype for Business Server にログオンしたとします。 この場合、Ken Myer は、[ログオン合計数] 指標でログオン セッション回数 5 回として記録されるのに対し、[一意のログオン ユーザー数] 指標ではログオン ユーザー 1 人として記録されます。 また、1 人のユーザーが複数のデバイスまたは複数の場所からログオンすることも珍しくありません。 たとえば、ユーザーは自分のデスクトップ コンピューターとノート PC を使用してログオンし、自動的に Skype for Business Server にログオンする IP 電話を使用できます。 この場合、一意のユーザー数は 1 人、ログオン回数は 3 回になります。
  
ログオン合計数と一意のログオン ユーザー数の相違についてさらにわかりやすく説明にするために、ある期間内のログオンが次の表のようになっていた場合について検討します。
  
|**ユーザー**|**ログオン時刻**|
|:-----|:-----|
|Ken Myer  <br/> |2015 年 7 月 7 日午前 8:45  <br/> |
|Ken Myer  <br/> |2015 年 7 月 7 日午前 8:46  <br/> |
|Pilar Ackerman  <br/> |2015 年 7 月 7 日午前 9:17  <br/> |
|Ken Myer  <br/> |2015 年 7 月 7 日午前 9:22  <br/> |
|Pilar Ackerman  <br/> |2015 年 7 月 7 日午前 9:31  <br/> |
   
この場合、ログオン合計数は 5 回ですが、ログオンしたユーザーは Ken Myer (ログオン回数 3 回) と Pilar Ackerman (ログオン回数 2 回) の 2 人だけです。ログオン合計数と一意のログオン ユーザー数はこのように異なります。
  
一意のログオン数を知るだけでなく、Skype for Business Server が有効になっているユーザーの総数を知る必要があります。 この値は、Skype for Business Server 管理シェルを開き、次のコマンドを実行Windows PowerShellできます。
  
```PowerShell
(Get-CsUser).Count
```

上記のコマンドの値が 1,236 を返し、[一意のログオン ユーザー数] 指標が平均値 667 を返す場合、Skype for Business を有効にするユーザーの半分以上が毎日システムに実際にログオンしています (つまり、667 を 1,236 で割った値は約 54%) です。
  
> [!CAUTION]
> ログオンの指標に記録されるのは、指定した期間内に実際にログオンしたユーザーの数である点に注意してください。 その前に既にシステムにログオンしていたユーザーは追跡されません。 たとえば、[一意のログオン ユーザー数] 指標が 667、ユーザー数が 1,236 の場合、約半分のユーザーがシステムにログオンしたことになります。 ただし、ログオン データのチェックを開始する前に既に 300 人のユーザーがシステムにログオンしていたとします。 つまり、実際には 1,000 人近くのユーザーが Skype for Business Server にログオンしていたので、ユーザーの 80% 近くがログオンしていたことを意味します。 
  
また、[一意のログオン ユーザー数] の値と [一意のアクティブなユーザー数] 指標も比較する必要があります。 [一意のアクティブユーザー数] 指標は、実際に Skype for Business Server を使用した一意のユーザーの数を示します。通話を行ったユーザー、Skype for Business Server 会議に参加したユーザー、または IM セッションに参加したユーザーです。 ユーザーが Windows を起動するごとに自動的に起動するように Skype for Business Server を構成できるので、これは便利な情報です。 このため、多くのユーザーが毎日 Windows にログオンするときに自動的に Skype for Business にログオンし、その期間中は実際には Skype for Business Server を使用しない場合があります。
  
また、一意のアクティブ ユーザーメトリックは、通常、ユーザーが 1 日の終わりに Windows からログオフしない組織で、より有意義なデータを提供します。 代わりに、コンピューターをロックし、Windows と Skype for Business を実行したままにします。 このような状態では、ユーザーは何日もログオンしたままでログオフしないので、1 日あたりのログオン数は非常に低くなる可能性があります。 ただし、一意のアクティブ なユーザーは、ユーザーが Skype for Business または別の Skype for Business Server クライアントをアクティブに使用するかどうかを示します。
  
## <a name="filters"></a>フィルター

フィルターは、細かく絞り込んだデータ セットを返したり、返されたデータをさまざま方法で表示したりする方法として利用できます。 たとえば、ユーザー登録レポートを使用すると、すべてのレジストラー プールとエッジ サーバーのデータを表示したり、個々のプールのデータを表示することができます。 また、データをグループ化する方法を選択することもできます。 この場合は、登録が、時間、日、週、または月を基準にグループ化されています。
  
次の表に、ユーザー登録レポートで使用できるフィルターを示します。
  
**ユーザー登録レポートのフィルター**

|**名前**|**説明**|
|:-----|:-----|
|**From** <br/> |時間範囲の開始日と開始時刻。データを時間単位で表示するには、次のように開始日と開始時刻の両方を入力します。  <br/> 2015 年 7 月 7 日午後 1 時  <br/> 開始時刻を入力しないと、レポートは自動的に指定日の午前 12:00 に開始します。データを日単位で表示するには、次のように日付のみを入力します。  <br/> 7/7/2015  <br/> 週単位または月単位で表示するには、表示する週または月の任意の日付を入力します (その週または月の最初の日である必要はありません)。  <br/> 7/3/2015  <br/> 週は、常に日曜日から土曜日までです。  <br/> |
|**To** <br/> |時間範囲の終了日と終了時刻。データを時間単位で表示するには、次のように終了日と終了時刻の両方を入力します。  <br/> 2015 年 7 月 7 日午後 1 時  <br/> 終了時刻を入力しないと、レポートは自動的に指定日の午前 12:00 に終了します。データを日単位で表示するには、次のように日付のみを入力します。  <br/> 7/7/2015  <br/> 週単位または月単位で表示するには、表示する週または月の任意の日付を入力します (その週または月の最初の日である必要はありません)。  <br/> 7/3/2015  <br/> 週は、常に日曜日から土曜日までです。  <br/> |
|**間隔** <br/> | 時間間隔です。次のいずれかを選択します。 <br/>  時間単位 (最大 25 時間の表示が可能) <br/>  日単位 (最大 31 日の表示が可能) <br/>  週単位 (最大 12 週の表示が可能) <br/>  月単位 (最大 12 か月の表示が可能) <br/>  開始日と終了日が、選択されている期間内で許容される値の最大数を超えると、(開始日から起算した) 値の最大数のみが表示されます。 たとえば、開始日が 2015 年 7 月 7 日で終了日が 2015 年 2 月 28 日の日間隔を選択した場合、2015 年 8 月 7 日午前 12:00 から 2015 年 9 月 7 日午前 12:00 (合計 31 日分のデータ) のデータが表示されます。 <br/> |
|**Pool** <br/> |レジストラー プールまたはエッジ サーバーの完全修飾ドメイン名 (FQDN)。 個別のプールを選択するか、[**すべて**] をクリックしてすべてのプールのデータを表示できます。 このドロップダウン リストは、データベース内のレコードに基づいて自動的に設定されます。 <br/> |
