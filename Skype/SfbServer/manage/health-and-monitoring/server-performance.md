# <a name="server-performance-report-in-skype-for-business-server"></a>Skype for Business Server のサーバー パフォーマンス レポート
 
**概要:** Skype for Business Server のサーバー パフォーマンス レポートについて学習します。
  
サーバー パフォーマンス レポートには、低品質通話の割合が最も高い Skype for Business Server サーバーの一覧が表示されます。 このレポートは、サーバーの種類別にサーバーを分割し、次の種類の個別の統計情報を報告します。
  
- 仲介サーバー
    
- 音声ビデオ会議サーバー
    
- 音声ビデオ エッジ サーバー
    
- ゲートウェイ (仲介サーバー)
    
- ゲートウェイ (仲介サーバー のバイパス)
    
- ビデオ (音声ビデオ会議サーバーおよび音声ビデオ エッジ サーバーのビデオメトリックを含む)
    
- アプリケーション共有 (音声ビデオ会議サーバーおよび音声ビデオ エッジ サーバーのアプリケーション共有指標を含む)
    
このレポートに表示されるランク付けは相対的なランク付けとして重要です。 たとえば、最もパフォーマンスの低いサーバーが 1,000 件の通話中に 1 件の低品質通話を行ったとします。 これは、許容される割合が .1% を超える値です。 ただし、それが最もパフォーマンスの低いサーバーである場合 (つまり、他のすべてのサーバーの通話のパーセンテージが .1% よりも低い場合)、そのサーバーはサーバー パフォーマンス レポートに表示されます。
  
## <a name="accessing-the-server-performance-report"></a>サーバー パフォーマンス レポートへのアクセス

サーバー パフォーマンス レポートには、[監視レポート] ホーム ページからアクセスします。 次のいずれかの指標をクリックすると [、Skype for Business Server](call-list-report-0.md) の通話リスト レポートにドリルダウンできます。
  
- [通話ボリューム]
    
- [低品質通話のパーセンテージ]
    
また、次の指標をクリックすると、サーバー メディア品質傾向レポートにドリルダウンできます。
  
- Trend
    
## <a name="making-the-best-use-of-the-server-performance-report"></a>サーバー パフォーマンス レポートを最適に利用する

サーバー パフォーマンス レポートには、データをフィルター処理するさまざまな方法があります。たとえば、ネットワークの種類 (有線接続からの呼び出しとワイヤレス接続からの通話) とアクセスの種類 (ファイアウォールの内側から行われた通話とファイアウォールの外側からの呼び出し) でフィルター処理できます。 サーバー パフォーマンス レポートを表示してこれらのフィルターを利用すると良い方法です。 たとえば、低品質な通話のパーセンテージが 3.24% の仲介サーバーがある場合です。 ワイヤレス通話のみを見ている場合、同じサーバーの通話のパーセンテージが 20% に近づいている可能性があります。 つまり、サーバーがワイヤレス通話に問題を抱え、サーバーに有線通話の問題が発生していないので部分的に見えなくなっていた問題です。
  
## <a name="filters"></a>フィルター

フィルターは、細かく絞り込んだデータ セットを返したり、返されたデータをさまざま方法で表示したりする方法として利用できます。 たとえば、サーバー パフォーマンス レポートを使用すると、返されたデータをサーバーの種類やネットワークの種類 (有線またはワイヤレス) でフィルター処理することができます。 データをグループ化する方法も選択できます。 この場合、データは時間、日、週、または月でグループ化されます。
