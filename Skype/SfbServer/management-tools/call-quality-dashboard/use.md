# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>Skype for Business Server の通話品質ダッシュボードを使用する

**概要:** 通話品質ダッシュボードの使用方法について説明します。 通話品質ダッシュボードは、Skype for Business Server のツールです。

通話品質ダッシュボード (CQD) を使用すると、IT 担当者は集計データを使って、ユーザーのグループの統計情報を比較して、傾向とパターンを特定することができます。 CQD は、個々の通話の問題を解決するためのものではありませんが、多くのユーザーに適用される問題と解決策を特定するためのものです。

## <a name="call-quality-dashboard-user-guide"></a>通話品質ダッシュボード ユーザー ガイド

CQD は、Quality of Experience (QoE) データに基づいてレポートをすばやく作成し、整理するための web ポータルです。 CQD は、SSAS キューブを展開して、QoE Metrics データベースのデータを集計し、管理者がリアルタイムでレポートの作成や変更を行うことができるようにします。 Excel を使用してキューブに直接接続することもできますが、このポータルは、QoE データを扱ういくつかのワークフロー向けに最適化されています。 次のデータが含まれます。

- すばやくアクセスできるようにキャッシュされたレポートデータ
- 情報の共有と公開のためのレポートページへのディープリンク
- 合理化されたレポートの編集と作成、およびレポートの説明の編集可能なメタデータ。

また、CQD は、カスタムダッシュボードで使用するために、ユーザーがキューブデータにアクセスできるようにする web Api を公開します。

### <a name="feature-overview"></a>機能の概要

通話品質ダッシュボードにアクセスすると、次の画面が表示されます。

![CQD の使用](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)

1. "サマリーウィンドウ" には、"レポートセット" (右) のコンテキストが表示されます。
2. 概要 PaneReport で [編集] をクリックして、レベルプロパティ (Y 軸の高さを含む) を設定します。
3. 階層リンクは、レポートセット階層内の現在の位置を特定するのに役立ちます。
4. サブレポートを含むレポートは、青色のリンクで示されています。 下位のレポートにドリルダウンするには、リンクをクリックします。

棒グラフの上にマウスを移動して、詳細な値を表示します。 フォーカスがあるレポートには、[編集]、[複製]、[削除]、[ダウンロード] というアクションメニューが表示されます。

### <a name="default-reports"></a>既定のレポート

通話品質ダッシュボードポータルに初めてアクセスすると、既定のレポートセットが自動的に作成されます。 これらのレポートは、「システム レポート」とも呼ばれます。 これらのレポートは、新しい兄弟レポートや子レポートを作成することにより、自由に変更または削除することができます。

最上位レベルでは、"オーディオストリームの月間トレンド" レポートには、すべてのオーディオストリームの月間トレンドが表示されます。 棒グラフの棒の上にマウスを移動すると、横棒グラフで表されるデータの詳細が表示されます。 [オーディオストリームの月間傾向] レポートのタイトルをクリックすると、管理された通話と非管理対象の呼び出しの間でレポートが分割された "マネージとアンマネージオーディオストリーム" レポートに移動します。 管理された通話とは、有線接続経由で企業のファイアウォール内から発信された通話のことです。 アンマネージ通話には、会社のファイアウォールの外部から発信された通話、および wi-fi 経由のすべての通話が含まれます。

もう1つのトップレベルレポートは、"ユーザーから報告された通話品質評価のヒストグラム" と呼ばれます。 通話品質評価は、通話の品質を示すために、通話の終了時に Skype for Business ユーザーから提供された電話番号です。 評価番号の範囲は 1 ~ 5 です。1は最も悪い and 5 として最適です。 ヒストグラムには、1か月の評価が表示された音声通話の数が表示されます。

レポートのタイトルをクリックすると、データにより多くのフィルターが追加されたレポートに移動します。 システムレポートでは、各子レポートには、親レポートで利用可能なデータのサブセットが表示されます。 問題解決モデルはシンプルです。どのサブレポートに問題が提案されているかを調査し、問題領域を段階的に絞り込むことができます。 サブレポートを作成する機能により、特定のデータの傾向の原因について独自の推測を調査することができます。

### <a name="create-and-edit-reports"></a>レポートを作成して編集する

レポートのアクションメニューで [編集] をクリックすると、レポートエディターが表示されます。 それぞれのレポートは、キューブに対するクエリに基づきます。 レポートは、クエリから返されたデータを視覚化したものです。 レポートエディターを使用すると、これらのクエリやレポートの表示オプションを編集できます。 レポートエディターを開くと、次のように表示されます。

![CQD の使用](../../media/e8969625-e6f9-4d67-873f-93e78dd12b35.png)

1. 左側のウィンドウでは、ディメンション、メジャー、およびフィルターを選択します。 既存の値のいずれかにマウスポインターを移動すると、その値を削除できる "x" ボタンが表示されます。 見出しの横にあるプラス記号 (+) をクリックすると、新しい寸法、測定、またはフィルターを追加できるダイアログが開きます。
2. 最上部には、グラフをカスタマイズするためのオプションが表示されます。
3. レポート エディターには、レポートのプレビュー機能があります。
4. 詳細なレポートの説明は、下部にあるエディットボックスを使って作成できます。

### <a name="sparklines-in-tables"></a>表のスパークライン

StartDate.Month をディメンションとして追加し、データを表形式の傾向として表示すると、表のセル内に棒グラフとスパークラインを表示できます。 各月の値を表示するには、マウスポインターを横棒グラフの上に移動し、スパークラインに移動します。

![CQD の使用](../../media/fe6b18d7-b8cf-472a-9c93-0f7703f5a700.png)

棒グラフとスパークラインが表示されるためには、レポートエディターの上部にある [スパークラインの表示] チェックボックスをオンにする必要があります。 これにより、[傾向] オプションが選択され、月が最後のサイズになるように移動します。これは、月をクリックして、上矢印と下矢印を使用して、前月または翌月に移動することもできます。

### <a name="settings"></a>設定

[設定] メニューには、[システム正常性] と [バージョン情報] などの便利なページへのリンクが表示され、ダッシュボードの右上隅にあります。

![CQD の使用](../../media/0e9ae123-e231-4fea-94e1-5788e8f3e1d3.png)

[ユーザー ダッシュボード リンク] を選択すると、CQD の他のユーザーや、そのレポートを表示できるページが表示されます。 レポートセットを共有するには、URL バーのリンクをコピーし、別の CQD ユーザーと共有します。 このリンクは、他のユーザーのユーザー名の下にある [ユーザーダッシュボードリンク] ページに表示されるリンクと同じです。

## <a name="cqd-streams"></a>CQD ストリーム

CQD ストリームは、良好、低品質、または未分類と見なされます。 CQM 1.5 は次の CQD 定義を使用するようになりました。

- 低品質ストリームは、しきい値を超えた低品質の通話メトリックの任意の組み合わせです。
- 通話中の1つのストリームの品質が悪い場合、通話の両方のストリームのフラグが低品質になります。 会議では、参加者はそれぞれ固有の通話としてカウントされ、他の参加者と個別に報告されます。
- 未分類のストリームは、品質指標 (つまり、合成トランザクションまたは短い通話) のないストリームです。
- 有効なストリーム = 非モバイル クライアント
- 分類子は変更できません

**低品質通話の定義/分類子**

|指標|しきい値|
|:-----|:-----|
|DegradationAvg |1.0 (-1 ネットワーク MOS) より大きい |
|RoundTrip |500 より大きい  |
|PacketLossRate |0.1 より大きい (10%) |
|JitterInterArrival |30 より大きい  |
|RatioConcealedSamplesAvg |0.07 より大きい |
|||

JPDR 定義 = 低品質通話の定義 - RatioConcealedSamplesAvg 

## <a name="where-is-callercallee"></a>呼び出し元/呼び出し先の場所

CQD は、"First" と "Second" を使います。代わりに、呼び出し元と呼び出し先の間にステップが介在するため、"First" と "Second" を使います。

 **最初**サーバーがストリームに関与している場合は、常にサーバーのエンドポイント (たとえば、AV MCU または仲介サーバー) になります。

 **2 番目** サーバー間のストリームでない限り、常にクライアントのエンドポイントになります。

**1 番目と 2 番目の分類の例**

|エンドポイント 1 UAType |エンドポイント 2 UUAType |1 番目|2 番目|
|:-----|:-----|:-----|:-----|
|2 (AVMCU)  |4 (Skype for Business)  |エンドポイント 1 |エンドポイント 2 |
|2 (AVMCU)  |1 (mMediationServer)  |エンドポイント 2 |エンドポイント 1 |
|4 (Skype for Business) |4 (Skype for Business)  |MediaLine での呼び出し元  |MMediaLine での呼び出し先  |
|||||

両方のエンドポイントが同じ型である場合は、CQD が最初に呼び出し元エントリを、呼び出し先が2番目になります。 エンドポイント名の詳細については、[このブログ](https://blogs.technet.com/b/jenstr/archive/2015/05/22/call-quality-dashboard-tips-and-tricks.aspx)を参照してください。

## <a name="accounting-for-vpn"></a>VPN のアカウンティング

VPN ソリューションで VPN フラグが正しく設定されていることがわかっている場合は、すべて設定されています。 それ以外の場合は、次のいずれかの方法を使用します。

- VPN というネットワークの種類を作成 (推奨) して、VPN サブネットをこの新しいネットワークの種類の VPN に関連付けます。
- VPN という建物を作成して、その建物に VPN サブネットを関連付けます。

## <a name="query-fundamentals"></a>クエリーの基本事項

正しい形式のクエリーには、次の 3 つのパラメーターがすべて含まれます。

- 測定
- ディメンション
- フィルター

正しい形式のクエリーの例は、「建物 6 [フィルター] のサブネット [ディメンション] ごとに不良ストリーム [測定] を表示してください。」のようになります

## <a name="what-does-union-do"></a>UNION で行えること

Union を使用すると、AND 演算子を使用して条件をフィルター処理することができます。 複数のフィルター条件を組み合わせて、OR 操作と同様の結果を得ることができます。

例: 建物からすべてのストリームを取得するには、UNION はマージされたデータセットの個別のビューを提供します。 UNION を使用するには、統合 (UNION) したい 2 つのフィルターについて、UNION フィールドに一般的なテキストを挿入します。
