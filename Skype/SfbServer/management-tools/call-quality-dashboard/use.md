# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>Skype for Business Server の通話品質ダッシュボードを使用する
 
**概要:** 通話品質ダッシュボードの使用方法について説明します。 通話品質ダッシュボードは、Skype for Business Server のツールです。
  
CQD により、IT 担当者は集計データを使用して、メディア品質の問題が発生している環境におけるフォーカス領域を識別できるようになります。これにより、IT 担当者は異なるユーザーのグループの統計情報を比較して、傾向とパターンを識別できます。個別の通話に関する問題の解決にフォーカスしていませんが、特定の環境内の多くのユーザーに該当する問題と解決策の識別にフォーカスしています。
  
## <a name="call-quality-dashboard-user-guide"></a>通話品質ダッシュボード ユーザー ガイド

通話品質ダッシュボード (CQD) は、QoE (Quality of Experience) データに基づいてすばやくレポートをすばやく作成および整理するための Web ポータルです。CQD は、SSAS キューブを展開して、データを QoE 指標データベースに集約します。これにより、ユーザーは、レポートの作成と変更、およびリアルタイムの調査を行うことができます。Excel を使用してキューブに直接接続することもできますが、このポータルは、QoE データを扱ういくつかのワークフロー向けに最適化されています。このようなデータには、高速アクセスのためのレポート データのキャッシュ、情報の共有と公開のためのレポート ページへのディープ リンク、合理化されたレポートの編集と作成、レポートの説明の編集可能なメタデータが含まれます。さらに、CQD の Web API が公開されているため、ユーザーは、カスタム ダッシュボードで使用するキューブ データにプログラムでアクセスできます。 
  
### <a name="feature-overview"></a>機能の概要

ユーザーが通話品質ダッシュボードにアクセスすると、次のような画面が表示されます。
  
![CQD の使用](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)
  
1. "サマリーウィンドウ" には、"レポートセット" (右) のコンテキストが表示されます。 
    
2. レポートセットレベルのプロパティ (Y 軸の高さを含む) を設定するには、[概要] ウィンドウで [編集] をクリックします。
    
3. 階層リンクにより、レポート セット階層内の現在の位置を識別できます。 
    
4. サブレポートを含むレポートには、青いリンクが示されます。リンクをクリックすると、子レポートにドリル ダウンします。 
    
マウス カーソルを棒グラフや傾向線の上に移動すると、詳細な値が表示されます。 フォーカスがあるレポートには、[編集]、[複製]、[削除]、[ダウンロード] というアクションメニューが表示されます。 
  
### <a name="default-reports"></a>既定のレポート

ユーザーが通話品質ダッシュボード ポータルに初めてアクセスすると、既定のレポートのセットが自動的に作成されます。これらのレポートは、「システム レポート」とも呼ばれます。ユーザーは、これらのレポートを自由に変更したり削除したりできます。通常は、これらのレポートを拡張して新しい兄弟レポートや子レポートを作成します。 
  
最上位レベルでは、"オーディオストリームの月間トレンド" レポートには、すべてのオーディオストリームの月間トレンドが表示されます。 横棒グラフの棒の上にマウスを移動すると、横棒グラフで表されるデータの詳細が表示されます。 オーディオストリームの月次トレンドレポートのタイトルをクリックすると、[管理された管理と非管理のオーディオストリーム] レポートに移動します。ここでは、管理と非管理の呼び出しの間でレポートが分割されます。 管理された通話とは、有線接続経由で企業のファイアウォール内から発信された通話のことです。 非管理された通話には、会社のファイアウォールの外側から発信された通話、および wi-fi 経由のすべての通話が含まれます。
  
その他のトップレベルレポートは、"ユーザーから報告された通話品質評価のヒストグラム" と呼ばれます。 通話品質評価は、通話の品質を示すために、通話の終了時に Skype for Business ユーザーから提供された電話番号です。 評価の番号の範囲は 1 ~ 5 です。1は最も悪い値で、5が最も良いものになります。 ヒストグラムには、1か月の評価が表示された音声通話の数が表示されます。 
  
通常、レポートのタイトルをクリックすると、データに対して追加のフィルターが適用されたレポートが表示されます。システム レポートの場合、それぞれの子レポートには、親レポートで使用できるデータのサブセットが表示されます。これにより、概念的にシンプルな問題解決のためのモデルが提供されます。つまり、問題を含むデータまたは傾向が示されるサブレポートを見極めることによって、問題空間を絞り込みます。新しいサブレポートを作成すると、特定のデータの傾向の起源に関して、独自の仮説を調査できます。
  
### <a name="creating-and-editing-reports"></a>レポートの作成および編集

レポートのアクションメニューで [編集] をクリックすると、レポートエディターが表示されます。 それぞれのレポートは、キューブに対するクエリに基づきます。 レポートは、クエリから返されたデータを視覚化したものです。 レポート エディターは、これらのクエリやレポートの表示オプションを編集するためのユーザー インターフェイスです。 レポート エディターを開くと、次のような画面が表示されます。
  
![CQD の使用](../../media/e8969625-e6f9-4d67-873f-93e78dd12b35.png)
  
1. 左側のウィンドウでは、ディメンション、メジャー、およびフィルターを選択します。 既存の値のいずれかにマウスポインターを合わせると、値を削除できる "x" ボタンが表示されます。 見出しの横にあるプラス記号 (+) をクリックすると、新しい寸法、測定、またはフィルターを追加するためのダイアログが表示されます。 
    
2. 最上部には、グラフをカスタマイズするためのオプションが表示されます。
    
3. レポート エディターには、レポートのプレビュー機能があります。 
    
4. 下部の編集ボックスを使用して、詳細なレポートの説明を作成できます。 
    
### <a name="sparklines-in-tables"></a>表のスパークライン

StartDate.Month をディメンションとして追加し、データを表形式の傾向として表示すると、表のセル内に棒グラフとスパークラインを表示できます。棒グラフまたはスパークライン上にマウス カーソルを移動すると、各月の値が表示されます。 
  
![CQD の使用](../../media/fe6b18d7-b8cf-472a-9c93-0f7703f5a700.png)
  
棒グラフとスパークラインが表示されるためには、レポートエディターの上部にある [スパークラインの表示] チェックボックスをオンにする必要があります。 [近似曲線] オプションを選択し、月を下方に移動して最後の寸法にします。これは、[月] をクリックし、上矢印と下矢印を使用して稼働日を上または下に移動することもできます。 
  
### <a name="settings"></a>設定

ダッシュボードの右上隅にある設定メニューには、[システム正常性]、[バージョン情報] などの便利なページへのリンクが用意されています。
  
![CQD の使用](../../media/0e9ae123-e231-4fea-94e1-5788e8f3e1d3.png)
  
説明とタイムスタンプを個別のユーザーが表示するかどうかにかかわらず、これらの設定は個々のユーザーにのみ影響します。これらの設定は、レポートセットの変更ではなく、他のユーザーに表示されるものではありません。 キャッシュをクリアすると、すべてのクエリによってキューブからデータが再読み込みされますが、既定値を復元すると、ユーザーが作成または変更したレポートのすべてが削除され、システムレポートセット (ユーザーが初めてログインしたときに表示される内容) が削除されます。
  
[ユーザー ダッシュボード リンク] を選択すると、CQD の他のユーザーや、そのレポートを表示できるページが表示されます。 レポート セットを共有する場合は、URL バーのリンクをコピーしてそれを他の CQD ユーザーと共有します。 このリンクは、他のユーザーのユーザー名の下にある [ユーザーダッシュボード] リンクページに表示されるものと同じです。

## <a name="cqd-streams"></a>CQD ストリーム

CQD ストリームは良好、不良、または未分類となります。CQM 1.5 は次の CQD 定義を使用するようになりました。 
  
- 不良ストリームは、しきい値を超えている低品質通話の指標の組み合わせになります。
    
- 通話中の1つのストリームの品質が悪い場合、通話の両方のストリームのフラグが低品質になります。 会議では、参加者はそれぞれ固有の通話としてカウントされ、他の参加者と個別に報告されます。
    
- 未分類のストリームは、品質の指標がないストリーム (代理トランザクション、短い通話など) です。
    
- 有効なストリーム = 非モバイル クライアント
    
- 分類子は変更できません
    
**低品質通話の定義/分類子**

|**指標**|**しきい値**|
|:-----|:-----|
|DDegradationAvg  <br/> |1.0 (-1 ネットワーク MOS) より大きい  <br/> |
|RoundTrip  <br/> |500 より大きい   <br/> |
|PacketLossRate  <br/> |0.1 (10%) より大きい   <br/> |
|JitterInterArrival  <br/> |30 より大きい   <br/> |
|RRatioConcealedSamplesAvg  <br/> |0.07 より大きい   <br/> |
   
JPDR 定義 = 低品質通話の定義 - RatioConcealedSamplesAvg  
  
## <a name="where-is-callercallee"></a>呼び出し元/呼び出し先の場所

CQD では、発信者番号と呼び出し先のフィールドは使用されません。 これらの名前は、"First" と "Second" の名前が変更されました。これは、呼び出し元と呼び出し先の間に手順が介在しているためです。
  
 **1 番目** サーバーがストリーム内に関与している場合は常にサーバーのエンドポイントになります (AV MCU、仲介サーバー)。
  
 **2 番目** サーバー間のストリームでない限り、常にクライアントのエンドポイントになります。
  
**1 番目と 2 番目の分類の例**

|**エンドポイント 1 UAType **|**エンドポイント 2 UUAType **|**1 番目**|**2 番目**|
|:-----|:-----|:-----|:-----|
|2 (AVMCU)   <br/> |4 (Skype for Business)   <br/> |エンドポイント 1  <br/> |エンドポイント 2  <br/> |
|2 (AVMCU)   <br/> |1 (mMediationServer)   <br/> |エンドポイント 2  <br/> |エンドポイント 1  <br/> |
|4 (Skype for Business)  <br/> |4 (Skype for Business)   <br/> |MediaLine での呼び出し元   <br/> |MMediaLine での呼び出し先  <br/> |
   
両方のエンドポイントが同じ種類の場合、CQD は呼び出し元のエントリを 1 番目にして、呼び出し先を 2 番目にします。 詳細については、[このブログ](https://blogs.technet.com/b/jenstr/archive/2015/05/22/call-quality-dashboard-tips-and-tricks.aspx)を参照してください。
  
## <a name="accounting-for-vpn"></a>VPN のアカウンティング

VPN ソリューションで VPN フラグが正しく設定されていることがわかっている場合は、すべて設定されています。 そうでない場合は、以下の方法のいずれかを実行します。
  
- VPN というネットワークの種類を作成 (推奨) して、VPN サブネットをこの新しいネットワークの種類の VPN に関連付けます。
    
- VPN という建物を作成して、その建物に VPN サブネットを関連付けます。 
    
## <a name="query-fundamentals"></a>クエリーの基本事項

正しい形式のクエリーには、次の 3 つのパラメーターがすべて含まれます。 
  
- 測定
    
- ディメンション
    
- フィルター
    
正しい形式のクエリーの例は、「建物 6 [フィルター] のサブネット [ディメンション] ごとに不良ストリーム [測定] を表示してください。」のようになります
  
## <a name="what-does-union-do"></a>UNION で行えること

Union は、AND 演算子を使って条件をフィルターすることができます。OR like での結果を得るために複数のフィルター条件を組み合わせる必要があるシナリオが考えられます。
  
例: 建物からのすべてのストリームを取得する必要がある場合に、UNION は結合したデータセットを別個に表示できます。UNION を使用するには、統合 (UNION) したい 2 つのフィルターについて、UNION フィールドに一般的なテキストを挿入します。

## <a name="operational-processes"></a>運用プロセス

管理されたストリームのレビューおよび修復によって開始します。この領域の品質は管理下で 100% となるべきものであるため、修復するのが最も簡単になります。 
  
### <a name="managed-streams"></a>管理されたストリーム 

次の順に管理されたストリームをレビューおよび修復します。 
  
1. サーバー間  
    
2.   サーバーと有線 (内部)
    
3. 有線と有線 (内部)  
    
### <a name="unmanaged-streams"></a>管理されていないストリーム

次の順に管理されていないストリームをレビューおよび修復します。
