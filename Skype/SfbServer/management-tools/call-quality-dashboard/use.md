# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>Skype for Business Server の通話品質ダッシュボードの使用

**概要:** 通話品質ダッシュボードを使用する方法について学習します。 通話品質ダッシュボードは、Skype for Business Server 用のツールです。

通話品質ダッシュボード (CQD) を使用すると、IT プロは集約データを使用して、ユーザーグループの統計を比較して傾向やパターンを特定することで、メディア品質の問題を作成する問題を特定できます。 CQD は、個々の通話の問題の解決ではなく、多くのユーザーに適用される問題とソリューションの特定に重点を置いて行います。

## <a name="call-quality-dashboard-user-guide"></a>通話品質ダッシュボードのユーザー ガイド

CQD は、QoE (Quality of Experience) データに基づいてレポートをすばやく作成および整理するための Web ポータルです。 CQD は SSAS キューブを展開して QoE Metrics データベース内のデータを集計し、管理者はレポートを作成および変更したり、リアルタイムで調査を行います。 Excel を使用してキューブに直接接続することもできますが、ポータルは QoE データを含む複数のワークフロー用に最適化されています。 データには次の情報が含まれます。

- 高速アクセスのためのキャッシュされたレポート データ
- 情報の共有と公開のためのレポート ページへのディープ リンク
- レポートの編集と作成、およびレポートの説明の編集可能なメタデータの合理化。

また、CQD は、カスタム ダッシュボードで使用するキューブ データへのプログラムによるアクセスをユーザーに提供する Web API を公開します。

### <a name="feature-overview"></a>機能の概要

通話品質ダッシュボードにアクセスすると、次の画面が表示されます。

![CQD を使用する](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)

1. "Summary Pane" は、"レポート セット" (右側) のコンテキストが見つかる場所です。
2. Summary PaneReport の [編集] をクリックして、レベル プロパティ (Y 軸の高さを含む) を設定します。
3. Breadcrumb を使用すると、レポート セット階層内の現在の場所を特定できます。
4. サブレポートを含むレポートは青いリンクで表示されます。 リンクをクリックして、子レポートにドリルダウンします。

棒グラフと傾向線の上にマウスを移動すると、詳細な値が表示されます。 フォーカスがあるレポートには、[編集] 、"Clone"、"Delete"、"Download" というアクション メニューが表示されます。

### <a name="default-reports"></a>既定のレポート

通話品質ダッシュボード ポータルに初めてアクセスすると、既定の一連のレポートが自動的に作成されます。 これらのレポートは、システム レポートとも呼ばれます。 新しい兄弟レポートと子レポートを作成することで、これらのレポートを自由に変更または削除したり、レポートを拡張することができます。

トップ レベルでは、"Audio Streams Monthly Trend" レポートに、すべてのオーディオ ストリームの月次傾向が表示されます。 棒グラフのバーの上にマウスを移動すると、棒グラフで表されるデータの詳細なビューが表示されます。 [オーディオ ストリームの月次傾向] レポートのタイトルをクリックして、[管理されたオーディオ ストリームと管理されていないオーディオ ストリーム] レポートに移動し、レポートは管理通話とアンマネージ通話の間で分割されます。 管理された呼び出しは、有線接続を使用して企業ファイアウォール内から行われた呼び出しです。 管理されていない呼び出しには、企業ファイアウォールの外部から行われた呼び出しと、Wi-Fi を使用して行われたすべての呼び出しが含まれます。

もう 1 つのトップ レベル レポートは、「ユーザーが報告した通話品質評価ヒストグラム」と呼ばれる。 通話品質評価は、通話の品質を示す通話の最後に Skype for Business ユーザーが指定した番号です。 評価番号の範囲は 1 ~ 5 で、1 は最悪で、5 が最適です。 ヒストグラムは、1 か月で指定された評価を受け取ったオーディオ通話の数を示します。

レポートのタイトルをクリックして、データに対するフィルターが追加されたレポートに移動します。 システム レポートでは、各子レポートには、親レポートで使用できるデータのサブセットが表示されます。 問題解決モデルは簡単です。問題が限定されているデータまたは傾向をどのサブレポートで調査し、徐々に問題領域を絞り込むかです。 サブレポートを作成する機能を使用すると、特定のデータ傾向の原因に関する独自の推測を調査できます。

![CQD を使用する](../../media/e8969625-e6f9-4d67-873f-93e78dd12b35.png)

1. ディメンション、メジャー、およびフィルターは、左側のウィンドウで選択されます。 既存の値の 1 つをポイントすると、値を削除できる "x" ボタンが表示されます。 見出しの横にある [プラス] ボタンをクリックしてダイアログを開き、新しいディメンション、メジャー、またはフィルターを追加できます。
2. グラフのカスタマイズのオプションが上部に表示されます。
3. レポートのプレビューは、レポート エディターで使用できます。
4. レポートの詳細な説明は、下部の編集ボックスで作成できます。

## <a name="cqd-streams"></a>CQD ストリーム

CQD ストリームは、良好、貧弱、または未分類と見なされます。 CQM 1.5 では、次の CQD 定義が使用されます。

- ストリームが低いとは、しきい値を超える低い通話指標の任意の組み合わせです。
- 呼び出しで 1 つのストリームが貧弱な場合、呼び出しの両方のストリームにフラグが設定されません。 会議では、各参加者は一意の呼び出しとしてカウントされ、他のすべての参加者とは独立して報告されます。
- 未分類のストリームは、品質指標のないストリームです (つまり、代理トランザクションまたは短い呼び出し)。
- 有効なストリーム = モバイル以外のクライアント
- 分類子を変更できません

**低い呼び出しの定義/分類子**

|測定基準|しきい値|
|:-----|:-----|
|DegradationAvg |1.0 より大きい (-1 ネットワーク MOS) |
|RoundTrip |500 を超える |
|PacketLossRate |0.1 より大きい (10%) |
|ジッターInterArrival |30 より大きい |
|RatioConcealedSamplesAvg |0.07 より大きい |
|||

JPDR 定義 = 低い呼び出し定義から RatioConcealedSamplesAvg を引いた値

## <a name="where-is-callercallee"></a>発信者/呼び出し先の場所

CQD は発信者/呼び出し先フィールドを使用しません。その代わりに、発信者と呼び出し先の間に介入する手順が存在するために、"First" と "Second" が使用されます。

 **最初に** サーバーがストリームに関与している場合は、常にサーバー エンドポイント (AV MCU や仲介サーバーなど) になります。

 **Second** クライアント エンドポイントは常にクライアント エンドポイントになります 。このエンドポイントがクライアント ストリームServer-Serverです。

**1 番目と 2 番目の分類の例**

|エンドポイント 1 UAType|エンドポイント 2 UUAType|第 1|秒|
|:-----|:-----|:-----|:-----|
|2 (AVMCU) |4 (Skype for Business) |エンドポイント 1 |エンドポイント 2 |
|2 (AVMCU) |1 (mMediationServer) |エンドポイント 2 |エンドポイント 1 |
|4 (Skype for Business) |4 (Skype for Business) |MediaLine の発信者 |MMediaLine の呼び出し先 |
|||||

両方のエンドポイントが同じ種類の場合、CQD は [発信者] エントリを [最初] および [第 2 呼び出し先] に設定します。 エンドポイント名の詳細については、このブログ [を参照してください](/archive/blogs/jenstr/call-quality-dashboard-tips-and-tricks)。

## <a name="accounting-for-vpn"></a>VPN の会計

VPN ソリューションが VPN フラグを正確に設定することが分かっている場合は、すべて設定されます。 それ以外の場合は、次のいずれかの方法を使用します。

- VPN (優先) というネットワークの種類を作成し、VPN サブネットをこの新しい VPN NetworkType に関連付ける。
- VPN という名前の建物を作成し、VPN サブネットをこの建物に関連付ける。

## <a name="query-fundamentals"></a>クエリの基本

整形式のクエリには、次の 3 つのパラメーターすべてが含まれます。

- 測定
- Dimension
- Filter

整形式のクエリの例として、「Building 6 [Filter]のサブネット [Dimension] による貧弱なストリーム [測定] を表示する」があります。

## <a name="what-does-union-do"></a>UNION は何をしますか?

Union を使用すると、AND 演算子を使用して条件をフィルター処理できます。 複数のフィルター条件を組み合わせて OR 操作のような結果を得るシナリオがあります。

例: 建物からすべてのストリームを取得するために、UNION は結合されたデータセットの個別のビューを提供します。 UNION を使用するには、UNION にしたい 2 つのフィルター条件の UNION フィールドに共通のテキストを挿入します。

## <a name="operational-processes"></a>運用プロセス

最初に管理ストリームを確認して修復します。 この領域の品質は、コントロール内で 100% にし、修正が最も簡単である必要があります。

### <a name="managed-streams"></a>管理ストリーム

管理ストリームを次の順序で確認し、修復します。

1. Server-Server
2. Server-Wired-Inside
3. Wired-Wired-Inside

### <a name="unmanaged-streams"></a>アンマネージ ストリーム

管理されていないストリームを次の順序で確認し、修復します。

1. Server-Wifi-Inside
2. Server-Wired-Outside
3. Server-Wifi-Outside
4. Wired-Outside-Direct
5. ワイヤード(有線)外部リレー
6. その他の管理されていない
