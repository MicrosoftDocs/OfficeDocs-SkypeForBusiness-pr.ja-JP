# <a name="design-the-sip-trunk-for-e9-1-1-in-skype-for-business-server"></a>~ 9-1-1 では、Skype のビジネス サーバーの SIP トランクを設計します。
 
~ 9-1-1 展開のビジネス サーバーのエンタープライズ VoIP の Skype で、SIP トランキング プロバイダーを使用する、SIP トランキングのトポロジを計画します。
  
ビジネス サーバーの Skype では、~ 9-1-1 のサービス プロバイダーへの緊急の呼び出しを接続するのに SIP トランクを使用します。 E9-1-1 用の緊急サービス SIP トランクは、1 つの中央サイト、複数の中央サイト、または各ブランチ サイトにセットアップできます。 ただし、WAN リンクに呼び出し元のサイトとサイトの間でホストする場合は、緊急サービスの SIP トランクは、接続されていないサイトのユーザーによって配置された呼び出しへの呼び出しをルーティングするユーザーの音声ポリシー内の特別な電話使用法レコード必要があります、ローカルの公衆交換電話網 (PSTN) ゲートウェイを介して ECRC。 通話受付管理で同時通話数制限が有効な場合も同様です。
  
Skype ビジネス サーバー環境での SIP トランクを実装するために 2 つの方法があります。
  
- SIP トランク プロバイダーと通信するために、外部に面したパブリックにルーティング インターフェイスを使用するマルチホームの仲介サーバーを使用します。
    
- 設置セッション ボーダー コント ローラー (SBC) を使用すると、仲介サーバーと SIP トランクの間でセキュリティで保護された境界点のサービス プロバイダーのサービスを提供します。
    
後者の方法を選択した場合は、選択する SBC の型とモデルが認定済みであり、SIP INVITE の一部としてプレゼンス情報データ フォーマット位置オブジェクト (PIDF-LO) 場所データの通過がサポートされていることを確認します。 サポートされてない場合、緊急サービスのサービス プロバイダーに到着した通話から場所情報が除去されます。 SBCs の認定についての詳細は、 [「インフラストラクチャ修飾 Microsoft Lync 用をいます」](https://go.microsoft.com/fwlink/p/?LinkId=248425) 、 [「テレフォニー インフラストラクチャ Skype のビジネス」](https://docs.microsoft.com/SkypeForBusiness/certification/infra-gateways)を参照してください。 
  
E9-1-1 サービス プロバイダーは、冗長性を確保するために SBC のペアへのアクセスを提供します。 仲介サーバーのトポロジに関するいくつかの決定を行い、ルーティングの構成を呼び出す必要があります。 たとえば、2 つの SBC を同等のピアとして扱い、SBC 間の通話にラウンドロビン ルーティングを使用するのか、または一方の SBC をプライマリ、もう一方をセカンダリとして指定するのかを決定します。
  
ビジネス サーバーの Skype での SIP トランクの展開に関する詳細については、 [SIP の Skype ビジネス サーバー用のトランク](sip-trunking.md)を参照してください。 E9-1-1 用 SIP トランクの展開方法を決定する際に次の質問が役立ちます。
  
 **SIP トランクの展開に、専用接続を使用するか、それとも共有のインターネット接続を使用するか。**
  
> 緊急電話は常につながることが重要です。専用線で接続すれば、ネットワーク上の他のトラフィックによって回線が専有されることがなくなり、サービス品質 (QoS) も実装できます。パブリック インターネット経由で緊急サービスのサービス プロバイダーに接続し、緊急電話の機密性を保証する必要がある場合は、IPsec 暗号化が必要です。 
    
 **~ 9-1-1 展開向けの耐障害性でしょうか。**
  
> これは緊急時向けのソリューションなので、回復性が重要です。 プライマリおよびセカンダリ仲介サーバーと SIP トランク災害トレラントの場所を展開します。 をサポートしているユーザーに最も近いプライマリ仲介サーバーを展開することをお勧めし、(別の地理的な場所にある) セカンダリ仲介サーバー経由のルートのフェイル オーバーを呼び出します。 
    
 **ブランチ オフィスごとに個別に SIP トランクを展開するかどうか。**
  
> Skype ビジネス サーバーなど、ブランチ オフィスでの音声の復元機能を処理するためいくつかの方法を提供する: 弾力性のあるデータ ・ ネットワークを持つ、各支店に SIP トランクを展開するまたはを押し出すことの呼び出しをローカルのゲートウェイの停止中にします。 詳細については、 [SIP の Skype ビジネス サーバー用のトランク](sip-trunking.md)を参照してください。
    
 **通話受付管理 (CAC) を有効にするかどうか。**
  
> ビジネス サーバー用の Skype は緊急通報をハンドルできませんいずれかの通常の呼び出しとは異なる。 そのため、帯域幅管理や通話受付管理 (CAC) が E9-1-1 の構成に悪影響を与える場合があります。 CAC が有効になっており、緊急電話がルーティングされているリンク上に構成された制限数を超過した場合は、緊急電話がブロックされたり、ローカル PSTN ゲートウェイにルーティングされたりします。 前述のように、このような通話には場所データが含まれていないので、ECRC に手動でルーティングする必要があります。
