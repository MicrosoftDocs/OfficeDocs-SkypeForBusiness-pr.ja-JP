# <a name="plan-for-call-park-in-skype-for-business"></a>プラン for Call Park in Skype for Business
 
通話を保留にし、Skype for Business Server エンタープライズ VoIPを部署に転送できる、通話パークの計画。 容量計画、サポートされている通話、サポートされているクライアントが含まれます。
  
コール パーク アプリケーションを使用すると、エンタープライズ VoIPユーザーは次の操作を実行できます。
  
- 通話を保留にしてから、同じ電話または別の電話から通話を取得します。
    
- 通話を保留にし、部署または一般領域 (たとえば、営業部門や共通エリア電話がある倉庫) に転送します。
    
- 通話を保留にし、元の応答電話を他の通話に対して無料で保持します。
    
ユーザーが通話をパークすると、Skype for Business Server呼び出しはオービットと呼ばれる一時的な番号に転送され、呼び出しは取得されるか、または時間が切れるまで保持されます。Skype for Business Serverをパークしたユーザーにオービットを送信します。 パークされた通話を取得するには、ユーザーがオービット番号にダイヤルするか、[会話] ウィンドウのオービット リンクまたはボタンをクリックします。 
  
呼び出しをパークしたユーザーは、インスタント メッセージング (IM) やページング システムなどの外部メカニズムを使用して、オービット番号を他のユーザーに伝達することで、通話を取得するユーザーに通知できます。 通話をパークしたユーザーは、[会話] ウィンドウを開いたままにし、呼び出しが取得されると通知を受け取ります。
  
オービット範囲はグローバルに一意なので、ルーティングが適切に構成されている場合は、Skype for Business Serverまたは PBX 電話から通話を取得できます。 構成可能な時間内に呼び出しを取得するユーザーがいない場合、その通話は、その通話をパークしたユーザーに呼び戻されます。 そのユーザーがリングバックに応答しない場合、呼び出しは、構成されている場合は、オペレーターなどのフォールバック先に転送されます。 通話が 1 回から 10 回に転送される前に呼び出しが戻る回数を構成できます。 転送された通話に誰も応答していない場合、通話は切断されます。 呼び出しが取得または切断されると、オービットは解放されます。
  
コール パークを展開する場合は、駐車場通話の内線番号の範囲を予約する必要があります。 これらの拡張機能は仮想拡張機能である必要があります。ユーザーまたは電話が割り当てられていない拡張機能です。 次に、内線番号の範囲を使用してコール パーク オービット テーブルを構成し、各範囲を処理するコール パーク アプリケーションをホストするアプリケーション サービスを指定します。 各フロントエンド プールには、プールにパークされている呼び出しを管理するために使用される、対応するバック エンド サーバー上のコール パーク テーブルがあります。 オービット範囲の一覧は、サーバーの全体管理ストアに格納され、オービットを移動先プールにルーティングするために使用されます。 Call Park アプリケーションSkype for Business Server展開および構成されている各プールには、1 つ以上のオービット範囲を設定できます。 オービット範囲は、展開全体でグローバルに一意Skype for Business Server必要があります。 
  
また、通話が時間外にリダイレクトされる場所や、通話中に電話の人が音楽を聞くかどうかなど、他の通話パーク設定も構成します。 通話が保留されている間に再生する音楽ファイルを指定できます。
  
> [!NOTE]
> コール パーク用のカスタマイズされた保留音ファイルは、Skype for Business Server 障害復旧プロセスの一部としてバックアップされません。プールにアップロードされたファイルが破損、破損、または消去されると失われます。 コール パーク用にアップロードしたカスタマイズされた保留音ファイルのバックアップ コピーは、常に個別に保持してください。 
  
コール パーク アプリケーションは、このアプリケーションのエンタープライズ VoIP。 このアプリケーションを展開エンタープライズ VoIP、コール パーク アプリケーションが自動的にインストールされ、アクティブ化されます。 ただし、通話パークを使用する前に、管理者エンタープライズ VoIPを構成し、音声ポリシーを使用してユーザーに対して有効にする必要があります。
  
## <a name="deployment-and-requirements"></a>展開と要件

コール パーク アプリケーションは、展開時に自動的にインストールエンタープライズ VoIP。 通話パークを有効にするには、音声ポリシーを構成します。
  
### <a name="software-requirements"></a>ソフトウェア要件

コール パークが展開されているすべてのフロントエンド サーバーと Standard Edition サーバーには、Windows Server 2008 R2 を実行しているサーバー用に Windows メディア形式ランタイムがインストールされている必要があります。または Windows Server 2012 を実行しているサーバーの場合は Microsoft Media Foundation がインストールされている必要があります。Windows Server 2012 R2. Server 2008 R2 Windows、メディア形式ランタイムWindowsデスクトップ エクスペリエンスの一部としてインストールWindowsされます。 Windowsパークが保留音で再生するメディア オーディオ (.wma) ファイルWindowsメディア形式ランタイムまたは Microsoft Media Foundation が必要です。
  
### <a name="port-requirements"></a>ポートの要件

コール パーク アプリケーションは、 **SIP リッスン要求にポート 5075**  を使用します。
    
> [!NOTE]
> このポートは既定の設定であり、**Set-CsApplicationServer** コマンドレットを使用して変更できます。 このコマンドレットの詳細については、「Lync Server 管理シェル」のドキュメントを参照してください。
  
### <a name="audio-file-requirements"></a>オーディオ ファイルの要件

コール パーク アプリケーションは、保留音Windowsメディア オーディオ (.wma) ファイルのみをサポートします。 保留音のファイルをカスタマイズするには、Microsoft Expression Encoder 4 を使用できます。 Expression Encoder 4 をダウンロードするには、「   [Expression Encoder 4」を参照してください](https://go.microsoft.com/fwlink/p/?linkId=202843)。 このツールを使用すると、ファイルを WMA 形式に変換できます。 コール パークの保留音ファイルとしての推奨形式は Media Audio 9、44 kHz、16 ビット、モノラル、CBR、32 kbps です。
  
> [!NOTE]
> 変換されたファイルは、44 kHz で録音された場合でも、電話では 16 kHz でのみ再生されます。 
  
## <a name="supported-clients-and-calls"></a>サポートされているクライアントと呼び出し

呼び出しパークでは、次のクライアントと呼び出しの種類がサポートされています
  
### <a name="clients-supported-for-parking-calls"></a>通話の保留をサポートするクライアント

IP、構内交換機 (PBX)、公衆交換電話網 (PSTN)、または携帯電話からの通話を保留できます。
  
> [!NOTE]
> 音声通話だけを保留できます。 インスタント メッセージと電話会議は保留できません。 
  
次のクライアントは、コール パークを使用して通話をパークできます。
  
- Skype for Business
    
- Lync 2013
    
- Lync 2010
    
- Lync 2010 Attendant
    
- Lync Phone Edition
    
> [!NOTE]
> 携帯電話はコール パークを使用して通話をパークできません。 
  
### <a name="clients-supported-for-retrieving-calls"></a>通話の取得をサポートするクライアント

オービット範囲は仮想の内線番号 (ユーザーまたは電話が割り当てられていない内線番号) のブロックとして構成されます。 オービットを仮想の内線番号として構成すると、携帯電話と PSTN 電話は保留された通話を取得できません。
  
フェデレーション ユーザーは保留された通話を取得できません。
  
次のクライアントは、通話パークにパークされている呼び出しを取得できます。
  
- Skype for Business
    
- Lync 2013
    
- Lync 2010
    
- Lync 2010 Attendant
    
- Lync Phone Edition
    
- IP 共通領域電話
    
- 共通領域電話やプライベート ブランチ 交換 (PBX) 電話を含む、Skype for Business Serverインフラストラクチャに接続されている非 IP 電話
