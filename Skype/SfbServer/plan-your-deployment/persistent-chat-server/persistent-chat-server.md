# <a name="plan-for-persistent-chat-server-in-skype-for-business-server-2015"></a>2015 年の常設チャット サーバー Skype for Business Serverする
 
**概要:** このトピックでは、2015 年に常設チャット サーバーを計画するSkype for Business Serverしてください。
  
常設チャット サーバーは、組織内の複数のユーザーがチャット ルームの会話に参加し、時間の長い間保持できるオプションの役割です。 ユーザーはチャット セッション中にリアルタイムで通信することができますが、テキスト、リンク、ファイルなどの各セッションのコンテンツは永続的であり、ユーザーはセッションのすべてのコンテンツをいつでも表示および検索できます。
  
常設チャット サーバーは、次の方法で組織内の通信を改善できます。
  
- 組織全体で情報の認識と参加を広げる
    
- 効率的な情報共有の有効化 
    
- 地理的に分散したチームや機能を持つチームを含む、チーム間のコミュニケーションを改善する
    
- 情報の過負荷を軽減する
    
- オプションで常設チャット コンプライアンス サービスを展開してコンプライアンス規制に従う

> [!NOTE] 
> 常設チャットは 2015 Skype for Business Serverで使用できますが、2019 年Skype for Business Serverではサポートされていません。 同じ機能は、Teams。 詳細については、「アップグレードの開始[方法」をMicrosoft Teamsしてください](/microsoftteams/upgrade-start-here)。 常設チャットを使用する必要がある場合は、この機能を必要とするユーザーを Teams に移行するか、2015 年Skype for Business Serverします。 
    
## <a name="persistent-chat-server-high-level-architecture"></a>常設チャット サーバーのハイレベル アーキテクチャ

次の図は、常設チャット サーバーアーキテクチャのハイレベル ビューを示しています。 
  
![常設チャット サーバー High-Levelアーキテクチャ。](../../media/0344f6e2-0c6d-4391-b4b3-ec31062b1576.png)
  
常設チャットは、常設チャット サービスを提供するフロントエンド サーバーの役割と、データベース コンポーネントのSQL構成されます。 フロントエンド コンポーネントとフロントエンド コンポーネントの両方が、専用の常設チャット プールに含まれています。 常設チャット サーバーをホストする各コンピューターは、2015 年 2015 年の既存Skype for Business Serverアクセスできる必要があります。 この図では、1 つの常設チャット サーバー プール (A) があります。これは、メッセージをルーティングするためにプール A Skype for Business Serverに依存します。
  
最大 80K の同時ユーザーをサポートする最大 4 つのアクティブな常設チャット サーバーを持つ 1 つ以上の常設チャット サーバー プールを展開できます。
  
Skype for Business Server 2015 は、セッション開始プロトコル (SIP) を使用して登録を行い、チャット用の拡張チャット通信 OVER SIP プロトコル (XCCOS) を使用して常設チャット サービスと通信します。 
  
## <a name="persistent-chat-services"></a>常設チャット サービス

次の図は、常設チャット サーバーのフロントエンド サービスと、これらのサービスがエンド エンド データベース コンポーネントと通信する方法を示しています。 フロントエンド コンポーネントには、常設チャット サービスとコンプライアンス サービスが含まれます。 バック エンド コンポーネントには、常設チャット ストアと常設チャット コンプライアンス ストアが含まれます。
  
![常設チャット サーバー High-Level サービス。](../../media/bcdbadbe-e868-4a46-8a73-36562648fdf7.png)
  
### <a name="chat-service"></a>チャット サービス

チャネル サービスとも呼ばれるチャット サービスは、常設チャット サーバーを担当するコア サービスです。 チャット サービスには、次の機能があります。
  
- 受信メッセージを受け入れます。
    
- 常設チャット ルーム内のオンライン参加者を登録および一覧表示する
    
- メッセージを他のチャネル サブスクライバーに再送信します。
    
- チャネル管理、チャット ルームへの招待、検索、および新しいコンテンツ通知のロジックを実装する
    
常設チャット サービスは、常設チャット ストアを使用して、チャット ルームのコンテンツや他のシステム メタデータ (承認ルールなど) を格納してアクセスします。 サービスは、常設チャット ファイル ストア内のチャット ルームにアップロードされたファイルを格納します。
  
### <a name="compliance-service"></a>コンプライアンス サービス

組織に常設チャット アクティビティをアーカイブする必要がある規制がある場合は、オプションの常設チャット コンプライアンス サービスを展開できます。 コンプライアンス サービスは、常設チャット コンプライアンス ファイル ストアにチャット コンテンツとイベント (会議室への参加や退出など) をアーカイブする責任があります。 コンプライアンス サービスは、常設チャット プール内の各常設チャット サーバーにインストールされます。 
  
### <a name="web-services"></a>Web サービス

常設チャット Web サービスは、フロントエンド Skype for Businessで実行されます。 Web サービスは、次インターネット インフォメーション サービス (IIS) に依存し、Web コンポーネントとして実装されます。
  
- ファイルのアップロードとダウンロードのための常設チャット Web サービスは、チャット ルームからのファイルの投稿と取得を担当します。
    
- チャット ルーム管理用の常設チャット Web サービスは、ユーザーがチャット ルームを管理し、新しいチャット ルームを作成する機能をユーザーに提供する責任を負います。
    
## <a name="defining-requirements-for-your-organization"></a>組織の要件の定義

常設チャット サーバーを展開する場合は、組織のビジネス要件を決定し、ビジネス ニーズをサポートするためにトポロジ、インフラストラクチャ、および技術的要件を定義する必要があります。 展開を最適化するには、次の質問に答える必要があります。
  
- 以前のバージョンのグループ チャット サーバーまたは以前のバージョンの常設チャット サーバーから移行するか、常設チャット サーバーを初めて展開しますか。
    
- Who常設チャット サーバーを使用できますか? 常設チャット ポリシーを指定して、グローバル、サイト、またはユーザー レベルでユーザー アクセスを決定します。
    
- 常設チャット サーバーへのアクセスが必要なユーザーの数。 常設チャット サーバーは、150,000 人のプロビジョニングされたユーザー (ポリシーで有効) をサポートし、最大 80,000 人の同時ユーザーをサポートします。 1 つの常設チャット サーバーで 20,000 人の接続ユーザーをサポートし、1 つの常設チャット サーバー プールに最大 4 台のアクティブ サーバーを含め、合計 80,000 人の同時接続ユーザーをサポートできます。
    
- スコープ、論理的境界、およびアクセスをどのように制御しますか。 これらの境界を分離するカテゴリを定義し、これらの各カテゴリで作成された会議室に入るユーザーを選択できます。
    
- ルームを作成できるユーザーをどのように制御しますか。 ルームを作成できる作成者を定義できます。 クリエイターは、他のメンバーをチャット ルーム管理者として割り当て、ルームを継続的に管理できます。
    
- ルームをどのように作成しますか。 常設チャット サーバーは、会議室を作成および管理するための Web ベースの機能を提供します。 これは、クライアントから起動Skype for Businessできます。 ビジネス要件とワークフローを実装する顧客ソリューションを定義し、ユーザーにカスタム ソリューションを提供する常設チャット サーバーを構成することができます。
    
- どのような種類のアドインをプロビジョニングしますか。 アドインは、Skype for Business クライアントの機能拡張ウィンドウを活用して、ルームに関連するコンテキストを提供することで、ルーム内のエクスペリエンスを強化します。 最も役立つと考えられる汎用的なアドイン (自社の Web サイト、社内のコラボレーション ドキュメントなど) を選択できます。 チャット ルーム マネージャーは、必要に応じて登録済みのアドインのいずれかを選択し、ルームに関連付けることができます。 
    
- どのような種類の高可用性要件と障害復旧要件がありますか。 常設チャット サーバーは、SQL ServerのミラーリングとSQL Serverをサポートします。 障害復旧の場合、常設チャット サーバーは、ログ配布が可能なストレッチ プール内の最大 8 つのサーバー (アクティブとスタンバイ 4) SQL Serverサポートします。 
    
- 規制要件はありますか? データを国内に保持する必要がある国または地域に会社がある場合は、複数の常設チャット サーバー プールを展開する必要があります。各ローカルから特定の地域に展開する必要があります。 ルーム、カテゴリ、アドインはプールにまたがるのではなく、1 つの常設チャット サーバー プールにのみ属します。 
    
    > [!NOTE]
    > 複数の常設チャット サーバー プールを使用しても、規模は大きくなされません (すべての常設チャット サーバー プールで同時に使用できるユーザー数は 80,000 人のみです)。 複数の常設チャット サーバー プールをサポートする主な理由は、規制上の懸念をサポートする場合です。
