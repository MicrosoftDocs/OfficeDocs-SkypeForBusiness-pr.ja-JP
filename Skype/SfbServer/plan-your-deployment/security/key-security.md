# <a name="key-security-features-in-skype-for-business-server"></a>Skype ビジネス サーバー用のキーのセキュリティ機能
 
Skype ビジネス サーバー用には、サーバーからサーバーへの認証、ロール ベースのアクセス制御、および構成データの一元的なストレージを含む、いくつかのセキュリティ機能が含まれています。 
  
この資料では、Skype のビジネス サーバーのセキュリティの高レベルな概要を提供します。 
  
## <a name="key-security-features-in-skype-for-business-server"></a>Skype ビジネス サーバー用のキーのセキュリティ機能

セキュリティは非常に広い範囲に及ぶトピックです。 セキュリティは、ビジネスのサーバーとデータベース、サービス、および、Skype ビジネス サーバーのエコシステムを構成するハードウェアの Skype のすべての機能に到達します。 この資料で説明 Skype の機能のいくつかのビジネス サーバーの具体的にはセキュリティが用意されています。
  
### <a name="planning-and-design-tools"></a>計画ツールと設計ツール

ビジネス サーバーの Skype では、プランニングと設計を容易にして、Skype をビジネスのサーバー コンポーネントを誤って構成する可能性を減らすために 2 つのツールを提供します。 
  
- **トポロジ計画ツール**は、トポロジの設計プロセスの大部分を自動化します。 計画ツールからトポロジ ビルダーでは、Skype をビジネスのサーバーを実行する各サーバーのインストールに必要なツールは、結果をエクスポートできます。
    
- **トポロジ ビルダー**は、すべての構成情報を中央管理ストアに格納します。
    
詳細については、これらのツールは、[ビジネスのサーバー管理ツールの Skype](../../management-tools/management-tools.md)を参照してください。
  
### <a name="central-management-store"></a>中央管理ストア

ビジネス サーバーの Skype では、サーバーおよびサービスに関する構成データは、中央管理ストアの部分です。 中央管理ストアでは、堅牢でスキーマを定義、設定、維持、管理、説明、およびビジネスのサーバーの展開に、Skype の動作に必要なデータのストレージを提供します。 また、データを検証して構成の一貫性を保ちます。 この構成データに対するすべての変更は、中央管理ストアは、「out-of-sync」問題を排除することで発生します。 
  
データの読み取り専用コピーは、エッジ サーバーや存続可能ブランチ アプライアンスなど、トポロジ内のすべてのサーバーにレプリケートされます。レプリケーションは、既定では、ネットワーク サービスのコンテキストで実行されるサービスが管理し、権限とアクセス許可が抑えられてコンピューター上の一般ユーザーと同じになります。 
  
### <a name="server-to-server-authentication"></a>サーバー間認証

ビジネス サーバーの Skype では、オープン認証 (OAuth) プロトコルを使用してサーバー間で認証を構成できます。 たとえば、2016 の Microsoft Exchange Server を実行しているサーバーで認証するためにサーバーをビジネス用の Skype を構成できます。 OAuth プロトコルを使用すると、Business Server と Microsoft Exchange Server の Skype が互いに信頼関係です。 これには、シームレスな方法で製品を統合する機能が用意されています。 詳細については、[管理サーバーからサーバーへの認証 (OAuth) とビジネス サーバーの Skype のパートナーのアプリケーション](../../manage/authentication/server-to-server-and-partner-applications.md)を参照してください。
  
### <a name="windows-powershell-based-management-and-web-based-management-interface"></a>Windows PowerShell ベースの管理と Web ベースの管理インターフェイス

Skype ビジネス サーバーは、Windows PowerShell のコマンド ライン インターフェイス上に構築された、強力な管理インタ フェースを提供します。 このインターフェイスにはセキュリティを管理するためのコマンドレットが含まれているうえ、Windows PowerShell のセキュリティ機能が既定で有効になっており、ユーザーが簡単にまたは知らないうちにスクリプトを実行することができないようになっています。 つまり、セキュリティを最大限に高め、攻撃の手段を減らすように、ソフトウェアの既定値が自動的に設定されています。 ビジネス サーバーの Skype での Windows PowerShell の管理のサポートについての詳細は、[ビジネスのサーバー管理シェルの Skype](../../manage/management-shell.md)を参照してください。 
  
### <a name="role-based-access-control-rbac"></a>役割ベースのアクセス制御 (RBAC)

Skype ビジネス サーバーの役割に基づくアクセス制御 (RBAC) で高水準のセキュリティを維持しながら管理タスクを委任できるようにすることを提供します。 そのため、業務上必要な管理権限のみをユーザーに与える "最小限の特権" の原則に従って RBAC を使用することができます。 Skype ビジネス サーバーは、新しいロールを作成する機能、および既存のロールを変更することも提供します。 
  
## <a name="network-address-translation-nat"></a>ネットワーク アドレス変換 (NAT)

ビジネス サーバーの Skype は、エッジ サーバーの内部インターフェイスでネットワーク アドレス変換 (NAT) の使用をサポートしていませんが、アクセス エッジ サービス、Web 会議エッジ サービス、および A の外部インターフェイスを配置すると音声ビデオ エッジ サービスルーターまたは 1 つと拡張の両方のネットワーク アドレス変換 (NAT) を実行するファイアウォールの背後にあるエッジ サーバー トポロジに統合します。 ハードウェア ロード バランサーの背後にある複数のエッジ サーバーでは、NAT を使用できません。 複数のエッジ サーバーの外部インターフェイスで NAT を使用する場合は、ドメイン ネーム システム (DNS) の負荷分散が必要です。 DNS 負荷分散を使用すると、エッジ サーバー プール内のエッジ サーバーあたりのパブリック IP アドレスの数を削減できます。 詳細については、 [Skype ビジネス サーバー用にエッジ サーバーのシナリオ](../../plan-your-deployment/edge-server-deployments/scenarios.md)を参照してください。
  
> [!NOTE]
> Microsoft Office Communications Server 2007 が展開されているエンタープライズとフェデレーションを行うときに、使用しているエンタープライズとフェデレーション先のエンタープライズとの間で音声ビデオを使用する必要がある場合、ポート要件は、展開されているエッジ サーバーのうち、古い方のバージョン用の要件になります。 などのポート範囲に必要なこれらの古いバージョンは、フェデレーション パートナーは、ビジネス サーバーの Skype に、エッジ トランスポート サーバーをアップグレードするまで、両方の企業を開く必要があります。 アップグレード後には、ポート要件を見直して、新しい構成に応じて削減できます。

## <a name="trustworthy-computing-security-development-lifecycle-sdl"></a>信頼できるコンピューティングのセキュリティ開発ライフサイクル (SDL)

Skype ビジネス サーバーのように設計された、 [Microsoft 信頼できるコンピューティングのセキュリティ開発ライフ サイクル](https://go.microsoft.com/fwlink/p/?linkid=68761)(SDL) に準拠して開発されました。
  
- **デザインで信頼できます。** 安全な統合コミュニケーション システムを作成する最初のステップは、脅威モデルを設計し、設計されていることと、各機能をテストしました。 Microsoft では加えて、設計動作以外でのテストも実施して、予期しない製品動作から生じるセキュリティ脆弱性を探しています。 コーディング プロセスとコーディング手法には、セキュリティに関連する複数の改善点が組み込まれていました。 ビルド時に使われるツールは、製品にコードが最終的にチェックインされる前にバッファ オーバーランなどの潜在的なセキュリティ上の脅威を検出します。 それでも、未知のセキュリティ上の脅威のすべてを防ぐように設計することは不可能です。 完全なセキュリティを保証できるシステムはありません。 ただし、製品開発は、最初からセキュリティで保護された設計の原則を受け入れるため Skype ビジネス サーバーの組み込まれている業界基準セキュリティ技術、アーキテクチャの基本的な部分として。
    
- **既定で信頼できます。** 既定では、Skype のビジネス サーバー用のネットワーク通信が暗号化されます。 TLS、セキュア リアルタイム転送プロトコル (SRTP)、およびその他の業界標準の暗号化手法が 128 ビット高度暗号化標準 (AES) 暗号化の事実上すべての Skype を含むすべてのサーバーは、証明書や Kerberos 認証を使用するためビジネス サーバーのデータは、ネットワーク上に保護されています。 さらに、役割ベースのアクセス制御は、ビジネス サーバーの各サーバーの役割サービスだけを実行し、サーバーの役割に適したこれらのサービスに関連するアクセス許可のみを持っているので、Skype を実行するサーバーを展開できます。
    
- **導入環境内で信頼できます。** ビジネス サーバー ドキュメントのすべての Skype には、決定して、展開に最適なセキュリティ レベルを構成し、既定以外のオプションをアクティブにするセキュリティ上のリスクを評価するための推奨事項とベスト ・ プラクティスが含まれています。
