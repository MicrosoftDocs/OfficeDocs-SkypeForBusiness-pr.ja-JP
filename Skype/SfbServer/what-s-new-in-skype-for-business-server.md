### <a name="improved-front-end-server-patching-and-upgrade-process"></a>フロントエンド サーバーのパッチおよびアップグレード プロセスの向上

場合、修正プログラムを適用またはサーバーをフロント エンド サーバーにその他のメンテナンスを実行するだけで**呼び出し CsComputerFailOver**を入力、そのサーバーの名前を指定する必要があります。 Skype ビジネス サーバーのプール内の他のサーバーにそのサーバーの負荷を一時的に移動します。 その後、メンテナンスを実行してから、**Invoke-CsComputerFailback** コマンドレットを使用してサーバーをサービスに戻します。 プール内の各サーバーにパッチを適用する必要がある場合は、サーバーごとにそれぞれ 1 回ずつこの手順を実行するだけです。 これらの新しいコマンドレットにより、以前のバージョンよりも迅速に、より高い信頼性と簡単なワークフローで、サーバーにパッチを適用できます。
  
### <a name="improved-front-end-pool-cold-start-capability"></a>フロントエンド プールのコールド スタート機能の向上

### <a name="sql-server-alwayson-support-for-on-premises-servers"></a>オンプレミス サーバーの SQL Server の AlwaysOn のサポート

ビジネス サーバー 2015 の Skype では、AlwaysOn 可用性グループの SQL Server と SQL Server の AlwaysOn フェールオーバー クラスター インスタンスの両方に対するサポートを追加します。 これらの機能だけでなく Skype ビジネス サーバーのデータベース ・ ミラーリングと、Lync Server の過去のバージョンと同様に、SQL Server がクラスタ リングのサポートを継続します。
  
AlwaysOn 可用性グループの SQL Server は、高可用性と災害復旧のソリューションでは、SQL Server 2012 と SQL Server の 2014 の代わりにデータベース ・ ミラーリングを提供します。 可用性グループは、一緒にフェールオーバーされる個別のデータベース セット (可用性データベースと呼ばれる) のフェールオーバー環境をサポートします。 可用性グループは、1 セットの読み取り/書き込みプライマリ データベースと、1 ～ 4 セットの対応するセカンダリ データベースをサポートします。 オプションで、セカンダリ データベースを読み取り専用アクセスおよび一部のバックアップ操作に使用できます。
  
SQL Server フェールオーバー クラスターのインスタンスは、サーバー インスタンス レベルの冗長性により、ローカルの高可用性を提供する Windows Server フェールオーバー クラスタ リング (WSFC) 機能を活用して、フェールオーバー クラスター インスタンス (FCI)。 FCI は、Windows Server フェールオーバー クラスタ リング (WSFC) ノードと、場合によっては、全体で複数のサブネットにインストールされている SQL Server の単一インスタンスです。
  
詳細については、「[Plan for high availability and disaster recovery in Skype for Business Server 2015](plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)」を参照してください。
  
### <a name="centralized-logging-and-troubleshooting-improvements-for-on-premises-servers"></a>オンプレミス サーバーの集中ログとトラブルシューティングの改善点

Skype のビジネス サーバー 2015 の集中ログ サービスを実行できます。 カスタム シナリオを作成できるのと同様に、組み込みシナリオ (定義済みのトレース) は従来のまま変わりません。 常に稼働しており、管理者がほぼリアルタイムで一般的な問題を特定できる、AlwaysOn と呼ばれる特別なシナリオがあります。
  
できる Snooper デバッグ ツールは、移動ログのデバッグを行うにも更新されてし、ビジネス サーバー 2015 の Lync 2013 または Skype のいずれかに接続するデバイスでは動作します。 ツールは、[デバッグ ツール](https://go.microsoft.com/fwlink/?LinkId=285257)からの Web ダウンロードとして利用できます。

## <a name="multi-factor-authentication"></a>複数要素の認証

複数要素の認証は、複数の確認方法の使用を義務付けて、ユーザーのサインインおよびトランザクションに対する重要な第 2 のセキュリティ レイヤーを追加する認証方法です。 たとえば、ユーザー名とパスワード、および証明書が必要とされます。 ビジネス サーバー 2015 の Skype は、Lync Server 2013 累積更新プログラムで利用可能な多要素認証機能の構築を続行します。 複数要素の認証の大幅な変更は次のとおりです。
  
- Exchange および SharePoint との統合への Office 2013 SP1 Active Directory 認証ライブラリの使用
- ビジネス Web アプリケーション クライアントの Skype の複数要素の認証機能のサポート
    
ビジネス多要素認証のための Skype で地理的な場所に基づいて、異なる認証オプションを提供することはようになりました。 たとえば顧客は、内部認証のベースには統合 Windows 認証を使用するとともに、社外から認証を求める従業員には複数要素の認証を使用するように、環境を構成することができます。 
  
多要素認証の業務経験を活かしての Skype は、シームレスに関係なくです。
  
- 地理的な場所は、組織の内外にからユーザー サインインするかどうか 
- クライアントとデバイスの種類 - ビジネスのクライアントは、Skype を使用するデバイス (PC、携帯、iPad など) で、クライアントが実行されています。
