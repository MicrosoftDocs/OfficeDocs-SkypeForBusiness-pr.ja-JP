# <a name="firstline-worker-features-coming-soon-to-microsoft-teams"></a>現場担当者の管理機能が間もなくMicrosoft Teamsに登場

> [!IMPORTANT]
> 2019 年 10 月 1 日より、Microsoft StaffHub が廃止されます。 Microsoft Teams で StaffHub 機能を構築しています。 現在、チームには、スケジュール管理のためのシフトアプリが含まれており、その他の機能も時間の経過と共にロールアウトされます。 2019年10月1日の StaffHub はすべてのユーザーに対して機能しなくなります。 StaffHub を開こうとしたユーザーには、チームをダウンロードするように指示するメッセージが表示されます。 詳細については、「[Microsoft StaffHub の廃止](microsoft-staffhub-to-be-retired.md)」を参照してください。

次に、チームに近い将来の最初の従業員向けの機能の一覧を示します。

- 企業向けお知らせ
- Kronos との統合
- Task
- ホーム画面のランディングページ
- チームにカスタムフィールドを追加する

Teams の独自のプラットフォーム構造のため、Teams では次の機能は利用できません。

- フローコネクタ
- Azure 以外の Active Directory (Azure AD) ユーザーのシフトまたは StaffHub アクセス
- StaffHub の条件付きアクセス
- StaffHub * の Intune MAM
- 組織で従業員のリソースを追加する * *

* StaffHub またはシフトの Intune MAM を管理することはできませんが、Teams で Intune のセキュリティを管理することはできます。 詳細については、「 [Teams のセキュリティとコンプライアンスの概要](../../security-compliance-overview.md)」を参照してください。

* * チームの規模に応じて、共有従業員のリソースを、最初の行の従業員と配布することはできなくなります。 代わりに、チーム内のファイルや web サイトのタブを手動で追加して、コンテンツに簡単にアクセスできるようにすることができます。 詳細については、「 [Teams のタブを使用する方法](https://office365adoption.com/microsoft-teams-tabs/)」を参照してください。 または、Teams Microsoft Graph Api を使用して、同じ共有リソースにアクセスする必要があるすべてのチームに、同じタブを自動的に追加することもできます。 詳細については、「[チャンネルにタブを追加](https://docs.microsoft.com/graph/api/teamstab-add?view=graph-rest-1.0)する」を参照してください。

お客様の最初のラインの労働力のニーズに合わせて機能を構築していて、お客様から[](https://microsoftteams.uservoice.com/forums/555103-public-preview/category/182881-developer-platform)のフィードバックを送信したいと考えています。
