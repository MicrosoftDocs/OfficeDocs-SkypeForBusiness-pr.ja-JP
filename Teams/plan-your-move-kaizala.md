# <a name="planning-for-a-successful-transition-from-kaizala-to-microsoft-teams"></a>移行から移行の成功を計画Kaizala Microsoft Teams

Kaizala から Microsoft Teams への移行に着手する場合は、移行の成功を計画する時間を取る必要があります。[](/microsoftteams/deploy-enterprise-setup?tabs=ChatTeamsChannels#plan-your-deployment) 確実な移行計画により、移行の構造が提供され、すべての関係者が共通の目標に向けて取り組む必要があります。 実証済みのフレームワークを使用して変更を実装すると、今すぐ移行を開始するのに役立ちます。

## <a name="transition-framework"></a>移行フレームワーク

![移行ガイダンスを示す画像](media/plan-for-successful-transition.png)

以下では、移行を成功するための推奨されるアプローチの概要を説明し、フレームワークの各手順が前の手順に基づいているので、フェーズを順番に実行することを推奨します。

- 組織のチームワークを思い描く

- チャンピオンと重要な利害関係者の識別

- プロジェクト スコープの定義

- パイロットMicrosoft Teams

開始する前に、次の確認を行います。

- [Teamsインストールされ、](/microsoftteams/get-clients)[ライセンスが環境](/office365/servicedescriptions/teams-service-description)に割り当てられます。 少なくとも最小ライセンスは、コア チームと、この初期プロジェクト フェーズに参加する早期導入者に割り当てる必要があります。

- デスクトップ アプリケーションとモバイル アプリケーションをダウンロードしました。

### <a name="envision-teamwork-for-your-organization"></a>組織のチームワークを思い描く

![追加の移行ガイダンスを示す画像](media/kaizala-framework-guidance.png)

健全な技術環境、持続的なユーザー採用、そしてビジネス目標の実現はすべて、適切な計画から始まります。 しっかりした計画はアップグレード プロジェクトのためのフレームワークを与え、すべての関係者が同じ目標に向かって取り組んでいることを保証します。 時間を取って、現在使用されているシナリオをKaizalaします。 次に、最新の職場では、その他のシナリオを想定Microsoft Teams。 使用シナリオの定義を開始するには、「使用シナリオを定義する」を参照[Microsoft Teams](/microsoftteams/teams-adoption-define-usage-scenarios)。

### <a name="identify-champions-and-critical-stakeholders"></a>チャンピオンと重要な利害関係者を特定する

まず、適切な関係者を集め、アップグレード計画を定義します。 適切な計画には、移行のスコープ、目標、明確なタイムラインを含める必要があります。 合意された計画を立て、技術環境とエンド ユーザーが準備ができていることを確認Teams。 新しいテクノロジに関心を持ち、他のユーザーを支援するユーザーを特定します。 切り [替え中の興奮](/microsoftteams/teams-adoption-create-champions-program) 、認識、導入を推進するチャンピオン プログラムを作成します。 組織でこのプログラムを開始するためのチャンピオン プログラム ガイドを作成しました。詳細については、「チャンピオン プログラムを構築 [する」を参照してください](https://view.officeapps.live.com/op/view.aspx?src=https://fto365dev.blob.core.windows.net:443/media/Default/DocResources/Adoption/Build_Champions_Program_Guide.pptx)。

### <a name="define-your-project-scope"></a>プロジェクトの対象範囲を定義する

ビジネスへの移行を計画するMicrosoft Teams、ビジネスにどのような成功が見えるかを定義する必要があります。  プロジェクト スコープは、現時点でのプロジェクトの焦点を定義し、プロジェクト チームが現在のタスクに集中し続けるのに役立ち、長期的なビジョンを実現できます。

結果と成功を定義するための推奨される戦略の詳細については、「成功と成功を定義する」を参照[Microsoft Teams。](/microsoftteams/teams-adoption-define-outcomes)

### <a name="pilot-microsoft-teams"></a>パイロットMicrosoft Teams

プロジェクトの範囲とビジョンを定義したら、プロジェクトのパイロットMicrosoft Teams。 初期導入者のユーザー [グループを特定](/microsoftteams/teams-adoption-onboard-early-adopters)し、初期導入者の実験に参加Teams。 パイロットの実施には、ユーザーとの通信、ネットワークと使用状況の監視によるネットワークパフォーマンスと通話品質の維持、参加者からのフィードバックの収集、Teams に関する質問のヘルプ デスク チケットの確認が含まれます。

## <a name="prepare-your-organization"></a>組織を準備する

移行を正常に計画するために推奨されるフレームワークに従った後Kaizala移行を成功に向けて組織をMicrosoft Teams。

### <a name="get-ready"></a>準備をして

 1. すべてのユーザーのライセンスTeams確認します。 [チーム プラン](/microsoft-teams/compare-microsoft-teams-options?rtc=1)と [Microsoft Teams サービスの説明](/office365/servicedescriptions/teams-service-description)を確認します。

 2. 導入[Microsoft Teamsガイダンスを使用](https://adoption.microsoft.com/microsoft-teams/#get-started)して、組織全体Teams展開します。

 3. スキルと導入リソースを通じて、管理者、チャンピオン、およびより大きな組織が、Teamsとのコミュニケーションを受け入れる準備をします。  

 4. 次のトレーニング リソースを使用して、組織内のユーザーが学習し、リソースを活用Teams。

    - IT 管理者の準備については、「管理者トレーニング リソース [」を参照してください](/microsoftteams/itadmin-readiness)。

    - 講師主導のトレーニングについては、「講師主導[のトレーニング」を参照Microsoft Teams](/microsoftteams/instructor-led-training-teams-landing-page)。
  
    - トレーニング ビデオについては、「ビデオ トレーニング[Microsoft Teams参照してください](https://support.microsoft.com/office/microsoft-teams-video-training-4f108e54-240b-4351-8084-b1089f0d21d7?ui=en-us&rs=en-us&ad=us)。
