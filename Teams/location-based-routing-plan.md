## <a name="overview-of-location-based-routing"></a>場所ベースのルーティングの概要

いくつかの国や地域では、長距離通話の呼び出しのコストを削減する公衆交換電話網 (PSTN) プロバイダーをバイパスすることはできません。 この資料では、有料のバイパスの地理的位置に基づいてマイクロソフトのチームのユーザーを制限するのには位置ベースのルーティングを使用する方法について説明します。 この資料は、電話システムの直接のルーティングにのみ適用されます。

ここでことを計画するためには、場所ベースのルーティングとガイダンスの概要が表示されます。 適用し、場所ベースのルーティングを有効にする準備ができたらを参照してください。
- [場所ベースのルーティングのネットワーク設定を展開します。](location-based-routing-configure-network-settings.md)
- [直接ルーティングのための場所ベースのルーティングを有効にします。](location-based-routing-enable.md)

場所ベースのルーティングは、着信または発信の PSTN 通話の時に、ポリシーとユーザーの地理的位置に基づいて、有料のバイパスを制限できる機能です。 

チームのユーザーは、場所ベースのルーティングを有効になっている、次の処理が適用されます。
- 発信の PSTN 通話をするためには、次のいずれかの注意してください。
    - ユーザーのエンドポイントは、場所ベースのルーティングが有効になっている対応するゲートウェイを介して、場所ベースのルーティングと呼び出し出口が有効になっているネットワークのサイトにあります。 
    - ユーザーのエンドポイントは、場所ベースのルーティングを無効になって、場所ベースのルーティングを無効になっているゲートウェイからの出口を呼び出してネットワーク サイトであります。

    送信呼び出しは、他のシナリオで許可されていません。

- PSTN 着信呼を受信するため、同一のネットワーク サイトにユーザーの応答側の端点を置く必要があります、場所ベースのルーティングが有効になっているゲートウェイ経由の呼び出しの ingresses です。 他のシナリオでは、場合など、ユーザーが移動では、呼び出しが許可されていないし、ユーザーの呼び出しの転送設定 (通常はボイス メール) にルーティングされます。
- 他のチームのユーザーをターゲットに PSTN 通話を転送は、転送を開始したユーザーと同じネットワーク サイトでユーザーのエンドポイントを置く必要があります。 転送は、他のシナリオで許可されていません。 
- チームの他のユーザーを PSTN に転送するには、最初の呼び出し元と同じネットワークのサイトにある場所に基づくルーティングを有効になっているゲートウェイを経由の呼び出しを転送する必要があります。 転送は、他のシナリオで許可されていません。

場所ベースのルーティング定義を使用する同じネットワーク領域、サイト、およびサブネット ビジネス サーバー用の Skype を使用します。 有料バイパスは、場所に制限されている場合、管理者は、各 IP サブネットとネットワーク サイトにその場所の場合は、各 PSTN ゲートウェイを関連付けます。 ユーザーの場所は、ユーザーのチームの端点は、PSTN の呼び出しの時点でに接続されている IP サブネットによって決定されます。 ユーザーは、場所ベースのルーティングが各クライアントがそのエンドポイントの場所に応じて個別にルーティングを適用する場合に別のサイトにある複数のチームのクライアントがあります。

## <a name="other-planning-considerations"></a>計画に関するその他の考慮事項

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>設置場所ベースのルーティングの配置の変更

ネットワーク サイトの音声ルーティング ポリシーは使用されません。 代わりに、ユーザーの音声ルーティング ポリシーを使用します。 これは、あるユーザーは他のサイトに移動するには、音声ルーティング ポリシーする必要があります移動のサイトのゲートウェイを意味します。 

### <a name="technical-considerations-for-location-based-routing"></a>場所に基づくルーティングに関する技術的考慮事項

IPv4 と IPv6 のサブネットをサポート、ただし、IPv6 が優先の一致をチェックするとき。 IPv6 のサポートは、現在実行中で、によって全般的な可用性 (GA) の場所ベースのルーティングを使用することが可能になります。 

### <a name="client-support-for-location-based-routing"></a>場所ベースのルーティングのクライアント ・ サポート

チームの次のクライアントがサポートされています。
- デスクトップ クライアント (Windows および Mac) のチーム
- モバイル クライアント (iOS および Android) のチーム
- チームの IP 電話

ビジネス クライアント用の Skype はサポートされていません。

### <a name="capabilities-not-supported-by-location-based-routing"></a>場所に基づくルーティングでサポートされていない機能

場所ベースのルーティングは、次の種類の相互作用に適用されません。 チームの端点が次のシナリオで、PSTN エンドポイントと対話するとき、場所ベースのルーティングを強制されません。 
- 通話保留による PSTN 通話の保留または再開 
- ビジネス ユーザーは、オンプレミス Skype やオンライン ビジネスのユーザーは、Skype は、チームのユーザーを呼び出す
