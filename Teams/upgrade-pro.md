# <a name="upgrade-pro"></a>Pro のアップグレードします。

大きな組織や Pro のアップグレードのアカウントとは、Microsoft を実行して、アップグレードのシナリオのさまざまなアップグレード ライフ サイクルのビジネス (たとえば、ハイブリッドまたはクラウド ボイス導入の場合)、Skype の複雑な展開を持つのために設計されていますとともに Skype ビジネス チームは、組織の準備が完了するまでのチームです。 

3 つのフェーズに分割すると、Pro のアップグレードについて説明します。

-   **アップグレード前**: チームの価値を実現する時間の経過と共に、アップグレードの準備を開始します。 確実に正の経験をする準備が整うと、チームへの移行とアップグレードの前提条件と手順を検討してください。 組織にビジネス用の Skype と共にチームが実行されているまだ場合は、チームにユーザーをアップグレードするのには、組織の準備のための検証チェックポイントとしてこれらのアップグレード前のアクティビティを使用します。
-    **アップグレード**: 組織のユーザーがビジネスの Skype の使用を停止する準備ができたらするトレーニングを受けるし、チームを使用する場合、まだ実行していない) を起動するに十分な通知があることを確認します。 また、ビジネス会議のため、Skype を最適なエクスペリエンスを確実にチームに移動するなどの手順を実行します。
-    **アップグレード後**: 組織全体では、チームでは後の値を最大化するための計画を実装します。 これには、品質のネットワークを監視する、ビジネスを使用するための Skype も、終盤にさしかかっていることを確認しながらチームの採用を推進およびロードマップの進行に伴って、チームとしての機能の拡張の計画が含まれます。
 
> [!NOTE]
> 複数の[モード](https://aka.ms/skypetoteams-coexist)を活用することとは異なって、チームに勢いを維持しながらユーザーのアップグレード操作を制御することが可能にするユーザーのグループをアップグレードするが、チームへの旅が含まれることがわかりました。 可能性があります、アップグレード、旅の展開方法を示すためには、ビジネス用の Skype からモードに移行中**島**チームにだけ旅を定義する計画を以下の例を提供してきました。 さらに、計画の例では、 _cohorts_アップグレードの 4 つのグループにユーザーを分割するが、組織について説明します。 これをテンプレートとして使用して、チームは、さまざまな[モード](https://aka.ms/skypetoteams-coexist)を使用して、アップグレードのグループにユーザーをセグメント化が数を組み込むことに、特定の旅を網羅するように計画をカスタマイズします。

## <a name="upgrade"></a>Upgrade

**チームには、公式の移動を行うためです。** アップグレードすると、ユーザー、**チームだけ**の共存モードに移動します。 チームのチャット、会議、通話、および共同作業では、プライマリ ・ アプリケーションになり、ビジネス アプリケーション、Skype へのアクセスが無効になっています。 この段階の技術的な側面が非常に簡単ですが、変更可能性がありますユーザー エクスペリエンスであり、チームにビジネス用の Skype からは、その活動を正式に移行するユーザーの時間を確保する効果を検討してください。 ユーザーの異なるクライアントとさまざまな経験することを減らすためには、45 日間、またはこれをエンド ・ ツー ・ エンドの「アップグレード」ウィンドウの制限をしてください。

日 1&ndash;2: 組織のアップグレードの準備ができていることを確認します。
-   上記で説明したアップグレード前の活動を完了するかを検証します。
-   アップグレードの 4 つのグループに、ユーザーをセグメント化します。
-   すべてのユーザーにアップグレードの通知を送信します。\*
-   すべてのユーザーには、 [Skype チームが使用可能にアップグレードするため、ビジネス ユーザー向けに通知](https://aka.ms/SkypeToTeams-SetCoexistence)を設定します。

_7 ~ 45 日間: チームにビジネス用の Skype からのアップグレードを実装します。_

7 日目
-   グループ 1 の 2 週間で、アップグレードを開始するをアップグレードするのには通知を送信します。\*

日 14

## <a name="post-upgrade"></a>アップグレード後

**チームとビジネス ・ バリューを最大化します。** チーム、組織を完全にアップグレードすると、自分の目標に対してのお客様の成功を評価し、躍進を続けるための計画を実装する時間がかかります。

日 1&ndash;14: のアップグレードをどの程度あるかを測定します。
-   アップグレード前の段階で設定した目標に対して、最初のアップグレードの成功を評価します。
-   トラック上ではない目標のリスク軽減計画を実装します。

進行中: 推進力とチームの配置のオペレーションの稼働状態を維持します。
-   技術またはユーザーのアップグレード作業を問題のネットワークの稼働状態、およびヘルプデスクの呼び出しを監視し、必要に応じてを軽減します。
-   [電子メール、ニュースレター、またはその他の通信の今後の新機能、成功事例、上のフォーカスを持つユーザーの勢いをドライブ](https://aka.ms/successwithteams)します。
