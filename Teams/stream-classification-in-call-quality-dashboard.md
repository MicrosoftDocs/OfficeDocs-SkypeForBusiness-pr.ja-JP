## <a name="unclassified-streams"></a><span data-ttu-id="dd209-244">未分類ストリーム</span><span class="sxs-lookup"><span data-stu-id="dd209-244">Unclassified Streams</span></span>

<span data-ttu-id="dd209-245">CQD では、対話型接続確立(ICE) 接続が失敗したとき、またはストリームの分類を計算するために必要なすべてのメトリックが報告されない場合、ストリームは [未分類] とマークされます。</span><span class="sxs-lookup"><span data-stu-id="dd209-245">In CQD, a stream is marked _Unclassified_ when Interactive Connectivity Establishment (ICE) connectivity fails or when all the metrics required to compute the stream classification are not reported.</span></span>

<span data-ttu-id="dd209-246">ICE接続障害をチェックするには、 "FAILED"値の "First Connectivity Ice"と "Second Connectivity Ice"のディメンションを調べます。</span><span class="sxs-lookup"><span data-stu-id="dd209-246">To check for ICE connectivity failures, examine the dimensions "First Connectivity Ice" and "Second Connectivity Ice" for a "FAILED" value.</span></span> <span data-ttu-id="dd209-247">いずれかの値が失敗を示す場合、ストリームは [未分類] _としてマークされます_。</span><span class="sxs-lookup"><span data-stu-id="dd209-247">If either value indicates a failure, the stream is marked as _Unclassified_.</span></span>

<span data-ttu-id="dd209-248">未分類ストリームで ICE 接続が成功した場合、主要なストリームメトリックが報告されていないので、ストリームは未分類と見なされる可能性があります。</span><span class="sxs-lookup"><span data-stu-id="dd209-248">If ICE connectivity succeeded for an _Unclassified_ stream, the stream is likely considered _Unclassified_ because key stream metrics were not reported.</span></span> <span data-ttu-id="dd209-249">これらの指標が報告されない理由はいくつかあります。</span><span class="sxs-lookup"><span data-stu-id="dd209-249">There are a few reasons these metrics may not be reported:</span></span>

- <span data-ttu-id="dd209-250">**QoE レポートは受信されません。** 分類に使用されるメトリックは、通話の最後に送信される QoE レポートで報告されます。</span><span class="sxs-lookup"><span data-stu-id="dd209-250">**QoE reports were not received** — The metrics used for classification are reported in a QoE report sent at the end of a call.</span></span> <span data-ttu-id="dd209-251">このレポートが生成されない場合 (たとえば、一部のサード パーティのエンドポイントは QoE を送信できない可能性がある)、または送信できなかった場合 (ネットワークの停止など)、CQD はストリームを分類できません。</span><span class="sxs-lookup"><span data-stu-id="dd209-251">If this report is not produced (for example, because some third-party endpoints may not send QoE) or could not be sent (for example, because of a network outage), CQD is unable to classify the stream.</span></span>

  > [!TIP]
  > <span data-ttu-id="dd209-252">「QoE Record Available」ディメンションを使用して、QoEレポートがストリームに対して受信されたかどうかを判断できます。</span><span class="sxs-lookup"><span data-stu-id="dd209-252">The "QoE Record Available" dimension can be used to determine whether a QoE report was received for a stream.</span></span> <span data-ttu-id="dd209-253">いずれかのエンドポイントからQoEレポートを受信した場合、このディメンションの値は「True」になります。</span><span class="sxs-lookup"><span data-stu-id="dd209-253">Note that this dimension will have a value of "True" if a QoE report was received from either endpoint.</span></span> <span data-ttu-id="dd209-254">最も正確な測定基準の報告には、両方のエンドポイントからのQoEレポートが必要です。</span><span class="sxs-lookup"><span data-stu-id="dd209-254">A QoE report from both endpoints is required for the most accurate reporting of metrics.</span></span>

- <span data-ttu-id="dd209-255">**短い通話** : 短い通話では、キー ストリームのメトリックを計算するのに十分なメディア アクティビティが不足している可能性があります。</span><span class="sxs-lookup"><span data-stu-id="dd209-255">**Short calls** — Short calls may not have enough media activity to compute key stream metrics.</span></span> <span data-ttu-id="dd209-256">これらのメトリックがなければ、CQDはストリームを分類できません。</span><span class="sxs-lookup"><span data-stu-id="dd209-256">Without these metrics, CQD is unable to classify the stream.</span></span>

  > [!TIP]
  > <span data-ttu-id="dd209-257">「期間（秒）」、「期間（分）」、「期間5秒以下」、および「期間60秒以上」のディメンションは、ストリームの期間を決定するために使用できます。</span><span class="sxs-lookup"><span data-stu-id="dd209-257">The dimensions "Duration (Seconds)", "Duration (Minutes)", "Duration 5 seconds or less", and "Duration 60 seconds or more" can be used to determine the duration of a stream.</span></span> <span data-ttu-id="dd209-258">測定値「平均通話時間」を使用して、一連のストリームの平均所要時間を計算することもできます。</span><span class="sxs-lookup"><span data-stu-id="dd209-258">The measurement "Avg Call Duration" can also be used to compute the average duration for a set of streams.</span></span>

- <span data-ttu-id="dd209-259">**パケット使用率が低** い —"短い通話" シナリオと同様に、主要なストリーム メトリックの計算には十分なパケット使用率が必要です。</span><span class="sxs-lookup"><span data-stu-id="dd209-259">**Low packet utilization** — Like the "short call" scenario, sufficient packet utilization is required for computation of key stream metrics.</span></span> <span data-ttu-id="dd209-260">これらのメトリックがなければ、CQDはストリームを分類できません。</span><span class="sxs-lookup"><span data-stu-id="dd209-260">Without these metrics, CQD is unable to classify the stream.</span></span>
  - <span data-ttu-id="dd209-261">一般的な低パケット使用率のシナリオは、出席者が発表者の話を聞くために会議に参加し、話さない場合に発生します (ほとんどの通話ではマイクがミュートされています)。</span><span class="sxs-lookup"><span data-stu-id="dd209-261">A common low packet utilization scenario occurs when an attendee joins a meeting to listen to the presenter, but never speaks (the microphone is muted for most of the call).</span></span> <span data-ttu-id="dd209-262">ここで、クライアントに受信するオーディオ ストリームのパケット使用率は高く、クライアントから送信されるオーディオ ストリームのパケット使用率はほとんどまたは全く発生します。</span><span class="sxs-lookup"><span data-stu-id="dd209-262">Here, the audio stream inbound to the client has high packet utilization while the audio stream outbound from the client has little to no packet utilization.</span></span> <span data-ttu-id="dd209-263">ストリームのデュレーションは 1 時間以上になる場合がありますが、マイクがミュートされ、ストリームの結果が未分類になった場合、クライアントからサーバーへのストリームのパケット使用率は低くなります。</span><span class="sxs-lookup"><span data-stu-id="dd209-263">The duration of the stream may be an hour or longer but the packet utilization on the stream from the client to the server is low since the microphone was muted, and an _Unclassified_ stream results.</span></span>

  > [!TIP]
  > <span data-ttu-id="dd209-264">「パケット使用率」ディメンションおよび「平均パケット使用率」測定値を使用して、ストリームのパケットアクティビティを判断できます。</span><span class="sxs-lookup"><span data-stu-id="dd209-264">The "Packet Utilization" dimension and "Avg Packet Utilization" measurement can be used to determine the packet activity of a stream.</span></span>
