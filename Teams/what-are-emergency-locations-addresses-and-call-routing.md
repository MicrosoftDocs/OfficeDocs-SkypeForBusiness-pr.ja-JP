# <a name="manage-emergency-calling"></a>緊急通報を管理する

この記事では、緊急通報&mdash;を管理するために知っておく必要がある概念について説明します。これには、緊急対応アドレス、動的緊急対応アドレス、緊急通報のルーティングに関する情報が含まれます。 この記事では、次の用語を使用します。

- **緊急時の住所** - 市民の住所&mdash;組織の勤務先の住所または住所。

  たとえば、住所 *12345 North Main Street、Redmond、WA 98052 は、* 緊急通報を適切なディスパッチ機関にルーティングし、緊急通報者の特定を支援するために使用されます。

- **場所** - 通常は、フロア、ビル、棟、またはオフィス番号です。 場所は、建物内のより正確な場所を指定するために緊急対応の住所に関連付けられます。 緊急対応の住所に関連付けられている場所の数に制限はありません。 たとえば、組織に複数の建物がある場合は、各建物と各ビル内のすべてのフロアの場所情報を含めることができます。  

- **緊急対応の場所** - 場所は、オプションの場所を持つ市民の住所&mdash;です。 ビジネスに複数の物理的な場所がある場合は、緊急対応の場所が複数必要になる可能性があります。 

  緊急対応アドレスを作成すると、このアドレスに対して一意の場所 ID が自動的に作成されます。 たとえば、緊急用住所に場所を追加する場合、建物の住所&mdash;にフロアを追加した場合、緊急住所&mdash;と場所の組み合わせに対して場所 ID が作成されます。  この例では、2 つの場所 ID があります。1 つは市民アドレス用です。1 つは、参加している市民の住所と関連する場所の 1 つです。

  緊急対応の場所をユーザーまたはサイトに割り当てると、ユーザーまたはサイトに関連付けられているのは、この一意の場所 ID です。

- **登録済みアドレス** - 各ユーザーに割り当てられる緊急アドレス。 登録されたアドレスは、静的な緊急アドレスまたはレコードのアドレスと呼ばれることもあります。 (現在、登録済みのアドレスはダイレクト ルーティングではサポートされていません。 すぐに更新プログラムを確認してください。)

>[!Note]
>[PSTN 接続](pstn-connectivity.md)に Microsoft 通話プラン、オペレーター接続、またはダイレクト ルーティングを使用しているかどうかに応じて、緊急通話を管理する方法にはいくつかの違いがあります。 これらの考慮事項については、この記事全体で説明します。

## <a name="emergency-address-validation"></a>緊急対応アドレスの検証

ユーザーまたはネットワーク識別子に緊急アドレスを割り当てるには、緊急対応アドレスが "検証済み" としてマークされていることを確認する必要があります。 アドレスの検証では、アドレスが正当であり、割り当てられた後に変更できないことが保証されます。 

Teams管理センターのアドレス マップ検索機能を使用して緊急対応の住所を定義すると、住所は自動的に検証済みとしてマークされます。 住所の形式または表現が変更された場合は、検証済みの緊急&mdash;対応アドレスを変更できないため、更新された形式で新しい住所を作成する必要があります。


## <a name="emergency-address-geo-codes"></a>緊急対応住所の地域コード

各緊急対応住所には、地理コード (緯度と経度) を関連付けることができます。 これらの地域コードは、一部の国で、動的な場所で緊急通報のルーティングを支援するために使用されます。 

Teams管理センターの住所マップ検索機能を使用して緊急対応住所を定義すると、地域コードは自動的に緊急対応住所に関連付けられます。 PowerShell を使用してアドレスを定義する場合は、地理コードをアドレスに関連付けることもできます。 

Microsoft では、Teams管理センターでマップ検索機能を使用して緊急対応の住所を作成することをお勧めします。これにより、住所の書式設定、検証、適切な地域コードの取得が保証されます。 

>[!Important]
>緊急対応の場所を動的緊急通報用のネットワーク識別子に割り当てるには、緊急対応の住所に適切な地域コードが含まれている必要があります。

### <a name="emergency-call-enablement-for-calling-plans"></a>通話プランの緊急通話の有効化

通話プランの各ユーザーは、緊急通報に対して自動的に有効になり、割り当てられた電話番号に関連付けられた緊急対応住所を登録する必要があります。 

電話番号に関連付けられている場所は、国/地域によって異なります。

- たとえば、米国とカナダでは、ユーザーに番号を割り当てるときに緊急の場所が必要です。

- ヨーロッパ、中東、アフリカ (EMEA)&mdash; などの他の国&mdash;では、Microsoft 365から電話番号を取得するとき、または別のサービス プロバイダーまたは運送業者から電話番号を転送する場合に緊急の場所が必要です。


### <a name="dynamic-emergency-calling-for-calling-plans"></a>通話プランの動的緊急通話

通話プランの動的緊急通話では、Teams クライアントの現在の場所に基づいて緊急通報を構成し、ルーティングする機能が提供されます。 適切な Public Safety Answering Point (PSAP) に自動ルーティングを行うか、セキュリティ デスク担当者に通知する機能は、Teams ユーザーの使用国によって異なります。  

緊急通報をルーティングするための動的な場所は、次のように米国でサポートされています。 

- 米国通話プラン ユーザーのTeams クライアントが米国内の緊急アドレスを動的に取得した場合、そのアドレスは登録されたアドレスの代わりに緊急ルーティングに使用され、呼び出しはアドレスのサービス領域の PSAP に自動的にルーティングされます。

- 米国通話プラン ユーザーのTeams クライアントが米国内の緊急アドレスを動的に取得しない場合は、登録された緊急アドレスを使用して通話の画面とルーティングを行います。 ただし、呼び出し元を適切な PSAP に接続する前に、更新されたアドレスが必要かどうかを判断するために呼び出しが画面に表示されます。

カナダでは、緊急通報をルーティングするための動的な場所は、次の例外を除き、米国と同じようにサポートされています。すべての緊急通報は、PSAP に転送される前に国内でスクリーニングされます。

詳細については、「 [動的緊急通報の計画と構成」を](configure-dynamic-emergency-calling.md)参照してください。

### <a name="emergency-call-routing-for-calling-plans"></a>通話プランの緊急通話ルーティング

Teams通話プランのユーザーが緊急電話番号をダイヤルすると、PSAP への通話のルーティング方法は、次によって異なります。

- 緊急対応アドレスがTeams クライアントによって動的に決定されるかどうか。

- 緊急対応の住所が、ユーザーの電話番号に関連付けられている登録済みのアドレスであるかどうか。

- その国の緊急通報ネットワーク。

次に例を示します。

**米国:**

- Teams クライアントがテナント定義の動的緊急対応の場所にある場合、そのクライアントからの緊急通話は、その地理的な場所にサービスを提供する PSAP に自動的にルーティングされます。

- Teams クライアントがテナント定義の動的緊急対応の場所にない場合、そのクライアントからの緊急通話は、その地理的な場所にサービスを提供する PSAP に呼び出しを転送する前に、呼び出し元の場所を決定するために、国内のコール センターによってスクリーニングされます。

- 緊急発信者が緊急対応の場所をスクリーニング センターに更新できない場合、通話は発信者の登録済みアドレスを提供する PSAP に転送されます。

**カナダ、アイルランド、英国では**、緊急通報が最初にスクリーニングされ、通話を適切なディスパッチ センターに接続する前に、ユーザーの現在の場所が決定されます。

**フランス、ドイツ、スペインでは**、緊急通報は、発信者の場所に関係なく、番号に関連付けられた緊急アドレスを提供する PSAP に直接ルーティングされます。

**オランダでは**、緊急通話は、発信者の場所に関係なく、番号のローカル エリア コードの PSAP に直接ルーティングされます。

**オーストラリアでは**、緊急アドレスが構成され、配送業者パートナーによってルーティングされます。

**日本では**、緊急通話はサポートされていません。

## <a name="considerations-for-direct-routing"></a>ダイレクト ルーティングに関する考慮事項

### <a name="emergency-call-enablement-for-direct-routing"></a>ダイレクト ルーティングの緊急通話の有効化

ダイレクト ルーティングの場合は、Teams緊急通報ルーティング ポリシーを使用して、緊急電話番号とそれに関連付けられたルーティング先を定義することで、ユーザーの緊急[通話ポリシー](manage-emergency-call-routing-policies.md)を定義する必要があります。 (現在、登録されている緊急対応の場所は、ダイレクト ルーティング ユーザーではサポートされていません。

緊急通話ルーティング ポリシーは、Direct Routing ユーザー アカウント、ネットワーク サイト、またはその両方に割り当てることができます。 Teams クライアントがネットワーク接続を開始または変更すると、Teamsクライアントが配置されているネットワーク サイトの参照が次のように実行されます。

- 緊急通話ルーティング ポリシーがサイトに関連付けられている場合は、サイト ポリシーを使用して緊急通報を構成します。

- サイトに関連付けられている緊急通話ルーティング ポリシーがない場合、クライアントが未定義のサイトで接続されている場合、またはダイヤルされた番号がサイトに関連付けられている緊急通話ルーティング ポリシーで定義されている緊急電話番号と一致しない場合は、ユーザー アカウントに関連付けられた緊急通話ルーティング ポリシーを使用して緊急通報を構成します。 

- Teams クライアントが緊急通話ルーティング ポリシーを取得できない場合、ユーザーは緊急通話を有効にしません。


### <a name="dynamic-emergency-calling-for-direct-routing"></a>ダイレクト ルーティングを呼び出す動的緊急対応

ダイレクト ルーティングの動的緊急通話では、Teams クライアントの現在の場所に基づいて緊急通報を構成およびルーティングする機能が提供されます。 適切な Public Safety Answering Point (PSAP) に自動ルーティングを行うか、セキュリティ デスク担当者に通知する機能は、Teams ユーザーの使用国によって異なります。

ダイレクト ルーティング ユーザーの場合、緊急通報をルーティングするための動的な場所は、次のように米国でのみサポートされます。

-   米国 ダイレクト ルーティング ユーザーのTeams クライアントが米国内の緊急アドレスを動的に取得した場合、そのアドレスは緊急ルーティングに使用され、通話はアドレスの配信領域の PSAP に自動的にルーティングされます。

-   米国 ダイレクト ルーティング ユーザーのTeams クライアントが米国内の緊急アドレスを動的に取得しない場合、呼び出し元を適切な PSAP に接続する前に、更新されたアドレスが必要かどうかを判断するために呼び出しが画面に表示されます。

カナダでは、緊急通報をルーティングするための動的な場所は、次の例外を除き、米国と同じようにサポートされています。すべての緊急通報は、PSAP に転送される前に国内でスクリーニングされます。

詳細については、「 [動的緊急通報の構成](configure-dynamic-emergency-calling.md)」を参照してください。

### <a name="emergency-call-routing-for-direct-routing"></a>ダイレクト ルーティングの緊急通話ルーティング

ダイレクト ルーティングの緊急通話ルーティング ポリシーは、オンライン PSTN の使用法を参照します。緊急通話を適切な PSTN ゲートウェイに適切にルーティングするには、適切なダイレクト ルーティング構成が必要です。 特に、緊急ダイヤル文字列に OnlineVoiceRoute があることを確認する必要があります。 詳細については、「[ダイレクト ルーティングを構成する](direct-routing-configure.md)」を参照してください。 

> [!NOTE]
> クライアントTeams緊急番号の前に "+" サインインが付加されなくなりました。つまり、+911 です。 その結果、緊急通報Teamsは、911 番号の前に "+" を送信しなくなります。 音声ルート パターンにこの変更が反映されていることを確認します。

Direct Routing ユーザーの緊急通話を動的にルーティングする機能は、特定の国の緊急通話ネットワークによって異なります。 次の 2 つのソリューションを使用できます。

- [緊急ルーティング サービス プロバイダー (米国のみ)](#emergency-routing-service-providers)
- [緊急場所識別番号アプリケーション](#emergency-location-identification-number-applications)

#### <a name="emergency-routing-service-providers"></a>緊急ルーティング サービス プロバイダー

米国には、発信者の場所に基づいて緊急通報を自動的にルーティングできる多数の認定緊急ルーティング サービス プロバイダー (ERSP) があります。

- 緊急ルーティング サービス プロバイダーがダイレクト ルーティング展開に統合されている場合、動的に取得された場所を持つ緊急通話は、その場所にサービスを提供するパブリック セーフティ 応答ポイント (PSAP) に自動的にルーティングされます。

-  動的に取得された場所のない緊急通話は、更新された場所に基づいて適切なディスパッチ センターに呼び出しを接続する前に、ユーザーの現在の場所を決定するために最初にスクリーニングされます。

詳細については、「 [ダイレクト ルーティングに認定されたセッション ボーダー コントローラー」を](direct-routing-border-controllers.md)参照してください。


#### <a name="emergency-location-identification-number-applications"></a>緊急場所識別番号アプリケーション

セッション ボーダー コントローラー (SBC) には、緊急場所識別番号 (ELIN) アプリケーションを含めることができます。 SBC ELIN アプリケーションがダイレクト ルーティング展開に統合されている場合は、ELIN アプリケーションで緊急アドレスと関連する電話番号を構成し、ELIN レコードをそれぞれの PSTN の緊急通話データベースにアップロードする必要があります。 ELIN 識別子を持つ緊急対応の場所Teams、ELIN アプリケーション内の場所と一致する必要があります。

動的に取得された場所を持つ緊急通話が適切な SBC にルーティングされると、ELIN アプリケーションは次のようになります。

- 呼び出し元の緊急の場所を解析します。
- ELIN レコードの場所と一致します。
- 緊急通報者の番号を ELIN 電話番号に置き換える。
- その場所にサービスを提供する PSAP に呼び出しをルーティングし、ディスパッチャーはアップロードされた ELIN レコードから場所を取得します。

緊急電話番号へのコールバック時に、ELIN アプリケーションは元の緊急通報者の番号を逆に置き換えます。 

詳細については、「 [ダイレクト ルーティングに認定されたセッション ボーダー コントローラー」を](direct-routing-border-controllers.md)参照してください。


## <a name="security-desk-notification"></a>セキュリティ デスクの通知

セキュリティ デスク通知は、Microsoft 通話プラン、オペレーター接続、およびダイレクト ルーティングの両方で利用できます。

Teams緊急通話ポリシー (TeamsEmergencyCallingPolicy) を使用して、緊急通話中に通知する必要があるユーザーとその通知方法を構成します。チャットのみ、会議への参加とミュート、会議への参加とミュートが行われますが、ミュートを解除できます。 また、緊急通報を呼び出して参加するユーザーまたはグループの外部 PSTN 番号を指定することもできます。 PSTN パーティはミュート解除を許可されないことに注意してください。

緊急通話ポリシーは、Teams ユーザー アカウント、ネットワーク サイトに割り当てられているユーザー アカウント、またはその両方に付与できます。  Teams クライアントがネットワーク接続を開始または変更すると、Teamsクライアントが配置されているネットワーク サイトの参照が実行されます。

- 緊急通報ポリシーがネットワーク サイトに関連付けられている場合は、サイト ポリシーを使用してセキュリティ デスク通知を構成します。

- サイトに緊急通話ポリシーが関連付けられていない場合、またはクライアントが未定義のサイトに接続されている場合は、ユーザー アカウントに関連付けられた緊急通話ポリシーがセキュリティ デスク通知を構成するために使用されます。  

- Teams クライアントが緊急通話ポリシーを取得できない場合、ユーザーはセキュリティ デスク通知を有効にしません。

緊急通話中は、セキュリティ デスクが通話に会議され、セキュリティ デスク ユーザーのエクスペリエンスは、Teams緊急通話ポリシーに基づいて制御されます。 グループ チャットは各セキュリティ デスク メンバーで開始され、緊急通報者の場所は重要なメッセージ通知を介して共有されます。  会議オプションがポリシーの一部として構成されている場合、各セキュリティ デスク ユーザーは、さらに会議の一部として呼び出されます。

### <a name="custom-emergency-disclaimer"></a>カスタム緊急免責事項

管理者は、ユーザーが E911 を有効にするために、テナントにカスタム バナーを追加できます。 ユーザーは自分のアドレスを確認するときにバナーを閉じることができ、Teamsが再起動されるとバナーが再表示されます。 この機能を有効にするには、緊急通報ポリシーの下で **緊急サービスの免責事項** Teamsを設定し、ユーザーに表示する文字列メッセージを入力します。 このフィールドは、カスタム ポリシーを設定するときに省略可能であり、文字列フィールドは 250 文字に制限されています。

> [!NOTE]
> 現在、これは EnhancedEmergencyServicesDisclaimer ポリシーで PowerShell を使用して構成できます。 今後は、Teams管理センターでも構成できます。
