# <a name="microsoft-teams-adoption-quick-start-checklist"></a>マイクロソフト チーム採用のクイック スタート チェックリスト

## <a name="get-started-driving-adoption-of-microsoft-teams"></a>マイクロソフト チームの採用を推進するを開始します。

- 採用の前提条件

## <a name="phase-1---start"></a>段階 1 - 開始

- チームとチャネルを理解します。
- 最初のチームを作成します。
- チームの準備ができているでしょうか。
- 利害関係者とのミーティングします。
- 初期採用者を識別します。

## <a name="phase-2---experiment"></a>フェーズ 2 - 実験

- エキスパート プログラムを作成します。
- ガバナンスのクイック スタートを完了します。
- 使用シナリオを定義します。
- ビジネス関係者にインタビューします。
- マップし、ビジネス シナリオの優先順位を付ける
- オンボードの早期導入とフィードバックの収集
- オンボードのサポート

## <a name="phase-3---scale"></a>フェーズ 3 - スケール

- 結果と成功のメジャーを定義します。
- サービスの有効化の戦略を選択します。
- 利害関係者を協力します。
- 実装チームを展開します。
- ガバナンスと情報管理のポリシーを展開します。
- フィードバックおよびレポート作成を最適化します。
- ドライブの認識とトレーニングを実装します。
- スケジュール サービスのレビュー
