## <a name="integration-models-for-solution-providers"></a>ソリューション プロバイダーの統合モデル

<a name="steps"></a>

コンタクト センター ソリューション プロバイダーとして、接続されているコンタクト センター ソリューションを次の 3 つのソリューションに統合するために、次の 3 つのTeams。

- 認定 SBC とダイレクト ルーティングを使用して、連絡先センター ソリューションを接続する場合は、Teams モデルをConnect[してください](?tabs=connect#steps)。

- Azure ボットと Microsoft Graph Communication API を使用して、ソリューション プロバイダーが Teams アプリを作成できる場合は、「モデルの拡張」を[参照してください](?tabs=extend#steps)。

- ソリューション プロバイダーがアプリ内のネイティブ エクスペリエンスをTeamsできる SDK を使用する場合は、Power モデル に関するページ[を参照してください](?tabs=power#steps)。 2021 年の終わりに向けて、SDK が利用可能な場合は、Power ソリューションが可能になります。

### <a name="the-connect-model"></a>[**新Connectモデル**](#tab/connect)

このConnectモデルでは、Microsoft 認定の SBC とダイレクト ルーティングを使用して、連絡先センター ソリューションを Teams 電話システム インフラストラクチャに接続し、ルーティング、構成、およびシステムの洞察を強化します。

エージェントは、自動化された仮想アシスタントとスキルベースのルーティング キューを設定して、情報を収集し、顧客を主題の専門家と結び付けできます。

**主要な機能**

これらの機能は、この統合モデルの機能の包括的な一覧ではサポートされませんが、主な領域は次のとおりです。

  - Office 365 CCaaS クライアントからエージェントが Microsoft テナントに接続するための認証N を作成する 

  - エージェントを使用してエージェントを使用できるTeams

  - 転送とグループ通話のサポート (Teams 

  - Teams との統合のための Teams Graph API とクラウド通信 API 

  - ソリューション プロバイダーの SBC で複数の顧客をサポートするマルチテナント SIP トランキング。  

  - Microsoft 認定セッション ボーダー コントローラー [ <span class="underline">(SBC) を使用する</span>ソリューション プロバイダー](../direct-routing-border-controllers.md)


### <a name="the-extend-model"></a>[**拡張モデル**](#tab/extend)

拡張モデルは、Microsoft Teams の Teams [Teams](/microsoftteams/platform/overview)クライアント プラットフォーム、Teams Graph [API、Cloud](/graph/api/resources/teams-api-overview?view=graph-rest-1.0) Communications API を使用して、Teams クライアントと[統合Graph。](/graph/api/resources/communications-api-overview?view=graph-rest-1.0) 拡張モデルでは、すべての連絡先センター通話と通話制御エクスペリエンスに Teams 電話システムも使用され、連絡先センター ソリューション プロバイダーは、Microsoft 365 と共にテレフォニー キャリアとして機能します。

エージェントは、Teams を使用して内部コラボレーションと外部通信を行い、エンゲージメントを開始する前に複数のシステムのデータを関連付ける動的なコンテキスト ノートを利用して、コストの高いコンテキスト切り替えを回避できます。

組織では、個人に向けてワークフローや高度なルーティング構成を設計し、システムとやり取りの品質を測定できます。

**主要な機能**

これらの機能は、この統合モデルの機能の包括的な一覧ではサポートされませんが、主な領域は次のとおりです。

  - Teams との統合のための Teams Graph API とクラウド通信 API 

  - Teams エージェント エクスペリエンスのための新しいアプリ 

  - エージェントのプライマリ通話エンドポイントとしての Teams 

  - すべての通話コントロールを対象とした Teams クライアントの呼び出し

  - Web クライアントとモバイル クライアントTeamsエージェント エクスペリエンス アプリ

  - 分析、ワークフロー管理、CCaaS アプリ内のエージェントのロールベースのエクスペリエンスTeams

  - Teams クライアントと統合されたチャットと共同作業操作 

  - すべてのアプリでの Teams クライアント操作のパフォーマンスと品質の維持  

### <a name="the-power-model"></a>[**Power モデル**](#tab/power)

Power モデルを使用すると、ソリューション プロバイダーは、Teams 呼び出しインフラストラクチャとクライアント プラットフォームを使用してネイティブの Azure ベースの音声アプリケーションを作成し、顧客とエージェントの共同作業を可能にする最新のインテリジェントなソリューションを提供できます。 Power モデルの目的は、1 つのアプリで 1 画面のコンタクト センター エクスペリエンスを提供する方法です。

**主要な機能**

これらの機能は、この統合モデルの機能の包括的な一覧ではサポートされませんが、主な領域は次のとおりです。

  - 正式なエージェントでは、Teams SDK を介してオムニチャネル通信がネイティブで有効になります 

  - エージェントTeams顧客とのやり取りのためにコラボレーション サービスを使用する  

  - クラウド サービスの迅速なプロビジョニング、どこからでも展開 

  - Teams での会話中に、直接会話を制御し、対話することができます 

>[!NOTE]
> Power モデルは 2021 年末に提供される予定です。
