# <a name="prepare-my-users"></a>ユーザーを準備する

この記事では、クラウド音声サービスを使用するためにユーザーを準備するための要件の概要を示します。 ユーザーを適切に準備することで、組織にとって最大限の価値を提供します。

<!--ENDOFSECTION-->

## <a name="preparing-for-user-adoption"></a>ユーザー導入の準備

プロジェクトチームをセキュリティで保護して、プロジェクトの範囲と目標を定義したら、次の手順として、組織とユーザーの準備を確認します。これには、技術的な準備と並行して作業を行うことができます。 展開から価値を引き出すには、ユーザーが実際にそれを使用して導入する必要があります。 ソリューションを有効にするだけでは、お客様の目標を達成できるとは限りません。 さらに、ユーザーはさまざまなユースケースやさまざまな学習スタイルを持ち、さまざまな速度で新しいテクノロジに対応しています。 変更を管理することは、それほど複雑ではありませんが、焦点を絞った作業が必要になります。 以下に記載されているガイド付きのディスカッションは、ユーザーベースを理解し、ユーザーの導入を容易にするための適切なレベルの教育を準備するために設計されています。

アクティビティの所有者: 潜在顧客、プロジェクトリーダー、役員スポンサー、マーケティング/トレーニング/サポートマネージャーとしての導入変更マネージャー

> [!NOTE] 
> 以下に記載されている評価アクティビティは、組織内のあらゆる変更イニシアチブに対応しています。 プロジェクトの範囲に基づいて質問に答えるだけです。 これらのアクティビティについては、「新しいソリューション」を、Skype for Business から Teams への音声通話、電話会議、またはアップグレードに適用することができます。

<!--ENDOFSECTION-->

## <a name="create-your-vision-to-value-statement"></a>ビジョンのステートメントを作成する

プロジェクト構想を設定することで、主要な関係者が全体像を見ることができます。 通常、これは、コスト&mdash;削減、運用効率、生産性&mdash;の向上などのビジネスドライバーと、プロジェクトの前に設定した目標を合わせたものになります。 ユーザーにとっては、同じプロジェクトについて、使い慣れたツール、生産性の中断、管理者による会社間の不足などを表示することができます。 ユーザーの靴に自分を追加して、プロジェクトのビジョンをお客様にとって魅力のある価値のあるステートメントに変換します。 コミュニケーションとトレーニングアクティビティでこの価値の説明を利用して、チームの作業方法をユーザーが確認できるようにします。

## <a name="determine-organizational-change-readiness"></a>組織変更の準備を決定する

組織の準備は、ユーザーが動作方法を変更することに receptive かどうかを判断するのに役立ちます。 ユーザーが変更にどのように対応するかを把握することにより、組織は積極的に問題を解決できるようになり、最適な受信のためにロールアウト計画を調整して、変更を容易にするために、チャンピオンやアンバサダーになるユーザーを特定することができます。

> [!TIP]
> ディスカッションの概要については、過去のロールアウトを評価してください。これは、ユーザーが変更にどのように対応するかを予測するのに役立ちます。また、予期した反応を事前に対処するのに役立ちます。

### <a name="what-percent-of-users-fall-into-each-bucket"></a>各バケットに入るユーザーの割合

| **ユーザーの種類**                                                             | **%** |
|------------------------------------------------------------------------------|-------|
| *最早採用*者は、利用可能になる前に解決策を要求します。                  |       |
| *ユーザー*には、その値が示されるとすぐに、ソリューションが受け入れられます。    |       |
| *Laggards*によって、変更された場合でも、ソリューションは拒否されます。                 |       |


> [!TIP]
> 早い採用者はパイロット展開の候補として、さらに優れたイニシアチブを導入することをお勧めします。 Laggards の割合が高い場合は、これらのユーザーの値のメッセージに焦点を当てて、変更をボードに表示するために、時間をかけて作業を続けます。

### <a name="what-percent-of-users-fall-into-each-competency"></a>各コンピテンシーには、ユーザーの割合はどの程度ありますか?

| **学習の種類**                                                        | **%** |
|----------------------------------------------------------------------------|-------|
| *自己*紹介の学習者はリソースを検索します。その方法について学習します。                         |       |
| *チームプレーヤー*は、グループと対話型のトレーニングをお楽しみいただけます。同僚と一緒に移動します。 |       |
| Seekers を*持っ*ている人は、"白グローブ" または1対1の支援を期待します。                   |       |

 
> [!TIP]
> 学習は、1つのサイズに適合するわけではありません。 ユーザーにとって最適な方法を理解することで、学習を加速する堅牢なトレーニングプログラムを作成できます。 独立しているユーザーが多い場合は、ビデオやユーザーガイドへのリンクが十分な場合があります。 特に注意が必要なユーザーには、1回限り&mdash;のスケジュールを設定するか、またはお客様が学習曲線を介してユーザーを支援するためのエキスパートを登録してください。

## <a name="assess-your-user-base"></a>ユーザーベースの評価

ユーザーレベルの分析を使用すると、ユーザーのペルソナ、cohorts、ユースケースについて理解することができます。 ユーザーが何をしているのかを把握し、そのしくみを理解することによって、変更を実装するときに、適切に対応する必要があるかどうかを確認できます。 これには、対象指定メッセージ、トレーニング、ヘッドセットや web カメラなどのデバイスも含まれます。 たとえば、経営陣は1対1のトレーニングを想定していますが、旅行現場の担当者は、各自の都合に応じて見ることができるトレーニングビデオを利用することができます。 さらに、ユーザーレベルの分析は、新しい機能の使用を開始すべきユーザーを特定するのに役立ちます。また、待機することによって最適なユーザーを見つけることができます。

<table>
<tr><td><img src="media/audio_conferencing_image7.png" alt="An icon depicting decision points"/> <br/>判断のポイント</td><td><ul><li>特定のユーザーまたは cohorts がこのソリューションに対するコアの依存関係を持ち、そのような作業を効率的に行うことができますか。</li><li>この新しいソリューションを使用することで、特定の用途に対応することができますか?</li></ul></td></tr>
<tr><td><img src="media/audio_conferencing_image9.png" alt="An icon depicting the next steps"/><br/>次のステップ</td><td><ul><li>ユーザーペルソナをマップします。 まず、ユーザーをペルソナ/cohort グループ (エグゼクティブ、道路用の戦士、グループ作業員、小売業など) に分割します。 管理性を向上させるには、最大で8つのグループを一覧表示します。 (サンプルの会話の初心については、以下を参照してください。)発見した結果を使って、ユーザーの準備キャンペーンをさらに設計し、後でアップグレードすることによって得られる可能性があるパイロットユーザーをセグメント化します。
</li><li>マップユースケース。 この新しいテクノロジの恩恵を受ける主要なユーザーグループを特定したら、一般的なユーザーの作成について検討してください。 チーム間のユースケースとシナリオについて理解することで、組織がこの新しいテクノロジで作業していることを視覚化できます。 さらに、お客様のコミュニケーションとトレーニングプランにユースケースの例を追加することで、ユーザーにとって適切な情報を提供することができます。</li></ul></td></tr>
</table>

各ペルソナのグループについて、プロジェクトチームで次のことを説明します。

-   このグループの主な特徴は何ですか (つまり、生活の中での一般的な日)。

-   このグループは、新しいソリューションにどのようなメリットがありますか? どの機能を有効にする必要がありますか?

-   必要なデバイス (ヘッドセットやスピーカーなど) と、それらのデバイスが既にインストールされている場合はどうすればよいですか?

-   このグループで新しいソリューションの使用を開始するために必要な追加の付加価値メッセージ (存在する場合) は何ですか?

-   このグループが受け取る最適な方法は、次のとおりです。

    -   認知度 (メール、ポスター、部門マネージャーなど)

    -   トレーニング (オンライントレーニング、ユーザガイド、ホワイトグローブなど)

    -   サポート (セルフサービス、ヘルプデスク、チャンピオン)?  

-   このグループは、パイロットまたは早期採用プログラムの候補として適していますか? その場合は、これらのユーザーからの購入を取得してパイロットに参加するための計画を立ててください。 パイロットプログラムの成功の実装について詳しくは、「 [Skype For business を使用したパイロットチーム](pilot-essentials.md)」をご覧ください。

次の図は、ペルソナテーブルの例です。

![ペルソナの詳細情報]が表示された値の一覧(media/prepare-my-users-image2.png "一般的な役割と、コミュニケーションとコラボレーションソリューションの依存関係を含む、ペルソナの一覧表。エグゼクティブペルソナが強調表示され、プロファイル (ライフサイクル) と機能、デバイス、トレーニングについての詳細が示されます。")

各ユースケースについて、プロジェクトチームで次のことを説明します。

-   このユースケース/シナリオの主な特徴は何ですか?

-   現在の解決策については、どうすればよいですか?

-   現在のソリューションでお客様が直面している問題は何ですか?

-   新しいソリューションはユーザーエクスペリエンスをどのように最適化するか

ユースケースの例については、 [Fasttrack の生産性ライブラリ](https://fasttrack.microsoft.com/microsoft365/productivitylibrary)にアクセスしてください。

## <a name="optimize-your-readiness-plan"></a>準備計画を最適化する
ユーザーがどのようなものであり、新しいソリューションでどのように機能するかを理解したら、準備計画を設計することができます。 お客様のお客様のニーズに合わせて、パイロット、オンボード、新しいソリューションの運用に向けて、ユーザーのコミュニケーション、トレーニング、およびサポートを行う方法が組み込まれています。 この記事の前の手順で発見したように、準備は1つのサイズに収めることはできません。 最初に、このプロセスの最初に定義した価値のあるメッセージを組み込んで、広範な認知度とトレーニング計画を作成します。 次に、必要に応じて、メッセージ、トレーニング、サポートを定義したペルソナまたは cohorts に合わせて調整します。 カスタマイズされたプランでは、ユーザーのニーズを理解して、移行を容易にするために、ユーザーが新しいソリューションにすばやく関連することができます。
