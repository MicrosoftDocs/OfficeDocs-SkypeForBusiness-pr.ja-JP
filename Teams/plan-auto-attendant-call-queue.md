# <a name="plan-for-teams-auto-attendants-and-call-queues"></a>Teams の自動応答と通話キューを計画する

自動応答を使用すると、呼び出し元の入力に基づいて通話をルーティングするメニュー オプションを設定できます。 自動応答のメニュー オプション ("For Sales,Press 1--For Services press 2") - 組織は、着信呼び出しを処理する人間のオペレーターに頼らずに、発信者を迅速に宛先に誘導する一連の選択肢を提供できるようにします。

呼び出しキューは、呼び出し元の待機領域です。 呼び出し元が特定の担当者ではなく、営業やサービスなど、特定の専門分野を持つユーザーに連絡する必要がある場合は、通話キューを使用して、発信者を支援できるエージェントのグループに接続できます。 呼び出し元は、キューに割り当てられたエージェントが呼び出しを実行できるようになるまで保留されます。

自動応答と通話キューを組み合わせて使用すると、発信者を組織内の適切なユーザーまたは部署に簡単にルーティングできます。

## <a name="auto-attendants"></a>自動応答

自動応答の主な目的は、指定されたメニュー オプションへの呼び出し元の入力に基づいて、呼び出し元を適切なユーザーまたは部署に誘導することです。 発信者は、組織内の特定のユーザー、次に利用可能なエージェントと話すのを待つキュー、またはボイスメールを呼び出すことができます。 営業時間、オフ時間、休日には、さまざまな通話ルーティング オプションを指定できます。

メニュー プロンプトは、テキスト読み上げ (システムによって生成されたプロンプト) を使用するか、記録されたオーディオ ファイルをアップロードすることによって作成できます。 音声認識は、ハンズフリー ナビゲーション用の音声コマンドを受け入れますが、電話のキーパッドを使用してメニューを移動することもできます。

各自動応答には、特定の言語とタイム ゾーンがあります。 複数の言語または世界の複数の部分でビジネスを行う場合は、呼び出し元に対応するために必要な数の自動応答を作成できます。

自動応答ごとに、オペレーターを構成できます。 オペレーターの呼び出しを構成してさまざまな宛先に移動できますが、オペレーター機能は、発信者が組織内の特定のユーザーと会話できるように設計されています。

自動応答は、発信者が組織のディレクトリを名前または内線番号で検索できるように構成できます。 自動応答内で、含めるユーザーまたは除外するユーザーのグループを選択することで、ディレクトリ検索に使用できるユーザーを指定できます。 (これは *ダイヤル スコープ* と呼ばれます。

発信者は、直接電話番号(構成されている場合)、または別の自動応答または通話キューからリダイレクトすることによって、自動応答に到達できます。

## <a name="call-queues"></a>通話キュー

通話キューは、物理的な建物の待機室に似ています。 呼び出し元は保留中で待機し、呼び出しはキュー内のエージェントにルーティングされます。 通話キューは、一般的に営業関数とサービス関数に使用されます。 ただし、通話キューは、通話の数がビジー状態の施設の受付担当者など、内部容量を超える場合に使用できます。

呼び出しキューでは、キュー内の呼び出し元の合計数または待機時間が指定した制限を超えた場合に、呼び出しの特定のルーティングを許可します。 通話は、特定のユーザー、ボイスメール、その他の通話キュー、または自動応答にルーティングできます。

自動応答と同様に、呼び出しキューにはそれぞれ言語設定があります。 複数の言語でビジネスを行う場合は、異なる呼び出しキューを使用できます。 エージェントが複数のキューのメンバーである場合は、複数のキューのメンバーを指定できます。

呼び出しキューごとに、キュー内のエージェントが通話をオプトアウトできるかどうかを指定し、Teams でのプレゼンス表示に基づいて通話をルーティングするかどうかを指定できます。

通話キューに電話番号を割り当てることができますが、通話キューでは、オフ時間と休日に個別の通話ルーティングは提供されません。 通話キューに 24 時間 365 日勤務している場合を除き、営業時間内に通話キューにリダイレクトされる自動応答に電話番号を割り当てることをお勧めします。

## <a name="prerequisites"></a>前提条件

自動応答と呼び出しキューを構成するには、次のリソースが必要です。

- 各自動応答と各呼び出しキューの[リソース アカウント](manage-resource-accounts.md)
- Teams ユーザーまたは外部の電話番号から直接ダイヤルできる、リソース アカウントごとに無料のMicrosoft Teams 電話 リソース アカウント ライセンス
- 外部電話番号から直接ダイヤル可能にするリソース アカウントごとに、少なくとも 1 つの [Microsoft サービス番号](getting-service-phone-numbers.md)、オペレーター接続番号、ダイレクト ルーティング番号、またはハイブリッド番号
  - サービス番号は、有料またはフリーダイヤルの番号である可能性があります

> [!NOTE]
> リソース アカウントはサインインに対して無効になっているため、引き続き使用する必要があります。 これらのアカウントでは、チャットとプレゼンスを使用できません。

通話キューから通話を受信するエージェントは、オンラインまたはオンプレミスのユーザーエンタープライズ VoIP有効にする必要があります。 また、通話キューでダイレクト ルーティング番号を使用している場合は、通話を会議または転送する必要があるエージェントにも次のものが必要です。

- 通話キューで転送モードが使用されている場合に割り当てられるオンライン音声ルーティング ポリシー
- 通話キューで電話会議モードが使用されている場合に割り当てられる電話会議ライセンスまたはオンライン音声ルーティング ポリシー

エージェントが通話キュー呼び出しに Microsoft Teams アプリを使用している場合は、TeamsOnly モードである必要があります。

通話キューで通話回線 ID の目的でリソース アカウントを使用する場合、リソース アカウントには Teams Phone リソース アカウント ライセンスが必要であり、次のいずれかが割り当てられている必要があります。

- [通話プラン](calling-plans-for-office-365.md)のライセンスと電話番号が割り当てられている
- [オペレーター接続](operator-connect-plan.md)の電話番号が割り当てられている
- [オンライン音声ルーティング ポリシー](manage-voice-routing-policies.md) (直接ルーティングを使用する場合は電話番号の割り当ては省略可能)

自動応答または通話キューが外部番号に通話を転送する場合は、次に示すように特定のリソース アカウントに Teams Phone リソース アカウント ライセンスが必要であり、次のいずれかが割り当てられている必要があります。

- [通話プラン](calling-plans-for-office-365.md)のライセンスと電話番号が割り当てられている
- [オペレーター接続](operator-connect-plan.md)の電話番号が割り当てられている
- [オンライン音声ルーティング ポリシー](manage-voice-routing-policies.md) (直接ルーティングを使用する場合は電話番号の割り当ては省略可能)

ライセンスを付与するリソース アカウント:
- 外部から通話を転送する他の自動応答または通話キューに自動応答が転送されたときに、通話を受信した最初の自動応答でリソース アカウントにライセンスを付与する
- 他のすべての呼び出しシナリオでは、外部転送を実行する自動応答または通話キューのリソース アカウントにライセンスを付与します

> [!NOTE]
> リソース アカウントに割り当てられた通話プランが無効になったり削除されたりした場合、テナントで使用可能な場合は (リソース アカウントに割り当てられていない) [通信クレジット](what-are-communications-credits.md)が使用されます。 通話プランまたは通信クレジットがない場合、通話は失敗します。

## <a name="business-decisions"></a>ビジネス上の決定

自動応答と通話キューを設定する前に、ビジネスでこれらの機能を使用する方法についていくつかの決定を行う必要があります。 これらの決定によって、自動応答と通話キューを構成するときに選択する設定が決まります。

これらの質問に対する回答を文書化し、構成を実行している管理者に情報を提供します。

- 必要な言語は何ですか? これらの言語はどこで必要ですか。どの部署またはグループですか?
- 発信者からの音声入力を許可するか、またはダイヤル入力のみを許可しますか?
- オフ時間または休日に個別の通話ルーティングが必要ですか? 時間と休日は何ですか?
- 通話キュー内のエージェントに通話のオプトアウトを許可しますか?
- 通話キュー内のエージェントまたはオペレーターがダイヤルアウトした場合に、特定の発信者 ID を持たせたい場合は、
- 組織内の [通話の駐車場と取得](call-park-and-retrieve.md) を有効にして、人や部署間の通話のハンドオフを支援しますか?
- 音声プロンプトの場合は、独自の音声を録音するか、システムによって生成された音声を使用しますか? (システムによって生成された音声は更新が簡単です。

## <a name="technical-decisions"></a>技術的な決定

自動応答と通話キューを使用して発信者を組織内のユーザーに接続する場合、構成を開始する前に技術的な決定事項がいくつかあります。

エージェントは、次の方法でキューを呼び出すために追加できます。

- 個々のユーザー
- 配布リスト
- メールが有効なセキュリティ グループを含むセキュリティ グループ
- Microsoft 365 グループまたは Teams

必要に応じて、キューごとにこれらのオプションを組み合わせて使用できます。 電子メール アドレスを持つグループは、ボイスメールに使用できます。 Teams を使用すると、共有ファイル ストレージとエージェント間のチャット、ボイスメールを受信できる一般的なメールボックス、基幹業務アプリケーションや Power Apps との統合を含めることができる拡張可能なプラットフォームなど、多くの利点があります。

構成を開始する前に、通話エージェントをキューに追加する方法を選択することをお勧めします。

既存の自動応答と通話キュー インフラストラクチャがあり、Teams に移行する場合は、既存の電話番号を新しい自動応答と通話キューに転送する計画が必要です。 別のプロバイダーから番号を移動するには、 [ポート順序](phone-number-calling-plans/port-order-overview.md) を作成する必要がある場合があります。 1 つ以上の新しい電話番号を一時的に取得し、自動応答と通話キュー フローをテストしてから、現在使用している番号に切り替えることが推奨されます。

**会議モード** は、Teams VOIP 通話と PSTN 通話をエージェントに接続するのに要する時間を大幅に短縮する通話キューのオプションです。 会議モードを機能させるには、通話キュー内のエージェントが次のいずれかのクライアントを使用する必要があります。

- Microsoft Teams デスクトップ クライアント、Android アプリ、または iOS アプリの最新バージョン
- Microsoft Phone System バージョン 1449/1.0.94.2020051601 以降
  
エージェントの Teams アカウントを Teams 専用モードに設定します。 要件を満たしていないエージェントは、通話ルーティングリストに含まれません。

エージェントがすべて互換性のあるクライアントを使用している場合は、通話キューの電話会議モードを有効にすることをお勧めします。

**通話ルーティング フロー** プランは、組織に呼び出すユーザーの最も効率的なルーティングを決定するのに役立ちます。 通話ルーティング フローを計画する方法については、「通話ルーティング [フローを計画する」を参照してください](plan-your-call-routing-flow.md)。

## <a name="getting-started"></a>はじめに

この記事の計画タスクを完了したら、次の手順に従って自動応答と通話キューを設定します。

1. 組織の外部から直接ダイヤルすることで、自動応答と通話キューに必要なサービス番号を取得します。 これには、 [別のプロバイダーからの番号の転送](phone-number-calling-plans/transfer-phone-numbers-to-teams.md) や [、新しいサービス番号の要求が含まれる場合があります](getting-service-phone-numbers.md)。

2. 作成する予定の [各リソース アカウントの Teams Phone リソース アカウント ライセンス](teams-add-on-licensing/virtual-user.md) を取得します。 これらのライセンスは無料なので、今後リソース アカウントを変更する場合に備えて、追加のライセンスを取得することをお勧めします。

3. [作成する](manage-resource-accounts.md) 自動応答と呼び出しキューごとにリソース アカウントを作成します。 各アカウントに Teams Phone リソース アカウント ライセンスと、必要に応じてサービス番号を割り当てます。

4. 自動応答で個別の通話ルーティングを行う[休日を作成](set-up-holidays-in-teams.md)します。

5. 必要に応じて、この機能を使用して通話転送に役立つ場合は、 [通話の駐車場と取得を設定](call-park-and-retrieve.md) します。

6. 呼び出しキューの呼び出しエージェントを含めるために使用するグループを作成します。

7. 内線番号によるダイヤルを許可する予定の場合は、ユーザーの内線番号が自分の Azure Active Directory プロファイルに追加されていることを確認します。

上記の手順を完了したら、自動応答を作成し、キューを呼び出す準備が整いました。 自動応答と呼び出しキューは互いに呼び出しをリダイレクトできるため、作成したワークフロー図を参照して、最初に作成する自動応答または呼び出しキューを決定します。 上の図の例では、Contoso のメイン自動応答を作成する前に、営業キューとサポート通話キューを作成します。これは、メインの自動応答が呼び出し元を営業に誘導し、通話キューをサポートする必要があるためです。

自動応答を作成し、キューを呼び出す方法については、次の記事を参照してください。

- [自動応答を設定する](create-a-phone-system-auto-attendant.md)
- [呼び出しキューを作成する](create-a-phone-system-call-queue.md)

ワークフロー、ボット、SMS との統合など、より広範な機能が必要な場合は、[Azure Communication Services](/azure/communication-services/overview)を検討してください。
