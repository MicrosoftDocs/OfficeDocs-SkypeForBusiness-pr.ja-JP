# <a name="best-practices-for-user-feedback-methods-in-microsoft-teams"></a>Microsoft Teams でのユーザー フィードバックの方法のベスト プラクティス

新しいコラボレーション エクスペリエンスの導入は、ユーザーの動作を変更することです。 人間の変化には、トレーニング、励まし、肯定的な例が必要です。 また、移行中にユーザーが聞こえて感じるのも重要です。 組織内で成功したフィードバック 方法を既に確立している場合は、Teamsの実装を含めるためにそれらを拡張することを検討してください。 これをまだ行っていない場合は、次のいずれかの方法を検討してください。

- 任意のユーザーが参加してエクスペリエンスに関するフィードバックを提供できるパブリック チームをMicrosoft Teams内に作成します (パブリック グループは 5,000 ユーザーに制限されていることに注意してください)。

- Yammerを使用して、エクスペリエンスのベスト プラクティスとサポートのためのオープン コミュニティを提供します。

Community主導のフィードバック方法では、チャンピオンとサービス所有者が関与する必要があります。 これらのメンバー間でローテーションされる監視スケジュールを作成することをお勧めします。 1 つのヒントは、Microsoft Teams内に個別のチームを作成し、これらのチャンピオンが共同作業、リソースの共有、ベスト プラクティスを行ってから、より広範なチームと共有することです。 チャンピオンは、新しいサービスを採用するユーザーに対してアドバイザリ ロールを実行し、変更に対してわかりやすい人間の顔を提供し、展開チームの他のメンバーと協力して問題を提起し、解決します。

## <a name="showcases"></a>ショーケース

この道のりを進むにつれて、Microsoft Teamsの創造性が驚きと喜びを持つ従業員の独自の環境でショーケースを見つけることができます。 これらのショーケースを広く共有します。 これにより、他のユーザーが革新的になり、導入曲線の早いユーザーを教育し、環境内でMicrosoft Teamsを最適に使用する方法を考える時間を取ったユーザーに報酬を与えることができます。 四半期ごとに一連のショーケースを特定するという測定可能な目標を設定することは、導入の勢いを維持するのに役立ちます。
