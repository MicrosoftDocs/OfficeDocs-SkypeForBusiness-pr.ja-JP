# <a name="steering-committee"></a>運営委員会

## <a name="steering-committee-overview"></a>運営委員会の概要

運営委員会とは、共同作業を行っている主要な関係者とプロジェクトリーダーの管理グループであり、定義されたビジネスの成果に向けてプロジェクトやプログラムを導くことができます。 運営委員会は、プロジェクトが*どのよう*に提供されるかについて直接責任を負うものではなく、プロジェクトがビジネスにもたらす*もの*です。

すべてのプロジェクトには、合意に基づくビジョンとチャーターが必要です。 プロジェクトの意図した結果を実現するには、ビジョンが明確に定義されている必要があります。これには、監視と保守を行う必要があります。 これは、運営委員会の責任となります。これは、プロジェクトのイニシアチブのために組織の代弁者として、また必要に応じてブロックを削除するために、運営委員会の責任となります。

組織は、運営委員会の結成に大きな配慮をする必要があります。 委員会は、組織全体の変化に対応するために定義したビジネス目標をプロジェクトが達成することを保証する必要があります。プロジェクトの現在のパルスについては定期的に相談して、その過程で発生した障害物を除去します。

委員会は、いくつかの主要な目標を含めるように、そのチャーターを定義する必要があります。

-   プロジェクトチームとエグゼクティブスポンサー、またはエグゼクティブリーダーとの間のバランスを強く維持します。

-   エグゼクティブスポンサーまたは経営陣にプロジェクトの状態を把握する。

-   エグゼクティブスポンサーまたはエグゼクティブリーダーシップチームが、プロジェクトの方向性と入力を提供し、プロジェクト計画、客観的な成果の結果 (OKRs)、およびその他のプロジェクトのアクティビティを調整することによって、プロジェクトへの方向性と入力を確実にします。

運営委員会は、プロジェクトのライフサイクルを通じて、組織のリーダーシップとプロジェクトチームとの間の連携を確実にするために、定期的な期間にわたって対応するものとします。 この重要な会議では、プロジェクトの方向性がリーダーシップの完全なサポートを受けていることを確認し、プロジェクトのリーダーによって提供されたすべてのフィードバックを反映して成功を推進します。 この委員会は、これらの会議を使用して、プロジェクトの状況を把握したり、次のことを把握したりします。

-   ビジネスケースに合わせてビジネスの成果に同意し、プロジェクトがこれらの結果の配信につながるようにします。

-   プロジェクトを確認および承認して、ビジネスケースの正確さとコンプライアンスを確保します。

-   ビジネスケースに加えられた変更を確認して、定義された結果に影響する可能性があることを確認します。

-   プロジェクト成果物の優先順位付けについての戦略的な決定を行い、中間成果物を承認します。

-   委員会から追加の影響が必要な場合は、ギャップ、リスク、および問題を特定、管理、軽減します。

-   エグゼクティブスポンサーまたはエグゼクティブリーダーシップチームから、ステークホルダーのビジネスユニット間でエスカレーション、優先順位付け、競合の解決を行う必要がある問題について、サポートを収集します。 

-   経営陣、変更諮問委員会、またはその他のビジネスおよび IT 関係者に対して、正式なフィードバックや推奨事項を適用します。

## <a name="identify-members-of-the-steering-committee"></a>運営委員会のメンバーを特定する

委員会は、参加者が影響を受けている場合にのみ有効です。 運営委員会は、ビジネスと技術の意思決定者、およびソリューションに寄与する主要なプロジェクトチームリーダーとパートナーチームの両方からの表現を維持する必要があります。

各運営委員会には、高いリーダーシップと明確に定義された役割と責任が必要です。 委員会の業績とそのメンバーの参加に対して現実的な期待を設定するには、割り当てられた役割と責任について文書化し、それに同意する必要があります。 組織全体に効果的に影響を与え、さまざまな観点から把握するためには、この委員会で確実なエグゼクティブスポンサーとビジネスリーダーシップを確保する必要があります。

> [!NOTE]
> 組織およびプロジェクトチームの設計に基づいて、役割と責任を調整します。 小規模の組織では、これらの役割を重ねることができます。

| タイトル | ロール | 問い合わせ | Email |
|----------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------|-----------|
| エグゼクティブ スポンサー | プロジェクトを代弁して、組織のビジョンと共にその方向性が保たれていることを確認します。 委員会がお客様の意思決定を行うための究極の権限を提供します。 | | |
| ビジネス上の意思決定者 (s) | プロジェクトの方向性を組織のビジョンに合わせて維持する | | |

| コアプロジェクトチーム | 技術面または技術以外の特定のコンポーネント (導入と変更管理など) に関する詳細情報を提供する | | |
| 品質チャンピオン | 品質基準をまとめるための責任を持ちます。 | | |

## <a name="schedule-steering-committee-meetings"></a>運営委員会の会議をスケジュールする 

すべての組織とプロジェクトチームは少し異なる方法で構成されていますが、運営委員会の会議は、単なるプロジェクトのスタンドアップ会議にはなりません。 運営委員会の会議の目標を考慮します。エグゼクティブスポンサーまたは役員のリーダーシップからサポートと方向性を収集することで、プロジェクトの進捗状況を把握することができます。 すべての参加者がこの会議に参加していることを確認してください。プロジェクトチーム全体と役員のリーダーの間の直接のメッセージがあり、メッセージが中継されたときに miscommunication がありません。

委員会の会議をスケジュールするときは、プロジェクトと組織のリズムを考慮してください。 毎週または隔週のどちらかのスケジュールを設定してください。エグゼクティブスポンサーの好みと組織のペースに基づきます。

> [!TIP]
> すばやく移動する環境の場合は、会議間で時間がかかりすぎないようにしてください。これにより、会議の頻度が高くなってしまう問題が発生する可能性があります。 ただし、ペースの遅い環境では、毎週会議を開催することで、疲労を招く可能性があるため、運営委員会の有効性が低下する可能性があります。 エクスペリエンスと judgement を使って適切なリズムを判断し、スケジュールを変更できることを常に覚えておいてください。 会議のためだけに会議を行うことはできません。

### <a name="prepare-for-the-meeting"></a>会議の準備をする

全員の時間を最大限に活用するには、会議の準備をする必要があります。 コアチームは、運営委員会会議の教材の準備を始めることができます。 会議の準備に役立つ PowerPoint プレゼンテーションを作成することができます。

会議のプレゼンテーションデッキには、次のアセットが記入され、含まれている必要があります。

-   以前の運営委員会の実施項目のフォローアップ

-   運営委員会の PowerPoint、プロジェクト高レベルの進捗レポート

    -   プロジェクトは、毎月、四半期、年のいずれかの方法で確認できます。

    -   主要なプロジェクトのマイルストーンを特定します。

        -   マイルストーンの重要性と影響について説明します (これらは、リスク登録に記録する必要があります)。

        -   今後の移行については、特にビジネス上の意思決定者に基づいて、今後の移行について決定してください。

    -   プロジェクトの実行日と影響度の高いアクティビティを呼び出します。

    -   委員会にとって重要であると判断された OKRs のレポート必ず、使用状況、ユーザー感情、品質、信頼性のメトリックを含めてください。

-   運営委員会の PowerPoint、project 週間進捗レポート

    -   強調表示と lowlights、重要な問題、および実施項目、および次の手順が含まれています。

    -   日常的な問題に焦点を絞った、全体的なプロジェクトの状況についての詳細な技術情報が記載されています。

-   懸案事項ログと[リスク登録](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)

    -   委員会が追跡している問題の状態を共有します。

        -   すべてのバグに関する更新を提供できるように準備してください。

    -   ビジネスの成果を達成するための成功に影響を与える可能性がある新しい項目を特定します。

        -   プロジェクトへの影響を確認します。

        -   影響を受ける個人または事業単位を特定します。

        -   次のステップと、委員会のメンバーから求められるサポートの内容を確認します。

-   品質の概要: ガイダンスを使用し[て、チームのネットワークを準備](prepare-network.md)します。

    -   これらの品質レポートの結果について説明し、記載されている問題を解決するための推奨事項を記載します。

    -   各週の品質レポートを調整して、次の領域に注目します。

        -   組織の全体的な品質傾向

        -   問題が発生している既知の問題のサイトまたはサイトの状態。サイトの見出しが表示される方向を示すため、または、修復作業からのポジティブな結果の報告

        -   プロジェクトの進捗状況に基づいて、最近有効になったサイトの品質状態

次に、運営委員会の会議を準備するための通話をスケジュールします。 この通話には、コアプロジェクトチームから重要なチームメンバーのみを含める必要があります。これには、委員会で対処される項目を含む、ネットワーク、デスクトップ、操作、他のチームからのリソースが含まれている可能性があります。 運営委員会の通話中にカバーすることを計画しているマテリアルによって、コアプロジェクトチームのメンバーがオフになることは決してありません。

この予備の通話は、運営委員会が対応する前に同期するための優れた方法です。 また、プロジェクトチーム間で開いているアイテム、disagreements、または grievances が明るくなるようにします。 これは、オープンコミュニケーションを促進することを目的としています。会議中に unpleasantly に驚かされないようにします。

この週のレポートをコアチームとパートナー関係者と共にレビューし、懸念事項や質問に対応します。 チーム内で仕事をして、ディスカッションに基づいてコンテンツを調整、追加、または削除したり、運営委員会から追加の説明を求めたりすることに同意します。 各セクションを発表するチームメンバーを割り当てて、フローと連続性を確認します。

### <a name="hold-the-meeting"></a>会議を開催する 

会議自体は、コアチームが実行した prep 作業を自然に継続する必要があります。 これらの会議の目標は、プロジェクトのビジョンを実現するためにコミュニケーションとコラボレーションの雰囲気を育成することです。 会議を成功させるには、次の方法をお勧めします。

1.  会議の予定を設定します。 会議室を特定するのに十分な大きさの表を作成します。 会議室にビデオ機能があることを確認します。 会議室に全員を追加するのが理想的ですが、会議の出席依頼には、会議室と、少なくとも音声を使用したオンライン会議情報が含まれていることを確認する必要があります。 会議に参加するときは、すべてのリモート参加者にビデオの使用を促します。

2.  議題には残しておく必要があります。ただし、対処が必要な集中は発生しません。 可能であれば (その重要性によります)、これらのアイテムを "パーキングロット" セクションに追加して、後でアドレスを入力します。 このトピックが重要すぎて延期したくない場合は、「同意する必要があります」という検討を続けてください。ただし、今後の会議には、他の議題を移動する必要があることにご注意ください。 1つの時間 (5 ~ 10 分) を設定して、委員会にすぐに注意が必要な項目について対処することに同意します。

3.  開いているサポート案件について説明します。 この会議でできるだけ高いレベルのディスカッションを使用してください。チケット数、チケットの種類、主に開いているアイテム。 小規模であっても、プロジェクトのビジネスや効果への影響を理解している場合でも、すべての問題に対処するための準備ができている必要があります。

4.  メモの担当係を委任します。 発表中や加熱中にノートを取るのが困難すぎている。 委員会の決定事項について、詳細が文書化されていることを確認します。 これらの意思決定を送信し、フォローアップメールの一部としてメモを取ります。

5.  他のユーザーの作業工数とユーザーのフィードバックについては、常に正のニュースを宣伝してください (使用可能な場合)。 話をして話します。 フィードバックが完全に否定的である場合は、それについて説明し、否定的なユーザーエクスペリエンスの問題を解決するために何が行われているかを説明します。

    a.  お客様が行っている優秀な作業について、チームメンバー全員に称賛を提供するチャンスを獲得します。 特にリーダーシップチームの前で、チームメンバーが強い作業を行っていることを認識することが重要です。

6.  常に会議の最後を保存して、次の手順に同意し、会議中に承諾されたアクションアイテムの所有者を特定します。

7.  作成したプレゼンテーションを含む出席者のフォローアップメールを送信し、撮ったノートを追加します。 まだ出席していない人も含めてください。

## <a name="next-steps"></a>次のステップ

運営委員会の概要を確認した後で、行動します。プロジェクトチームは、運営委員会をプロジェクトのタイムラインに実装し、定期的な運営委員会の会議を準備するための作業を開始する必要があります。 理想的には、プロジェクトが開始される前に委員会が形成され、すべてのフェーズ全体でガイダンスを提供しますが、そのような状況にならない場合は、プロジェクトチームの成果を導くための委員会を形成します。

1.  運営委員会のメンバーを特定します。 Ask 長らく:

    -   ビジネスとリーダーシップから適切な表現を持っていますか?

    -   自分が識別できるメンバーは、他の単位での事業単位またはピアの影響を受けますか?

2.  運営委員会の各メンバーに送信する通信を作成します。

    -   ロールテーブルを含めます。

    -   委員会の重要性を売り込む。

3.  最初の会議を設定して、役割と責任の討議、委員会のチャーターの定義、OKRs の検証または定義を行います。 議題には次の項目も含めることができます。

    -   会議のリズムについて説明します。

    -   チャーターと OKRs が、意味のあるビジネス上の成果につながることを検証します。

    -   戦略が OKRs と連携していることを確認します。

4.  すべての会話の品質を開始:

    -   品質支持を持つことの重要性を明確にします。

    -   品質推進者の役割を果たす人またはチームを特定します。

運営委員会の成功は、運が良ければ発生しません。 最善策として、適切なメンバーを選択すると、パフォーマンスが低下する可能性があります。 運営委員会の成功を促進するために、次のものがあることを確認する必要があります。

-   適切に定義された、明確で測定可能なビジネスの結果を得た、プロジェクトの合意に基づくビジョンとスコープ。委員会が advocating します。

-   明確会議を推進する、明確に開催された議題。

-   オープンコミュニケーションの大気。

    -   コミュニケーションとコラボレーションを開くと、委員会が強化されます。 すべてのボイスが聞こえて意見が尊重される環境を作ります。 独自性のある視点が最も影響を与える可能性があります。

    -   オープン環境は、リーダーがメンバーの共同作業とコミュニケーションを促進するときに有効にすることができます。 次に、いくつかのガイドラインを示します。

        -   委員会のメンバー全員が、組織に導入されている変更に対して、強力な代弁者になる必要があります。

        -   正確な画像を提供するために必要となる、またはビジネスユニット全体の意思決定を行うことができるユーザーに、委員会のサイズを制限します。 ブロックが表示される場合、組織の階層と各メンバーの直接の影響を理解することが重要です。

        -   プロジェクトマネージャーとチームリーダーは、分科会のディスカッションと deliberations で協力してください。 追加のコアチームメンバーを招待して、特定の領域で必要となる可能性のある詳細なディスカッションを明確にすることができるようにします。

ソリューションのライフサイクルを通じて、委員会の役割が変更されます。 構想とオンボードがドライブ値のフェーズに移動すると、委員会にいくつかの調整が必要になることがあります。

-   会議の間隔を調整します。 組織の動的を見て、会議の頻度を少なくする必要があるかどうかを確認します。 組織の変更が dwindle されると、高レベルの通信の頻度が減少する可能性があります。 必要に応じて、会議の間隔または四半期ごとの会議の間隔を変更する

-   会議の担当者を調整します。 ソリューションがドライブ値のフェーズに移動すると、コアプロジェクトチームが新しいプロジェクトに再割り当てされる可能性が高くなります。 サポートとして報告されているチームを更新する必要があります。操作は、ソリューションの状態に応じて潜在顧客になります。

-   オペレーショナル・ OKRs の定義とレポート。 重要な結果と成功インジケーターは、ソリューションのライフサイクルを通じて変わります。 目的のビジネスの成果は、引き続き追跡して報告する必要があります。 ビジネスの成果に向けて推進するコンテキストで、音声ソリューションの品質と信頼性について引き続き報告します。 その他のメトリックと主要な結果 (ユーザー感情、ユーザー導入、サポート基準など) が、成功のトップインジケーターとなります。

-   より大きな運営委員会の傘を使用して、複数のソリューションのドライブ値のフェーズをカバーします。 運営委員会をより効率的に使用できるチャンスを探します。 委員会を使って、ソリューションの大きな球体に関するガイダンスを提供します。
