# <a name="list-of-session-border-controllers-certified-for-direct-routing"></a><span data-ttu-id="1ab52-103">ダイレクト ルーティングに対応する認定済みセッション ボーダー コントローラーのリスト</span><span class="sxs-lookup"><span data-stu-id="1ab52-103">List of Session Border Controllers certified for Direct Routing</span></span>

<span data-ttu-id="1ab52-104">マイクロソフトは、選択したセッション ボーダー コントローラー (SBC) ベンダーと提携して、SBC がダイレクト ルーティングと連携していることを保証しています。</span><span class="sxs-lookup"><span data-stu-id="1ab52-104">Microsoft partners with selected Session Border Controllers (SBC) vendors to certify that their SBCs work with Direct Routing.</span></span> 

<span data-ttu-id="1ab52-105">Microsoft は、各ベンダーと次の取り合いを行っています。</span><span class="sxs-lookup"><span data-stu-id="1ab52-105">Microsoft works with each vendor to:</span></span>

- <span data-ttu-id="1ab52-106">SIP 相互接続プロトコルで共同で作業します。</span><span class="sxs-lookup"><span data-stu-id="1ab52-106">Jointly work on the SIP interconnection protocols.</span></span>
- <span data-ttu-id="1ab52-107">サードパーティラボを使用して、テストを行います。</span><span class="sxs-lookup"><span data-stu-id="1ab52-107">Perform intense tests using a third-party lab.</span></span> <span data-ttu-id="1ab52-108">テストに合格したデバイスだけが認定されます。</span><span class="sxs-lookup"><span data-stu-id="1ab52-108">Only devices that pass the tests are certified.</span></span> 
- <span data-ttu-id="1ab52-109">実稼働環境とプリプロダクション環境で、認定済みのすべてのデバイスで毎日のテストを実行します。</span><span class="sxs-lookup"><span data-stu-id="1ab52-109">Run daily tests with all certified devices in production and pre-production environments.</span></span> <span data-ttu-id="1ab52-110">実稼働前環境でデバイスを検証することにより、クラウド内の新しいバージョンのダイレクト ルーディング コードが認定済み SBC で機能することが保証されます。</span><span class="sxs-lookup"><span data-stu-id="1ab52-110">Validating the devices in pre-production environments guarantees that new versions of Direct Routing code in the cloud will work with certified SBCs.</span></span> 
- <span data-ttu-id="1ab52-111">SBC ベンダーとの共同サポート プロセスを確立します。</span><span class="sxs-lookup"><span data-stu-id="1ab52-111">Establish a joint support process with the SBC vendors.</span></span>


  > [!NOTE]
  > <span data-ttu-id="1ab52-112">Microsoft は、認定されたデバイスまたはデバイスがダイレクト ルーティングを介して接続されている場合にのみ電話システムをサポートします。</span><span class="sxs-lookup"><span data-stu-id="1ab52-112">Microsoft only supports Phone System if a certified device or devices are connected through Direct Routing.</span></span> <span data-ttu-id="1ab52-113">Microsoft は、認定されていないデバイスが直接ルーティングを通じて電話システムに接続されているサポート ケースを拒否する権利を持っています。</span><span class="sxs-lookup"><span data-stu-id="1ab52-113">Microsoft reserves the right to reject support cases where a non-certified device is connected to the Phone System through Direct Routing.</span></span> <span data-ttu-id="1ab52-114">Microsoft が顧客のダイレクト ルーティングの問題がベンダーの SBC デバイスにあると判断した場合、顧客は SBC ベンダーにサポートを提供する必要があります。</span><span class="sxs-lookup"><span data-stu-id="1ab52-114">If Microsoft determines that a customer’s Direct Routing issue is with a vendor’s SBC device, the customer will need to engage the SBC vendor for support.</span></span>
