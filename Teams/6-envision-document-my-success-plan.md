# <a name="document-my-success-plan"></a>成功計画を文書化する

この記事では、クラウドボイスの展開を適切に文書化するための要件の概要を示します。 クラウドの音声展開を計画しながら、すべての意思決定点と次の手順を定義して文書化することで、すべての関係者とプロジェクトチームメンバーが成功した結果を提供できるように調整できます。 

## <a name="execution-planning"></a>実行計画 

組織の通話プランソリューションで電話会議や電話システムを実装する方法を定義したら、実装プロジェクトの実行を計画する必要があります。

組織に1つまたは2つのサイトしかない場合は、この記事で説明されているすべての詳細情報を入力する必要はありませんが、解決方法について説明します。

## <a name="deployment-model"></a>展開モデル 

組織でのユーザーの作業方法を変革するテクノロジの実装と同様に、展開を行うための適切な方法を選ぶことは、クラウド音声の実装の成功に大きく影響します。

次のような展開モデルが考えられます。

-   **サイトごと:** このモデルは、組織が地理的に分散しており、支社に従業員数が多い場合に適しています。 ただし、この展開モデルは、部門従業員が複数の場所に分散している部門内の通信を中断する可能性があります。

-   **部門ごと**: 通常、中小企業向けにこのモデルが適しています。また、所属する部署が同じエクスペリエンスを備えていることが保証されます。

-   **一度に会社全体:** 通常、このモデルは小規模企業向けの最適な選択肢であり、すべての従業員が1日の展開の1つと同じエクスペリエンスを実現します。

## <a name="quarterly-execution-plan"></a>四半期ごとの実施計画

管理された作業の単位でクラウド音声の展開を実行するには、客観的な目標 (OKRs)、選択した展開モデル、組織のプロジェクト実行機能に基づいて、四半期ごとの実行プランを作成することをお勧めします。

これにより、四半期ごとに進捗状況を管理したり、必要に応じてプランを変更したり、組織のキャパシティに基づいてクラウド音声機能を展開したりすることができます。

組織に1つまたは2つのサイトしかない場合は、短時間で完全に展開されることを想定しているため、四半期ごとの実行プランは必要ありません。

## <a name="finalize-my-success-plan"></a>成功計画を完成する

成功計画は、Envision フェーズで作成したドキュメントの概要です。

成功計画には、FastTrack または展開パートナーを含めることができるプロジェクトチームが用意されています。組織の目標を達成するための十分な情報は、電話会議または電話システムと通話プランサービスを実装することによって実現できます。

通常、成功計画には次のメインセクションが含まれています。これには、ビジョン化フェーズを通じて、次のような機能があります。

-   ビジネス ケース

-   サービスの準備状況

-   サービスの決定事項

-   実行計画

-   導入計画

-   運用計画

### <a name="business-case"></a>ビジネス ケース

ビジネス用のユースケース、関係者の一覧、OKRs、KSIs、リスクレジスタ、およびプロジェクトタイムラインは、通常、ビジネスケースに必要な大量の情報を作成します。 これらは、成功計画の一部として文書化する必要があります。

### <a name="service-readiness"></a>サービスの準備状況

お客様の環境評価は、電話会議や電話システムを導入するための組織の技術上の準備を決定するために必要な初期情報を提供します。

ここに含まれるのは、サービス準備評価と、環境評価によって検出された修復が必要な領域に対処するためのプランです。

### <a name="service-decisions"></a>サービスの決定事項

電話会議または電話システムを計画した方法を文書化して、組織のための通話プランサービス技術実装を利用します。

### <a name="execution-plan"></a>実行計画

組織全体にソリューションを実装するために、プロジェクトの実行を計画した方法を文書化します。

### <a name="adoption-plan"></a>導入計画

導入準備の評価を実行した後、プロジェクトチームは、一連のコミュニケーション計画、トレーニング計画、および起動前、立ち上げ、発売後の導入アクティビティの計画を把握している必要があります。

お客様の組織の要件を満たすために必要なカスタマイズと共に、チラシ、ウェルカムメール、トレーニング資料などの導入活動をサポートするためのリソースを特定します。

[Microsoft Teams のカスタマーサクセスキット](https://www.microsoft.com/download/details.aspx?id=54244)から導入活動用のテンプレートをダウンロードします。

### <a name="operational-plan"></a>運用計画

運用ロールのマッピングを行うと、役割と責任が確立され、各運用ロールに割り当てられているチームが、電話会議の実装をサポートする必要があります。

ソリューションの運用準備を確実に行うため、この作業を完了させて、成功計画の一部として運用計画を含める必要があります。
