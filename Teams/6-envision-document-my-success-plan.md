# <a name="document-my-success-plan"></a>成功計画を文書化する

この記事では、クラウド ボイスの展開について正しく文書化するための要件の概要を取り上げます。 クラウド ボイスの展開について計画しながら、判断ポイントと次の手順すべてを定義および文書化することによって、すべての関係者とプロジェクト チームのメンバーが足並みを揃えて成功に向けて進むことができます。 

## <a name="execution-planning"></a>実行計画 

組織で電話会議や通話プランが設定された電話システムのソリューションを実装する方法を定義した後、実装プロジェクトの実行について計画する必要があります。

組織に 1 つまたは 2 つのサイトしかない場合は、この記事で説明されているすべての詳細を実行する必要はありませんが、ご使用のアプローチの指針として全体を読み通してください。

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>展開モデル 

組織内の人々の働き方を変革するテクノロジを実装する場合と同様、展開を行うための適切な方法を選ぶことは、クラウド ボイスの実装を成功させるうえで大きな影響を及ぼします。

次のような展開モデルが考えられます。

-   **サイトごと:** このモデルは、組織が地理的に分散しており、支社の従業員数が多い場合に適しています。 ただし、この展開モデルは、部門従業員が複数の場所に分散している部門内での通信を中断する可能性があります。

-   **部署ごと**: このモデルは通常、中規模の企業に適しています。また、この場合には関係する各部門が同じエクスペリエンスになります。

-   **一度に会社全体:** 通常、このモデルは小規模の企業に最適な選択肢であり、すべての従業員が展開の初日から同じエクスペリエンスを利用できます。

## <a name="quarterly-execution-plan"></a>四半期実行計画

管理可能な作業単位でクラウド ボイスの展開を実行するには、目標の主要結果 (OKR)、選択した展開モデル、組織のプロジェクト実行能力に基づいて、四半期ごとの実行計画を策定することをお勧めします。

これにより、四半期ごとに進捗状況を追跡し、必要に応じて計画を変更したり、組織の実行能力に基づいてクラウド ボイス機能を展開したりできます。

組織に 1 つまたは 2 つのサイトしかない場合は、短時間で完全に展開できることが想定されるので、四半期ごとの実行計画は必要ない可能性があります。

## <a name="finalize-my-success-plan"></a>成功計画を完成させる

成功計画は、想定段階で作成した文書の要約です。

成功計画によって、FastTrack や展開パートナーを含めることができるプロジェクト チームには、電話会議や通話プランが設定された電話システム サービスを実装する組織の目標を実現するために十分な情報が提供されます。

通常、成功計画には次のような主なセクションが含まれています。これらの多くは想定段階で取り組みます。

-   ビジネス ケース

-   サービスの準備

-   サービスの決定

-   実行計画

-   導入計画

-   運用計画

### <a name="business-case"></a>ビジネス ケース

一般的に、ビジネス ケースに必要な主な情報には、ビジネス ユース ケース、関係者の一覧、OKR と KSI、リスク登録、プロジェクト タイムラインがあります。 これらを成功計画の一部として文書化する必要があります。

### <a name="service-readiness"></a>サービスの準備

環境評価によって、電話会議や通話プランが設定された電話システムの計画を実装するための組織の技術的な準備状況について判断するうえで必要な初期情報を把握できます。

これには、サービス準備評価と、環境評価で検出された改善が必要な分野に対応するための計画が含まれます。

### <a name="service-decisions"></a>サービスの決定

組織で電話会議や通話プランが設定された電話システム サービスを技術的に実装するために、どのように計画したかを文書化します。

### <a name="execution-plan"></a>実行計画

組織全体にソリューションを実装するために、プロジェクトの実行をどのように計画したかを文書化します。

### <a name="adoption-plan"></a>導入計画

導入準備評価を実行した後、プロジェクト チームは、コミュニケーション計画、トレーニング計画、開始前、開始中、開始後の導入アクティビティに関する計画で構成される包括的な計画を立案する必要があります。

チラシ、ウェルカム メール、トレーニング資料といった導入アクティビティをサポートするリソースを特定し、さらに組織の要件を満たすために必要なカスタマイズを特定します。

[Microsoft Teams Customer Success Kit](https://aka.ms/TeamsCustomerSuccess) から導入アクティビティのテンプレートをダウンロードします。

### <a name="operational-plan"></a>運用計画

運用の役割のマッピングに関する演習では、役割と責任を確立し、電話会議の実装をサポートするために必要な運用の各役割に割り当てるチームを決定します。

ソリューションの運用準備を整えるには、この点を完了させ、成功計画の一部として運用計画を含める必要があります。
