### <a name="test-for-network-impairments"></a><span data-ttu-id="089b4-236">ネットワーク障碍のテスト</span><span class="sxs-lookup"><span data-stu-id="089b4-236">Test for network impairments</span></span>

<span data-ttu-id="089b4-237">ユーザーの満足度を高めるために、ネットワーク上の障害を制限する必要があります。</span><span class="sxs-lookup"><span data-stu-id="089b4-237">To increase user satisfaction, you should limit any impairments on your network.</span></span> <span data-ttu-id="089b4-238">最も一般的なネットワーク障害としては、遅延 (待ち時間)、パケット損失、ジッタがあります。</span><span class="sxs-lookup"><span data-stu-id="089b4-238">The most common network impairments are delay (latency), packet loss, and jitter:</span></span>

- <span data-ttu-id="089b4-239">**待ち時間:** これは、IP パケットがネットワーク上のポイント A からポイント B に到達するまでにかかる時間です。</span><span class="sxs-lookup"><span data-stu-id="089b4-239">**Latency:** This is the time it takes to get an IP packet from point A to point B on the network.</span></span> <span data-ttu-id="089b4-240">このネットワーク伝播遅延は、さまざまなルーターの間の追加のオーバーヘッドを含めて、2つのポイントとライトの速度の間の物理的な距離に関連しています。</span><span class="sxs-lookup"><span data-stu-id="089b4-240">This network propagation delay is essentially tied to physical distance between the two points and the speed of light, including additional overhead taken by the various routers in between.</span></span> <span data-ttu-id="089b4-241">待ち時間は、一方向またはラウンドトリップの時間として測定されます。</span><span class="sxs-lookup"><span data-stu-id="089b4-241">Latency is measured as one-way or round-trip time.</span></span>

- <span data-ttu-id="089b4-242">**パケット損失**: これは、特定の時間帯に失われるパケットの割合として定義されることがよくあります。</span><span class="sxs-lookup"><span data-stu-id="089b4-242">**Packet loss**: This is often defined as a percentage of packets that are lost in a given window of time.</span></span> <span data-ttu-id="089b4-243">パケット損失は、オーディオの品質に直接影響を与えます。これは、音声が完全に途切れることを引き起こすバックツーバックのバースト損失に影響を及ぼすことなく、個々の損失の小さいパケットに影響します。</span><span class="sxs-lookup"><span data-stu-id="089b4-243">Packet loss directly affects audio quality—from small, individual lost packets having almost no impact to back-to-back burst losses that cause audio to cut out completely.</span></span>

- <span data-ttu-id="089b4-244">**パケット間の着信のジッターまたは単なるジッター:** これは、連続するパケット間の遅延の平均変更です。</span><span class="sxs-lookup"><span data-stu-id="089b4-244">**Inter-packet arrival jitter, or simply jitter:** This is the average change in delay between successive packets.</span></span> <span data-ttu-id="089b4-245">Skype for Business などの最新の VoIP ソフトウェアは、多くの場合、バッファリングによって一定レベルのジッターに対応できます。</span><span class="sxs-lookup"><span data-stu-id="089b4-245">Most modern VoIP software, including Skype for Business, can adapt to some levels of jitter through buffering.</span></span> <span data-ttu-id="089b4-246">これは、ジッタがジッターの効果に気付くというバッファリングを超える場合に限られます。</span><span class="sxs-lookup"><span data-stu-id="089b4-246">It's only when the jitter exceeds the buffering that a participant will notice the effects of jitter.</span></span>

<span data-ttu-id="089b4-247">これらの障碍の最大値については、「[メディアの品質とネットワーク接続性のパフォーマンス](/SkypeForBusiness/optimizing-your-network/media-quality-and-network-connectivity-performance)」を参照してください。</span><span class="sxs-lookup"><span data-stu-id="089b4-247">The maximum values for these impairments are described in [Media quality and network connectivity performance](/SkypeForBusiness/optimizing-your-network/media-quality-and-network-connectivity-performance).</span></span> <span data-ttu-id="089b4-248">このような障碍をテストするときは、2つの異なるセグメントを区別します。</span><span class="sxs-lookup"><span data-stu-id="089b4-248">When testing for these impairments, we distinguish between two separate segments:</span></span>

- <span data-ttu-id="089b4-249">*エッジセグメント*とは、ルーターが存在するセグメントのことです。</span><span class="sxs-lookup"><span data-stu-id="089b4-249">The *edge segment* is the segment in which your router lives.</span></span> <span data-ttu-id="089b4-250">これは、各地域でインターネットに接続されている最も近い論理ネットワークセグメントです。</span><span class="sxs-lookup"><span data-stu-id="089b4-250">This is the closest logical network segment connected to the internet at each of your locations.</span></span> <span data-ttu-id="089b4-251">ほとんどの場合、これはルータの接続ポイント、または境界ネットワーク ( *dmz*、*非武装地帯*、*スクリーンサブネット*とも呼ばれます) になります。</span><span class="sxs-lookup"><span data-stu-id="089b4-251">In most cases, this is the connection point of the router, or possibly a perimeter network (also known as *DMZ*, *demilitarized zone*, and *screened subnet*).</span></span> <span data-ttu-id="089b4-252">ルーター以外のデバイスに影響を与えるトラフィックは、このセグメントとインターネットの間で行われることはありません。</span><span class="sxs-lookup"><span data-stu-id="089b4-252">No further traffic that affects devices other than the router should occur between this segment and the internet.</span></span>

- <span data-ttu-id="089b4-253">*クライアントセグメント*は、クライアントが存在する論理ネットワークセグメントです。</span><span class="sxs-lookup"><span data-stu-id="089b4-253">The *client segment* is the logical network segment in which your clients reside.</span></span>

<span data-ttu-id="089b4-254">ネットワーク評価ツールを使用して、両方のセグメントをテストする必要があります。</span><span class="sxs-lookup"><span data-stu-id="089b4-254">You should test both segments by using the Network Assessment Tool.</span></span> <span data-ttu-id="089b4-255">セグメントをテストするには、ディレクトリに移動し、コマンドプロンプトで**networkassessmenttool**と入力します。</span><span class="sxs-lookup"><span data-stu-id="089b4-255">To test the segment, navigate to the directory and enter **networkassessmenttool.exe** at the command prompt.</span></span> <span data-ttu-id="089b4-256">結果は、.tsv という名前のファイルに書き込まれ、各セグメントの[要件](/SkypeForBusiness/optimizing-your-network/media-quality-and-network-connectivity-performance)と比較できます。</span><span class="sxs-lookup"><span data-stu-id="089b4-256">The results are written to a file named Results.tsv, and you can compare them to the [requirements](/SkypeForBusiness/optimizing-your-network/media-quality-and-network-connectivity-performance) for each segment.</span></span>

<span data-ttu-id="089b4-257">どちらのセグメントも、高品質の展開の要件を満たしている必要があることに注意してください。</span><span class="sxs-lookup"><span data-stu-id="089b4-257">Note that both segments must meet the requirements for a high-quality deployment.</span></span> <span data-ttu-id="089b4-258">ネットワークのパフォーマンスを適切に示すために、このツールを1時間に複数回実行することをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="089b4-258">We recommend that you run the tool multiple times for one hour straight to get a good indication of your network’s performance.</span></span>
