# <a name="use-call-analytics-to-troubleshoot-poor-call-quality"></a>通話分析を使用して低品質の通話のトラブルシューティングを行う

この記事では、通話分析を使用して、Teams 管理者または Teams コミュニケーション サポート スペシャリストまたはエンジニアの場合に、個々のユーザーの低品質の Microsoft Teams 通話や会議の品質をトラブルシューティングする方法について説明します。

## <a name="call-analytics-permissions"></a>通話分析のアクセス許可

この記事では、通話分析を既に設定済みである前提とします。 まだ行っていない場合は、「通話分析をセットアップする」を参照[Teams。](set-up-call-analytics.md)

## <a name="introduction-to-call-analytics"></a>通話分析の概要

通話分析では、アカウント内の各ユーザー Teams通話と会議に関する詳細情報がOffice 365されます。 これには、デバイス、ネットワーク、接続性、通話品質に関する情報が含まれます (これらの情報は、通話や会議の品質低下の要因になる可能性があります)。 建物、サイト、テナントの情報をアップロードすると、この情報も通話と会議ごとに表示されます。 通話分析を使用すると、ユーザーの通話や会議のエクスペリエンスが低下した理由を把握できます。

通話分析では、1 人の参加者から 2 人目の参加者まで、通話や会議の各時間が表示されます。 これらの詳細を分析することで、管理者はTeams領域を分離し、品質の低下の根本原因を特定できます。

管理者Teams、各ユーザーのすべての通話分析データにフル アクセスできます。 さらに、Azure Active Directory の役割をサポートスタッフに割り当てることができます。 これらのロールの詳細については、「サポートとヘルプデスクスタッフに [アクセス許可を付与する」を参照してください](set-up-call-analytics.md#give-permission-to-support-and-helpdesk-staff)。 以下の各サポート ロール[のTeamsをお見逃](#what-does-each-teams-support-role-do)しなく。

## <a name="troubleshoot-user-call-quality-problems"></a>ユーザーの通話品質に関する問題のトラブルシューティング

1. Teams 管理センター ( ) を開き、Teams通信のサポートまたは管理者の資格情報Teams <https://admin.teams.microsoft.com> サインインします。

2. [**ダッシュボード**] の [**ユーザー検索**] で、トラブルシューティングを行う呼び出しを行うユーザーの名前またはSIP アドレスを入力するか、[ユーザーの表示] を選択してユーザーの一覧を表示します。

3. リストからユーザーを選択します。

4. [**通話履歴**] を選択し、トラブルシューティングする通話または会議を選択します。

5. [**詳細**] タブを選択して、通話の品質が低下したり、接続の問題が発生していることを示す黄色と赤色アイテムを探します。

   各通話や会議のセッションの詳細において、小さな問題は黄色で表示されます。 何かが黄色の場合は、正常な範囲を外し、問題の原因になる可能性がありますが、問題の主な原因になる可能性は低いと思います。 赤色の場合は重大な問題であり、このセッションの通話が低品質である主な原因の可能性があります。

まれに、オーディオ セッションでは Quality of Experience データが受信されません。 多くの場合、これは削除された呼び出し、またはクライアントとの接続が終了した場合に発生します。 このような場合、セッションの評価は **使用** できません。
