# <a name="record-an-issue-ticket"></a>問題チケットを記録する

MTR Pro がフラグを設定しなかったあらゆる種類の問題について、顧客または顧客の代理のパートナーは、問題の記録機能を使用できます。 これは、 **内部またはパートナー** のサポートに関する問題の追跡を容易にすることを目的としています。 

> [!NOTE]
> Microsoft は、問題チケットの記録を監視しません。 問題にさらなる支援が必要な場合は、[Microsoft Teams Rooms Pro管理のサポートを受ける](https://aka.ms/mtrp/openingTickets)方法に関するこちらの記事を参照してください。

[説明] フィールドで、できるだけ問題を説明します。 1 つ以上のルームを追加できます。

インシデントが重大な会議室の正常性の大幅な低下を引き起こしている場合は、[ **高い影響** ] を確認してサポートの重要性を高めます。

[問題を記録する] の一部として部屋が選択されている (または部屋が選択されている) 場合、記録された問題が閉じられるまで、各部屋は **異常** としてマークされます。

## <a name="closing-record-an-issue-tickets"></a>終了 発行チケットを記録する

記録された問題は、顧客またはパートナーが顧客に代わって会議室を管理することで閉じることができます。

**記録された問題チケットを閉じるには**

1. [ **チケットを閉じる] を選択します**。

2. 一覧からカテゴリを選択して、閉じの理由を選択します。

   終了の理由を確認すると、チケットは閉じられ、 **解決済** みに移動されます。

チケットの [詳細の概要] セクションには、チケットが顧客/パートナーの名前で閉じられたことが表示されます。  

## <a name="faq"></a>よくあるご質問 (FAQ)

**誰でもチケットを閉じることができますか?**

ユーザーが閉じることができるのは、顧客から報告されたインシデントのみです。 チケット管理アクセス許可を持つ会議室を管理するためにパートナーを割り当てたお客様は、記録された発行チケットを閉じられます。

**チケットを閉じるユーザーを制限できますか?**

現時点ではありません。 チケット管理アクセス許可を持つすべてのユーザーは、記録された発行チケットを閉じることができます。

**問題チケットの記録が終了したときに通知を受け取りますか?**

現時点ではありません。
