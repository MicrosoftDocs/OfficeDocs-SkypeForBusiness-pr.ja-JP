# <a name="plan-for-sip-gateway"></a>SIP ゲートウェイの計画

SIP ゲートウェイを使用すると、組織は、Microsoft Teamsと互換性のある SIP デバイスを使用して、SIP デバイスへの投資を維持できます。 これで、会社の資格情報を使用してTeamsにサインインし、互換性のある SIP デバイスを使用して通話を発信および受信できます。 互換性のあるデバイスは、標準の SIP ファームウェアを備えた IP フォン、マルチプラットフォーム SIP ファームウェアを備えた Cisco IP フォン、Poly、Yealink、AudioCodes などのベンダーからの SIP デバイスをSkype for Businessできます。 SIP ゲートウェイ用の SIP デバイスを構成する方法については、「SIP ゲートウェイの [構成」を](sip-gateway-configure.md)参照してください。

## <a name="benefits-of-sip-gateway"></a>SIP ゲートウェイの利点

SIP ゲートウェイは、互換性のある SIP デバイスをTeamsに接続して、ユーザーがTeamsテレフォニーにシームレスに移行できるようにします。 SIP ゲートウェイを使用すると、ユーザーは次のすべての操作を実行できます。

- **呼び出しを行う:** SIP デバイス ユーザーは、公衆交換電話網 (PSTN)、他の SIP デバイス、およびTeamsおよびSkype for Business ユーザーに対して通話を行うことができます。 SIP デバイス ユーザーは、電話番号を持つユーザーのみを呼び出すことができます。
- **呼び出しを受信します。** SIP デバイス ユーザーは、PSTN、SIP デバイスを持つTeamsユーザーまたはSkype for Business ユーザー、およびTeamsおよびSkype for Business クライアント アプリケーションから通話を受け取ることができます。 SIP デバイスは、Teams エンドポイントとして機能します。 受信呼び出しは、ユーザーの SIP デバイスにもフォークされます。
- **複数の同時呼び出し:** 通話の SIP デバイス ユーザーは、通話を保留にして、他の通話を発信または受信できます。 SIP デバイス ユーザーは、2 つの呼び出しを電話会議することもできます。
- **応答しない:** SIP デバイス ユーザーは、デバイスが着信呼び出し用に呼び出されないように、デバイスで応答を停止するように設定できます。 これは、他のすべてのTeams エンドポイントに対するユーザーの状態には影響しません。
- **保留/再開とミュート/ミュート解除:** SIP デバイス ユーザーは、デバイス上のこれらのアクションの機能を使用して、通話の保留と再開、ミュート解除を行うことができます。
- **ボイスメール：** SIP デバイス ユーザーは、発信者が自分のために残す電子的に保存された音声メッセージをリッスンできます。
- **メッセージ待機インジケーター:** SIP デバイス ユーザーは、新しいボイスメール メッセージがあるときにアラートを送信する通知を受け取ることができます。
- **サインインとサインアウト:** SIP デバイスユーザーは、デバイスでサインインしてTeamsからサインアウトできます。
- **デュアルトーンマルチ周波数:** SIP デバイスユーザーは、番号キーを押して、対話型音声応答通話中に入力を提供できます。
- **Teams会議:** SIP デバイス ユーザーは、会議のアクセス番号をダイヤルすることで、Teams会議に参加できます。 会議参加者は、ユーザーの電話番号にダイヤルアウトするか、[参加要求] をクリックして参加者を追加するだけで、SIP デバイス ユーザーを会議に追加できます。また、ユーザーの SIP デバイスにもアラートが表示されます。 別の組織のゲスト ユーザーは、ゲスト ユーザーの番号にダイヤルアウトしてそのゲストを含める参加者によって、Teams会議に追加できます。
- **通話転送:** SIP デバイスユーザーは通話を転送できます。 SIP ゲートウェイは、ブラインド転送とコンサルティング転送の両方をサポートします。
- **ローカル通話転送:** SIP デバイス ユーザーは、デバイスの転送ルール (常に、タイムアウト時、ビジー状態) を設定できます。 デバイスが SIP ゲートウェイに接続されている場合、デバイス ユーザーが設定したルールに基づいて、呼び出しがターゲット アドレスにリダイレクトされます。 ローカル通話転送を機能させるには、管理者が属性`Set-CsTeamsCallingPolicy``Enabled`を `AllowCallRedirect` .

## <a name="requirements-to-use-sip-gateway"></a>SIP ゲートウェイを使用するための要件

TEAMSユーザーは、SIP ゲートウェイを使用するために PSTN 通話が有効になっている電話番号を持っている必要があります。

### <a name="hardware-software-and-licenses"></a>ハードウェア、ソフトウェア、ライセンス

3PIP または SIP デバイスがある場合は、次の機能が必要です。

- 電話システムのライセンス (E5 またはスタンドアロン ライセンス経由)
- Microsoft Teams通話プラン、ダイレクト ルーティング、またはオペレーター接続を介した PSTN の有効化 (電話番号など)
- 共通領域デバイスの共通領域電話 ライセンス
