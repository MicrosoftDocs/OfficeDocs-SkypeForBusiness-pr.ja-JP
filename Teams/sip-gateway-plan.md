# <a name="plan-for-sip-gateway"></a>SIP ゲートウェイの計画

SIP ゲートウェイを使用すると、組織は、SIP デバイスへの投資を維持するためにMicrosoft Teams互換性のある SIP デバイスを使用できます。 これで、会社の資格情報を使用Teamsにサインインし、互換性のある SIP デバイスで通話を送受信できます。 互換性のあるデバイスには、Skype for Business SIP ファームウェアを使用する IP 電話、マルチプラットフォーム SIP ファームウェアを使用する Cisco IP Phone、Poly、Yealink、AudioCodes などのベンダーの SIP デバイスを使用できます。 SIP ゲートウェイ用に SIP デバイスを構成する方法については、「SIP ゲートウェイの構成 [」を参照してください](sip-gateway-configure.md)。

## <a name="benefits-of-sip-gateway"></a>SIP ゲートウェイの利点

SIP ゲートウェイは、互換性のある SIP デバイスTeams接続して、ユーザーがテレフォニーにシームレスに移行Teamsします。 SIP ゲートウェイを使用すると、ユーザーは次の操作を行います。

- **通話を行う:** SIP デバイス ユーザーは、公衆交換電話網 (PSTN)、その他の SIP デバイス、およびユーザーに電話Teams発信Skype for Businessできます。 SIP デバイス ユーザーは、電話番号を持つユーザーにのみ通話できます。
- **通話を受信する:** SIP デバイス ユーザーは、PSTN、SIP デバイスを持つ Teams または Skype for Business ユーザー、およびクライアント アプリケーションの Teams および Skype for Business から通話を受信できます。 SIP デバイスは、アプリケーション エンドポイントTeamsします。 受信呼び出しは、ユーザーの SIP デバイスにもフォークされます。
- **複数の同時呼び出し:** 通話中の SIP デバイス ユーザーは、通話を保留にし、他の通話を行う、または受信することができます。 SIP デバイス ユーザーは、2 回の通話を会議することもできます。
- **[取り込み中]:** SIP デバイス ユーザーは、デバイスが着信呼び出しで呼び出し音を鳴らさないように、デバイスで応答不可を設定できます。 これは、他のすべてのエンドポイントに対するユーザーの状態Teamsはありません。
- **保留/再開とミュート/ミュート解除:** SIP デバイス ユーザーは、デバイス上のこれらのアクションの機能を使用して、通話を保留または再開またはミュート解除できます。
- **ボイスメール:** SIP デバイス ユーザーは、発信者が送信する電子的に保存された音声メッセージを聞き取ります。
- **メッセージ待機インジケーター:** SIP デバイス ユーザーは、新しいボイスメール メッセージが届くときに通知を受け取ります。
- **サインインとサインアウト:** ユーザーがデバイスにサインインしてサインインし、Teamsにサインアウトできる SIP デバイス。
- **デュアルトーンマルチ周波数:** SIP デバイス ユーザーは、対話形式の音声応答呼び出し中に数値キーを押して入力を提供できます。
- **Teams:** SIP デバイス ユーザーは、会議のアクセスTeamsダイヤルすることで、会議に参加できます。 会議の参加者は、ユーザーの電話番号にダイヤルアウトするか、[参加を要求] をクリックして参加者を追加するだけで、SIP デバイス ユーザーを会議に追加できます。 別の組織のゲスト ユーザーは、ゲスト ユーザーの番号にダイヤルアウトしてそのゲストを含める参加者が Teams 会議に追加できます。
- **通話転送:** SIP デバイス ユーザーは通話を転送できます。 SIP ゲートウェイは、ブラインド転送とコンサルティング転送の両方をサポートします。
- **ローカル通話の転送:** SIP デバイス ユーザーは、デバイスの転送ルール (常に、タイムアウト時、ビジー時) を設定できます。 デバイスが SIP ゲートウェイに接続されている場合、呼び出しはデバイス ユーザーが設定したルールに基づいてターゲット アドレスにリダイレクトされます。 ローカル呼び出しの転送を機能するには、管理者が 属性を に `AllowCallRedirect` 設定する必要 `Set-CsTeamsCallingPolicy` があります `Enabled`。 


## <a name="requirements-to-use-sip-gateway"></a>SIP ゲートウェイを使用する要件

Teams SIP ゲートウェイを使用するには、PSTN 通話が有効になっている電話番号が必要です。

### <a name="hardware-software-and-licenses"></a>ハードウェア、ソフトウェア、ライセンス

3PIP または SIP デバイスを使用している場合は、以下が必要です。 
- (E5 または電話システムを使用して) ライセンスを取得する
- PSTN の有効化 (つまり、電話番号) は、Microsoft Teams 通話プラン、直接ルーティング、またはオペレーター サービスを使用Connect
- 共通領域デバイス電話共通領域ライセンス
