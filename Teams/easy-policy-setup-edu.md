# <a name="use-the-teams-for-education-policy-wizard-to-easily-apply-policies-for-a-safe-learning-environment"></a>Microsoft Teams for Education ポリシー ウィザードを使用して、安全な学習環境にポリシーを簡単に適用する

## <a name="overview"></a>概要

Microsoft Teams for Education ポリシー ウィザードを使用すると、学生と教育者のポリシーの管理が簡略化されます。 これを使用すると、安全で生産性の高い学習エクスペリエンスの作成に関連する最も重要な一連のポリシーを簡単かつ迅速に適用できます。

Teamsのポリシーを使用すると、環境でのTeamsの動作方法と、ユーザーが使用できる機能を制御できます。 たとえば、いくつかの名前を付ける通話ポリシー、会議ポリシー、メッセージング ポリシーがあり、組織のニーズに合わせて各ポリシー領域をカスタマイズできます。

安全で集中的な学習環境を維持するには、学生がTeamsで何ができるかを制御するポリシーを設定することが重要です。 たとえば、ポリシーを使用して、プライベート チャットとプライベート通話を使用できるユーザー、会議をスケジュールできるユーザー、共有できるコンテンツ タイプを制御できます。 ポリシーを使用して、学習エクスペリエンスを強化するTeams機能を有効にすることもできます。

学習エクスペリエンスを安全に保つために、学生と教育者の両方に対してポリシーを調整する必要があります。 学生が不適切なレベルのアクセスを受けるリスクを軽減するには、より制限の厳しいポリシーが必要です。 教育者とスタッフは、それらを成功させるために、より許容できるポリシーのセットを個別に必要とします。 たとえば、教育者が会議のスケジュールを設定し、学生が会議を行うのを制限できるようにします。

:::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="ウィザードのスクリーンショット。":::

この記事では、ウィザードを実行する方法について説明します。

> [!IMPORTANT]
> ウィザードによって適用されるポリシーは、大多数のMicrosoft Teams for Education顧客のニーズを満たします。 ウィザードは、学生の安全のために推奨される設定を使用して、コア ポリシー セットのグローバル (組織全体の既定) 定義を調整し、学生に適用します。 ウィザードでは、一連のカスタム ポリシーも作成し、教育者とスタッフに割り当てます。 ほとんどのMicrosoft Teams for Educationのお客様は、このウィザードの実行後に他のポリシー割り当て方法を使用する必要はありません。 他のポリシー割り当て方法は、学生、教育者、スタッフのポリシーを手動で作成および管理する場合 *にのみ* 使用します。

## <a name="teams-for-education-policy-wizard"></a>Microsoft Teams for Education ポリシー ウィザード

<a name="polwiz_intro"> </a>

ウィザードでは、一連のコア ポリシー定義が学生に適用され、個別のコア ポリシー定義のセットが教育者とスタッフに適用され、それぞれに適した設定が適用されます。 ウィザードの実行時の動作を次に示します。

ウィザードは、教育機関の種類 (**初等教育または中等教育または****高等教育**) に基づいてポリシーを設定します。 教育機関の種類を選択すると、ウィザードは次の操作を行います。

- **学生**: ウィザードは、ウィザードの対象となる各ポリシー領域のグローバル (組織全体の既定) ポリシー定義を、学生の安全を維持するのに適した新しい既定の設定で調整します。 これにより、現在の学生とすべての新入生が最も制限の厳しい一連のポリシーを確実に取得できます。
- **教育者とスタッフ**: ウィザードは、教育者とスタッフのニーズに合わせて調整された設定を使用して、ウィザードの対象となるポリシー領域ごとに一連のカスタム ポリシー定義を作成します。 次に、選択した教育者とスタッフのグループにポリシー定義を割り当てます。 このように、教育者とスタッフは、より寛容な一連のポリシーを取得して、それらを成功させます。

ウィザードを 1 回だけ実行する必要があります。 新しい学生は、ウィザードによって適用されたグローバル (組織全体の既定) ポリシー定義を自動的に取得し、選択したグループに追加した新しいスタッフにはカスタム ポリシーが自動的に割り当てられます。

また、新しい機能がTeamsに追加されるたびに、管理者の介入を必要とせずに、その機能に対するポリシーの適切な EDU 関連の既定値が自動的にグローバル (組織全体の既定値) に追加されます。 これは、学生の安全と関与を維持するために適切なポリシーが確実に実施されるようにするのに役立ちます。

> [!NOTE]
> [ウィザードによって適用されるポリシー定義の](#policies-applied-by-the-wizard)詳細な一覧については、ウィザードによって適用されるポリシーに関するページを参照してください。

それでは、作業を開始しましょう。

## <a name="run-the-wizard"></a>ウィザードを実行する

<a name="polwiz_run"> </a>

ウィザードを実行するには、次の手順に従います。

1. Teamsを初めて使用する場合は、ウィザードが自動的に開始されます。 それ以外の場合は、ダッシュボードからいつでもウィザードを開始できます。 Microsoft Teams管理センターの左側のナビゲーションで、[**ホーム**] に移動し、[**安全な学習環境の簡易ポリシー設定**] タイルで [**クイックセットアップ**] を選択します。

    :::image type="content" source="media/easy-policy-setup-quick-setup.png" alt-text="ダッシュボードのウィザードのスクリーンショット。":::

2. 教育機関の種類 (**初等教育または中等****教育または教育機関**) を選択し、[**次へ**] を選択します。

    :::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="教育機関の種類を選択するウィザードのページのスクリーンショット。":::

3. 教育者とスタッフを含むグループを検索して選択し、[ **次へ**] を選択します。 教育者とスタッフ用にまだグループが設定されていない場合は、 [グループを作成](/microsoft-365/admin/create-groups/create-groups)してからウィザードを再実行します。 <br/><br/>最大 3 つのグループを選択できます。 選択したグループの教育者とスタッフには、ニーズに合わせて調整された [一連のカスタム ポリシー](#policies-applied-by-the-wizard) が割り当てられます。 この一連のポリシーは、学生に適用されるポリシーとは別であることに注意してください。

    :::image type="content" source="media/edu-policy-wizard-add-3-groups.png" alt-text="教育者グループとスタッフ グループを選択するウィザードのページのスクリーンショット。":::

4. 選択内容を確認します。

    :::image type="content" source="media/edu-policy-wizard-3-groups-review.png" alt-text="選択内容を確認するためのウィザードのページのスクリーンショット。":::

5. [ **適用]** を選択して変更を適用します。 完了までに数分かかる場合があります。<br/><br/>グローバル (組織全体の既定) ポリシー定義は、すぐに学生に適用されます。 教育者とスタッフの場合、グループのサイズによっては、選択したグループの各メンバーにカスタム ポリシーが割り当てられるまでに数時間かかる場合があります。 これは、この手順が正常に完了した後、バックグラウンドで発生します。
6. あなたは途中ですが、まだ完了していません。 考慮すべき点がいくつかあります。 次に、この記事のウィザードを [実行した後の操作](#what-to-do-after-running-the-wizard) に関するセクションの手順を確認します。

    :::image type="content" source="media/easy-policy-setup-on-way.png" alt-text="次の手順に関するウィザードのページのスクリーンショット。":::

## <a name="what-to-do-after-running-the-wizard"></a>ウィザードの実行後の操作

<a name="polwiz_remove"> </a>

### <a name="step-1-remove-existing-policy-assignments-that-conflict-with-policies-applied-by-the-wizard"></a>手順 1: ウィザードによって適用されたポリシーと競合する既存のポリシー割り当てを削除する

> [!IMPORTANT]
> **ウィザードを実行する *前* に、既存のポリシーが学生または教育者およびスタッフに割り当てられている場合にのみ、この手順を完了します**。 Teamsを初めて使用していて、ウィザードによって作成されたポリシー以外の既存のポリシーがない場合は、これをスキップして手順 2 に進みます。

Teamsでは、特定のポリシー領域に対して、次の方法でポリシーをユーザーに適用できます。

- ユーザーへの直接割り当て
- ユーザーがメンバーであるグループへの割り当て
- ユーザーがポリシーを直接割り当てられていない場合、またはポリシーが割り当てられているグループのメンバーでない場合、ユーザーはグローバル (組織全体の既定) ポリシーを自動的に取得します。

ユーザーに対してこれらのポリシー割り当てが複数存在する場合、Teamsは次の順序を使用して、有効になるポリシーの割り当てを決定します。 詳細については、「 [どのポリシーが優先されるか](policy-assignment-overview.md#which-policy-takes-precedence) 、または [グループの優先順位ルール」を](assign-policies-users-and-groups.md#precedence-rules)参照してください。

|ユーザーのポリシー割り当て|有効なポリシー|
|---|---|
|グループに割り当てられたポリシー: いいえ <p> ユーザーに直接割り当てられたポリシー: いいえ|グローバル (組織全体) の既定のポリシー|
|グループに割り当てられたポリシー: いいえ <p> ユーザーに直接割り当てられたポリシー: はい|ユーザーに直接割り当てられたポリシー|
|グループに割り当てられたポリシー: はい <p> ユーザーに直接割り当てられたポリシー: はい|ユーザーに直接割り当てられたポリシー|
|グループに割り当てられたポリシー: はい <p> ユーザーに直接割り当てられたポリシー: いいえ|グループに割り当てられたポリシー <p> ユーザーが複数のグループのメンバーであり、各グループに同じポリシー領域のポリシーが割り当てられている場合、 [グループの割り当て順位](assign-policies-users-and-groups.md#group-assignment-ranking) が最も高いポリシーが有効になります。|

この順序のため、ユーザーが既存の直接割り当てまたはグループ割り当てを持っている場合、ウィザードによって作成されたポリシーは有効になりません。 つまり、ウィザードによって適用されたポリシーが有効になるように、ユーザーから既存のポリシーの割り当てを削除する必要があります。

[ウィザードによって適用されるポリシー領域](#policies-applied-by-the-wizard)ごとに、次の操作を行います。

- ウィザードによって適用されるグローバル (組織全体の既定) ポリシー定義が有効になるように、既存のすべての直接割り当てとグループ割り当てを学生から削除します。
- ウィザードによって作成されたカスタム ポリシー定義が有効になるように、教師とスタッフの競合する直接割り当てを削除します。 上記の表を使用して、自分に当てはまるシナリオを決定します。 <p> ウィザードでは、グループの割り当て順位 1 を使用して、教育者とスタッフ グループにポリシーが割 [り当](assign-policies-users-and-groups.md#group-assignment-ranking) てられます。これは最上位のランク付けであることに注意してください。 教育者とスタッフ グループに同じポリシー領域の既存のポリシーが割り当てられている場合、その既存のポリシーは下位ランクに移動され、ウィザードによって割り当てられたポリシーが有効になります。

ユーザーに直接割り当てられているポリシーを削除する方法[について説明](batch-group-policy-assignment-edu.md#remove-a-policy-that-was-directly-assigned-to-users)します。

たとえば、教育者に会議ポリシーを直接割り当て、学生にグローバル (組織全体の既定) 会議ポリシーがあるとします。 このシナリオでは、ウィザードによって作成された会議ポリシーのカスタム ポリシー定義が有効になるように、教師に直接割り当てた会議ポリシーを削除します。 他の会議ポリシーと競合しないため、学生向けの既存のグローバル (組織全体の既定) 会議ポリシーで何もする必要はありません。

<a name="polwiz_addmeasures"> </a>

### <a name="step-2-check-for-additional-measures-that-you-can-take-for-student-safety"></a>手順 2: 学生の安全のために実行できるその他のメジャーを確認する

ウィザードは、 [これらのポリシー](#policies-applied-by-the-wizard)を自動的に調整して適用します。 安全を維持するために、教育機関のニーズに基づいて行う必要がある追加の措置がいくつかあります。

その他の安全に関する推奨事項については、「[Teamsを使用して遠隔学習を行う際に学生を安全に保つ](https://support.microsoft.com/office/keeping-students-safe-while-using-teams-for-distance-learning-f00fa399-0473-4d31-ab72-644c137e11c8#ID0EBBAAA)」を参照してください。

<a name="polwiz_mc"> </a>

### <a name="step-3-check-message-center-for-policy-updates"></a>手順 3: メッセージ センターでポリシーの更新プログラムを確認する

現在、ウィザードを実行すると、推奨されるポリシーが適用されます。 新しいポリシーがTeamsで使用できるようになると、学生の安全に関するグローバル (組織全体の既定) 設定がウィザードによって自動的に更新されることを知しておくことが重要です。

ただし、メッセージ [センター](https://admin.microsoft.com/AdminPortal/Home?#/MessageCenter) (Microsoft 365 管理センター) を頻繁に確認して、新機能とそのポリシーとポリシー設定をTeamsで最新の状態に保ちます。

## <a name="make-changes-in-the-wizard"></a>ウィザードで変更を加えます

<a name="polwiz_change"> </a>

ウィザードの実行後に変更を加える必要がある場合は、再度実行して選択内容を変更できます。

1. Microsoft Teams管理センターの左側のナビゲーションで、[**ホーム**] に移動し、[**安全な学習環境の簡易ポリシー設定**] タイルで [**変更**] を選択します。
2. ここから、ウィザードの各ページに進み、変更を加えます。 教育機関の種類、ポリシーを割り当てる教職員のグループ、またはその両方を変更できます。

次の表は、ウィザードで変更を加えた場合の動作をまとめたものです。

|変更の種類|ポリシーの動作|
|---|---|
|教育機関の種類と教育者とスタッフ グループの両方を変更する|<ul><li>**学生**: 新しい教育機関の種類に基づくグローバル (組織全体の既定) ポリシー定義が学生に適用されます。</li><li>**教育者とスタッフ**: 新しい教育機関の種類に基づく一連のカスタム ポリシー定義が作成され、新しい教育者およびスタッフ グループに割り当てられます。 以前のカスタム ポリシー定義は、以前の教育者およびスタッフ グループから削除されます。</li></ul>|
|教育機関の種類のみを変更する|<ul><li>**学生**: 新しい教育機関の種類に基づくグローバル (組織全体の既定) ポリシー定義が学生に適用されます。</li><li>**教育者とスタッフ**: 新しい教育機関の種類に基づく一連のカスタム ポリシー定義が作成され、教育者とスタッフ グループに割り当てられます。 以前の教育機関の種類に対して作成されたカスタム ポリシー定義は、教育者とスタッフ グループから削除されます。</li></ul>|
|教育者とスタッフ グループのみを変更する|<ul><li>**学生: 学生** に適用されるグローバル (組織全体の既定) ポリシー定義に変更はありません。</li><li>**教育者とスタッフ**: カスタム ポリシー定義は、新しい教育者とスタッフ グループに割り当てられ、以前の教育者およびスタッフ グループから削除されます。</li></ul>|

### <a name="policy-settings"></a>ポリシー設定

ポリシー領域ごとにウィザードによって適用される設定の概要を次に示します。

> [!NOTE]
> 共有チャネルを作成できるのは、チームの所有者のみです。<br><br>他の組織との共有チャネルでは、既定で無効になっている [Azure AD B2B 直接接続](/azure/active-directory/external-identities/b2b-direct-connect-overview) の構成が必要です。 この機能を有効にするには、「 [チャネル内の外部参加者と共同作業](/microsoft-365/solutions/collaborate-teams-direct-connect) する」を参照してください。
