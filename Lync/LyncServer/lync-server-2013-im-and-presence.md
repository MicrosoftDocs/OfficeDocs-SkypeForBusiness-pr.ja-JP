# Lync Server 2013 IM とプレゼンス

 

_**トピックの最終更新日:** 2013-10-07_

インスタント メッセージング (IM) とプレゼンス機能は、どの Lync Server 展開においても自動でインストールされます。

プレゼンス機能の情報によって、ユーザーは適切な時刻に適切な通信形態で同僚と連絡を取ることができ、これによってさらに生産的な作業環境を実現できます。ユーザーのプレゼンスとは、ユーザーの空き時間、応答の意思、追加情報 (場所、状態など)、および可能な連絡方法を含む一連の情報です。Lync Server では、写真、場所情報、一連の豊富なプレゼンス状態により、プレゼンス機能が強化されています。プレゼンス状態には "連絡可能"、"取り込み中"、"電話会議中" といった基本的な状態のほかに、"業務時間外"、"応答不可"、"一時退席中" があります。また、管理者はカスタマイズされた組織固有のプレゼンス状態を定義できます。

ユーザーは、連絡先の管理とユーザー アクセスのオプションを使って、他のユーザーが表示できる情報を制御できます。ユーザーは異なるレベルの連絡先を設定でき、それぞれの連絡先は異なるレベルのプレゼンス情報を表示できます。

連絡先リストを見るだけで、ユーザーは知る必要があるすべての情報を一目で確認できます。シンプルな色付きのアイコンは他のユーザーのプレゼンス状態を示し、写真や場所も表示されます。

Lync Server と Outlook、SharePoint など他の製品の統合により、電子メール メッセージまたはチーム Web サイトに連絡先の名前が表示されるたびに、その状態および連絡先情報も表示されます。さらに、Exchange 2013 を展開すると、Lync Server と Exchange 2013 が統合連絡先ストアを共有でき、両方の製品のクライアントがその統合連絡先ストアにアクセスできます。

Lync Server のインスタント メッセージングを使用し、ユーザーはすばやくメッセージを送り、タイムリーな情報をやり取りできます。必要に応じて、組織のユーザーは、MSN/Windows Live、Yahoo\!、AOL といったパブリック IM ネットワークのユーザーとも通信できます。ただし、Windows Live、AOL、Yahoo\! とのパブリック IM 接続には、個別のライセンスが必要な場合があります。また、Lync Server には Extensible Messaging and Presence Protocol (XMPP) との互換性があるため、組織のユーザーは、Google Talk などの XMPP サービスのユーザーと IM メッセージやプレゼンス情報を交換できます。

会話履歴では、古い IM 会話の経過をたどることができ、何か月も前に IM でやり取りした可能性のある情報を取得できます。

常設チャット機能を使用すると、トピックに基づく長時間のマルチパーティ会話に参加できます。チャット ルーム (ディスカッション フォーラム) に投稿されたメッセージは永続的 (すなわち、長期間使用可能) にできるので、異なる場所や部門に所属するユーザーが常に同時にオンラインでない場合でも会話に参加できます。

組織で法令上の規制に従う必要がある場合、メッセージ アーカイブ機能を展開し、組織内のすべてのユーザーまたは指定した一部のユーザーのみのインスタント メッセージの内容をアーカイブできます。Exchange 2013 も展開すると、IM のアーカイブを Exchange のインプレース保持機能と統合して、コンプライアンスのために単一の管理操作を提供できます。
