# Lync Server 2013 の通話転送と着信転送

 

_**トピックの最終更新日:** 2015-03-09_

PSTN エンドポイントを使用する場合、場所に基づくルーティングでは、発信者のエンドポイントと、通話または着信の転送先のエンドポイント (つまり通話転送/着信転送のターゲット) の場所が分析されます。場所に基づくルーティングでは、両方のエンドポイントの場所に基づいて、通話または着信を転送する必要があるかどうかが判断されます。

次の表は、PSTN エンドポイントを使用する通話での Lync ユーザーのシナリオを示しています。このシナリオでは Lync ユーザーが通話を別の Lync ユーザーに転送します。転送先エンドポイントのネットワーク サイトの場所によって、場所に基づくルーティングは通話転送または着信転送のルーティングに影響します。

例: Lync ユーザーが PSTN エンドポイントとの通話中に、同じネットワーク サイトに属する別の Lync ユーザーに通話を転送したとします。この場合、通話の転送が許可されます。

次の表は、別の Lync ユーザーと通話中の Lync ユーザーのシナリオです。一方のユーザーが通話を PSTN エンドポイントに転送した場合に、通話の転送先ユーザーの場所によって、場所に基づくルーティングが通話にどのように影響するかを表しています。
