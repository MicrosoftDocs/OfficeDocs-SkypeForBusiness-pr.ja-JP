# Lync Server 2013 でのポートの概要 - 自動検出

 

_**トピックの最終更新日:** 2016-04-06_

Lync Server 2013 自動検出サービスは、ディレクターおよびフロントエンド プールのサーバーで実行されます。`lyncdiscover.<domain>` および `lyncdiscoverinternal.<domain>` ホスト レコードを使用している DNS で発行されるとき、クライアントが Lync Server 機能を探す目的で使用できます。Lync Mobile を実行中のモバイル デバイスが Autodiscover を使用するには、最初に、自動検出サービスを実行中のフロント エンド サーバーおよびディレクター上で、証明書サブジェクト別名リストを変更する必要がある場合があります。また、リバース プロキシ上の外部 Web サービス公開ルールで使用される証明書についても、サブジェクトの別名リストの変更が必要になる場合があります。

リバース プロキシでサブジェクトの別名リストを使用するかどうかについては、自動検出サービスをポート 80 またはポート 443 のどちらで公開するかに基づいて決定します。

  - **ポート 80 で公開する**   モバイル デバイスでは、自動検出サービスへの最初のクエリをポート 80 で実行する場合、証明書を変更する必要はありません。これは、Lync を実行するモバイル デバイスがポート 80 のリバース プロキシに外部的にアクセスした後に、ポート 8080 のディレクターまたはフロント エンド サーバーに内部的にリダイレクトされるためです。

  - **ポート 443 で公開する**   外部 Web サービス公開ルールで使用される証明書上のサブジェクトの別名リストには、組織内の各 SIP ドメインの `lyncdiscover.<sipdomain>` エントリが含まれる必要があります。
    

    > [!IMPORTANT]
    > モビリティを展開した Lync Server 2010 から新しいインストールまたはアップグレードを行った場合、モビリティ サービスの Autodiscover でポート 80 を使用したか、適切なサブジェクト名およびサブジェクト別名を持つ証明書を再発行しています。ディレクターおよびフロント エンド サーバー上の証明書で、どちらを選択したか確認してください。
