# <a name="planning-for-enterprise-voice-resiliency-in-lync-server-2013"></a>Lync Server 2013 でのエンタープライズ Voip の復元の計画

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**トピックの最終更新日:** 2014-10-17_

音声の復元とは、WAN (ワイドエリアネットワーク) 障害またはその他の原因によって、Lync Server 2013 をホストするセントラルサイトが使用できなくなった場合に、ユーザーが通話を継続することができるかどうかを表します。 中央サイトで障害が発生した場合、エンタープライズ VoIP サービスでは、バックアップ サイトへの円滑なフェールオーバーにより中断せずにサービスを提供し続ける必要があります。 WAN 障害の場合は、ブランチ サイトの通話をローカルの PSTN ゲートウェイにリダイレクトする必要があります。 ここでは、中央サイトまたは WAN で障害が発生した場合の音声の復元の計画について説明します。
