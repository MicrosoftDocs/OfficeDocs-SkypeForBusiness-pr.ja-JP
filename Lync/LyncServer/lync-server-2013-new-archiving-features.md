# Lync Server 2013 のアーカイブの新機能

 

_**トピックの最終更新日:** 2016-12-08_

Lync Server 2013 でアーカイブを実行すると、次の種類のコンテンツをアーカイブすることができます。

  - ピアツーピアのインスタント メッセージ

  - マルチパーティのインスタント メッセージである電話会議 (ミーティング)

  - アップロードされたコンテンツ (配付資料など) およびイベント関連のコンテンツ (参加、退席、アップロード、共有、表示設定の変更など) を含む会議コンテンツ

また、 Lync Server 2013 のアーカイブには、展開と運用の効率を向上させる新機能があります。これらの新機能には、次のものがあります。

  - **フロントエンド サーバーでのアーカイブの併置。**    Lync Server 2013 には個別のアーカイブ サーバー役割はありません。アーカイブは、Enterprise Edition 展開のすべてのフロントエンド サーバーおよび Standard Edition サーバーにインストールできるオプションの機能であり、プールまたはサイトに対する構成として実装できます。

  - **Microsoft Exchange の統合。**   アーカイブを展開するときに、 Exchange 2013 に所属しておりメールボックスがインプレース保持になっているすべてのユーザーについて、アーカイブ用のデータ ストレージと既存の Exchange 2013 ストレージを統合できるため、 Lync データをアーカイブするために別個の SQL Server データベースを展開する必要がありません。 Exchange 2013 展開がない場合、統合したくない場合、または Exchange 2013 に所属していないがメールボックスがインプレース保持になっている Lync 2013 ユーザーがいる場合は、 SQL Server を使用して個々のアーカイブ データベースを展開し、 Lync との通信からアーカイブされたデータを格納することができます。展開内の一部のユーザーに対してだけ Microsoft Exchange 統合を使用したい場合は、 Microsoft Exchange 統合と Lync Server 2013 アーカイブ データベースの両方を使用できます。インプレース保持の詳細については、「インプレース保持」( [http://go.microsoft.com/fwlink/?linkid=267500\&clcid=0x411](http://go.microsoft.com/fwlink/?linkid=267500%26clcid=0x411)) を参照してください。

  - **SQL ストアのミラーリング。**   アーカイブを展開するときに、アーカイブ データベース用の SQL Server データベースのミラーリングを有効にできます。

  - **ホワイトボードとポーリングのアーカイブ。**   アーカイブする会議コンテンツに、会議中に共有されるホワイトボードとポーリングを含めることができるようになりました。

次の種類のコンテンツはアーカイブされません。

  - ピアツーピアのファイル転送

  - ピアツーピアのインスタント メッセージおよび会議の音声ビデオ

  - ピアツーピア インスタント メッセージおよび電話会議のアプリケーション共有
