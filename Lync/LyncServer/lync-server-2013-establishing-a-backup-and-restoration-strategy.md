# バックアップと復元の戦略の確立

 

_**トピックの最終更新日:** 2013-03-26_

Lync Server のバックアップと復元の計画を作成する前に、組織の目標に合った戦略を確立する必要があります。効果的なバックアップと復元の戦略を確立するには以下が必要です。

  - ビジネスの優先順位の確立

  - バックアップと復元の要件の特定

## ビジネスの優先順位の確立

組織のビジネスの優先順位を評価します。一般に、ビジネスの主な優先順位のうちで、バックアップと復元の戦略に関係するものには以下があります。

  - 業務継続の要件

  - データの完全性

  - データの重要性

  - ポータビリティの要件

  - コストの制約

このようなビジネス要件に応じて、顧客との間で作成するサービス レベル アグリーメント (SLA) は決まります。サービス レベル アグリーメントはバックアップと復旧の戦略に大きな影響を及ぼします。

## バックアップと復元の要件の特定

ビジネスの優先順位とサービス レベル アグリーメントに応じて、Lync Server のバックアップと復元に関する組織の要件が決まります。以下についての要件を特定し、文書化します。

  - **バックアップの頻度**   「[Lync Server 2013 での高可用性および障害復旧の計画](lync-server-2013-planning-for-high-availability-and-disaster-recovery.md)」に記載されているようにペアの関係にあるフロント エンド プールを除き、Lync Server は単純復旧モデルのみをサポートしていることに注意する必要があります。つまり、最後の完全バックアップの状態に復元することになります。完全バックアップを実行することが必要な頻度を綿密に計画します。バックアップの頻度に関するベスト プラクティスの詳細については、「[バックアップと復元のベスト プラクティス](lync-server-2013-best-practices-for-backup-and-restoration.md)」を参照してください。

  - **バックアップと復元のツール**   どのユーザーがどのコンピューターでツールを使用するかという点も対象になります。このトピックで説明するツールと必要な権限の詳細については、「[バックアップと復元の要件: ツールとアクセス許可](lync-server-2013-backup-and-restoration-requirements-tools-and-permissions.md)」を参照してください。

  - **バックアップの場所**   セキュリティとアクセスのしやすさを考慮に入れて、バックアップをローカルとリモートのどちらで保持するかを特定します。バックアップに使用するメディアを指定します。

  - **ハードウェアとソフトウェアの要件**   ハードウェアとソフトウェアの具体的な要件を特定し、文書化します。たとえば、特定のコンポーネントのバックアップ ストレージと復元に使用するハードウェア、バックアップと復元をサポートするために必要なソフトウェアとネットワーク接続などです。ハードウェアとソフトウェアの要件を作成するときには、次に説明するさまざまな復元のシナリオに注意してください。

  - **復元のシナリオ**   ここでは、以下のシナリオの復元のプロセスを説明しています。
    
      - Lync Server プールに障害が発生した場合。このシナリオでは、プール内の各サーバーを構築し直す必要があります。
    
      - Standard Edition サーバーに障害が発生した場合。このシナリオでは、新しいコンピューターまたはクリーンな状態のコンピューターにサーバーを構築し直してデータベースを復元する必要があります。
    
      - 中央管理ストアが失われた場合。このシナリオでは、少なくとも中央管理ストアを復元および公開する必要があります。
    
      - 中央管理ストアが正常に稼働している状態でバックエンド サーバーが失われた場合。このシナリオでは、新しいコンピューターまたはクリーンな状態のコンピューターにサーバーを構築し直してデータベースを復元する必要があります。
    
      - Lync Server プールのメンバーのサーバーに障害が発生した場合。このシナリオでは、新しいコンピューターまたはクリーンな状態のコンピューターにサーバーを構築し直す必要があります。
    
      - ファイル ストアで障害が発生した場合。このシナリオでは、ファイル サーバーまたはファイル クラスターを復元する必要があります。
    
      - アーカイブ データベース、監視データベース、または常設チャット データベースに障害が発生した場合。このシナリオではデータベースを作成し直す必要があります。組織にとって重要なデータの場合にはデータの復元も必要です。アーカイブ、監視、および常設チャットのデータは、Lync Server のバックアップと実行には必要ありません。
