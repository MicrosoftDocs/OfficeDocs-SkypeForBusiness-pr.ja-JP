# Lync Server 2013 メディア バイパスの技術要件

 

_**トピックの最終更新日:** 2012-09-21_

PSTN へのそれぞれの通話に対し、仲介サーバーは、発信元の Lync エンドポイントからのメディアを、仲介サーバーを通過させずに直接 仲介サーバー ピアへ送信できるかどうかを判断します。ピアには、通話がルーティングされる仲介サーバー間のトランクに関連付けられた PSTN ゲートウェイ、IP-PBX、またはインターネット テレフォニー サービス プロバイダー (ITSP) のセッション ボーダー コントローラー (SBC) を使用できます。

メディア バイパスを使用できるのは、次の要件を満たす場合です。

  - 仲介サーバー ピアは、メディア バイパスに必要な機能をサポートする必要があります。最も重要なのは、複数の分岐応答 ("早期ダイアログ" として知られています) を処理する能力です。ゲートウェイ、PBX、または SBC が受け入れることができる最大早期ダイアログ数については、ゲートウェイ、PBX、または ITSP の製造元に問い合わせてください。

  - 仲介サーバー ピアは、Lync エンドポイントから直接メディア トラフィックを受け入れる必要があります。多くの ITSP は、仲介サーバーからのトラフィックの受信のみを SBC に許可しています。SBC が Lync エンドポイントから直接メディア トラフィックを受け入れるかどうかについては、ITSP に問い合わせてください。

  - Lync クライアントおよび 仲介サーバー ピアの接続は安定している必要があります。つまり、同じネットワーク領域に配置されているか、または帯域幅の制限がない WAN リンクを介して領域に接続するネットワーク サイトに配置されている必要があります。
