# 付録 B: Lync Server 2013 での存続可能ブランチ アプライアンスの管理

 

_**トピックの最終更新日:** 2012-10-18_

このトピックでは、存続可能ブランチ アプライアンスの管理手順について説明します。具体的には、存続可能ブランチ アプライアンスの置換方法、名前変更の方法、および 存続可能ブランチ アプライアンスが関連付けられている Lync Server 2013 フロントエンド プールの変更方法について説明します。

## 存続可能ブランチ アプライアンスを置き換えるには

1.  存続可能ブランチ アプライアンスですべての Lync Server 2013 サービスを停止します (存続可能ブランチ アプライアンス ベンダーのドキュメントを参照してください)。

2.  (推奨) ドメインから 存続可能ブランチ アプライアンスを削除します。

3.  次の手順に従って、存続可能ブランチ アプライアンス 内の Active Directory ドメイン サービス コンピューター オブジェクトを削除します。
    
      - Enterprise Admins グループのメンバーとしてメンバー サーバーにログオンします。
    
      - \[**スタート**\]、\[**管理ツール**\]、\[**Active Directory ユーザーとコンピューター**\] の順にクリックします。
    
      - 存続可能ブランチ アプライアンス オブジェクトを右クリックし、\[**削除**\] をクリックします。

4.  存続可能ブランチ アプライアンス コンピューター オブジェクトを再度追加します (「[Lync Server 2013 での、Active Directory への存続可能ブランチ アプライアンスの追加](lync-server-2013-add-a-survivable-branch-appliance-to-active-directory.md)」を参照してください)。

5.  Active Directory のレプリケーションが実行されるまで待機します。

6.  Lync Server 管理シェルを開き、「**Enable-CSTopology**」と入力します。

7.  新しい 存続可能ブランチ アプライアンスをネットワークに接続し、「[Lync Server 2013 による存続可能ブランチ アプライアンスまたはサーバーの展開 - 中央サイトのタスク](lync-server-2013-deploying-a-survivable-branch-appliance-or-server-central-site-tasks.md)」および「[Lync Server 2013 での存続可能ブランチ アプライアンスまたは存続可能ブランチ サーバーの展開 - ブランチ サイトのタスク](lync-server-2013-deploy-a-survivable-branch-appliance-or-server-branch-site-task.md)」の手順に従います。

## 存続可能ブランチ アプライアンスの名前を変更するには

1.  ユーザーをセントラル サイトに移動します。詳細については、「[別のプールへのユーザーの移動](lync-server-2013-move-users-to-another-pool.md)」を参照してください。

2.  存続可能ブランチ アプライアンスですべての Lync Server 2013 サービスを停止します (存続可能ブランチ アプライアンス ベンダーのドキュメントを参照してください)。

3.  トポロジから 存続可能ブランチ アプライアンスを削除し、次の手順に従います。
    
      - \[**スタート**\]、\[**すべてのプログラム**\]、\[**Microsoft Lync Server**\]、\[**Lync Server トポロジ ビルダー**\] の順にクリックします。
    
      - コンソール ツリーで \[**ブランチ サイト**\] を展開し、存続可能ブランチ アプライアンスをクリックして、操作ウィンドウの \[**削除**\] をクリックします。

4.  メインから 存続可能ブランチ アプライアンスを削除します。

5.  次の手順に従って、Active Directory 内の 存続可能ブランチ アプライアンス コンピューター オブジェクトを削除します。
    
      - Enterprise Admins グループのメンバーとしてドメイン コントローラーにログオンします。
    
      - \[**スタート**\]、\[**管理ツール**\]、\[**Active Directory ユーザーとコンピューター**\] の順にクリックします。
    
      - 存続可能ブランチ アプライアンス オブジェクトを右クリックし、\[**削除**\] をクリックします。

6.  存続可能ブランチ アプライアンスを工場出荷時の既定値に戻します (存続可能ブランチ アプライアンスのベンダーによるドキュメントを参照してください)。

7.  「[Lync Server 2013 による存続可能ブランチ アプライアンスまたはサーバーの展開 - 中央サイトのタスク](lync-server-2013-deploying-a-survivable-branch-appliance-or-server-central-site-tasks.md)」および「[Lync Server 2013 での存続可能ブランチ アプライアンスまたは存続可能ブランチ サーバーの展開 - ブランチ サイトのタスク](lync-server-2013-deploy-a-survivable-branch-appliance-or-server-branch-site-task.md)」の手順に従います。

8.  名前を変更した 存続可能ブランチ アプライアンスにユーザーを移動します。詳細については、「[別のプールへのユーザーの移動](lync-server-2013-move-users-to-another-pool.md)」を参照してください。

## 存続可能ブランチ アプライアンスが関連付けられている Lync Server のフロントエンド プールを変更するには

1.  存続可能ブランチ アプライアンスからセントラル サイトの Lync Server フロントエンド プールにユーザーを移動します。詳細については、「[別のプールへのユーザーの移動](lync-server-2013-move-users-to-another-pool.md)」を参照してください。

2.  存続可能ブランチ アプライアンスですべての Lync Server サービスを停止します (存続可能ブランチ アプライアンス ベンダーのドキュメントを参照してください)。

3.  存続可能ブランチ アプライアンス が関連付けられている Lync Server のフロントエンド プールを更新するには、次の手順に従ってください。
    
      - \[**スタート**\]、\[**すべてのプログラム**\]、\[**Microsoft Lync Server**\]、\[**Lync Server トポロジ ビルダー**\] の順にクリックします。
    
      - \[**ブランチ サイト**\] を展開します。
    
      - 変更する 存続可能ブランチ アプライアンス オブジェクトを右クリックして、\[**プロパティの編集**\] をクリックします。
    
      - \[復元\] の下で、存続可能ブランチ アプライアンスを関連付ける新しい フロント エンド プールを選択して、\[**次へ**\] をクリックします。
    
      - コンソール ツリーで新しい 存続可能ブランチ アプライアンスを右クリックし、\[**トポロジ**\] をクリックしてから \[**公開**\] をクリックします。

4.  存続可能ブランチ アプライアンスですべての Lync Server サービスを再開します。

5.  存続可能ブランチ アプライアンスをテストします。詳細については、「[Lync Server 2013 で存続可能ブランチ アプライアンスまたは存続可能ブランチ サーバーにユーザーを所属させる](lync-server-2013-home-users-on-a-survivable-branch-appliance-or-server.md)」を参照してください。

6.  セントラル サイトの Lync Server のフロントエンド プールから 存続可能ブランチ アプライアンスにユーザーを移動します。
