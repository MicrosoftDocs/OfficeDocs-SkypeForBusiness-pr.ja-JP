# <a name="in-lync-server-2013-add-kerberos-authentication-to-other-sites"></a>Lync Server 2013 の他のサイトへの Kerberos 認証の追加

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**トピックの最終更新日:** 2010-11-03_

Kerberos 認証用に構成した既存の展開にサイトを追加する必要がある場合は、新しいサイト用に既存の Kerberos 認証アカウントを使用するか、新しいアカウントを作成できます。 サイトの新しいアカウントを作成する方法の詳細については、「 [Create a Kerberos authentication account In Lync Server 2013](lync-server-2013-create-a-kerberos-authentication-account.md)」を参照してください。 既存のアカウントをサイトに割り当てることによって、新しいサイトに同じアカウントを使用する方法の詳細については、「 [Assign a Kerberos 認証アカウントを Lync Server 2013 のサイトに割り当てる](lync-server-2013-assign-a-kerberos-authentication-account-to-a-site.md)」を参照してください。
