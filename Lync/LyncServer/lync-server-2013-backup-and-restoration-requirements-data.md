# Lync Server 2013 でのバックアップと復元の要件: データ

 

_**トピックの最終更新日:** 2016-12-08_

Lync Server では、データベースに格納されている設定と構成情報、およびデータベースとファイル ストアに格納されているデータを使用します。このトピックでは、組織で障害や停止が発生した場合にサービスを復元できるようにバックアップしておく必要があるデータについて説明します。また、Lync Server が使用するデータとコンポーネントのうちで、別個にバックアップする必要があるものも示します。

## 設定と構成の要件

このトピックでは、Lync Server サービスを復旧するために必要な設定と構成情報をバックアップおよび復元する手順について説明しています。この構成情報は、中央管理ストア、または別のバックエンド データベースか Standard Edition サーバーにあります。

## データの要件

以下に、障害が発生した場合に Lync Server サービスを復元できるようにバックアップしておく必要がある Lync Server データをまとめます。

一部の種類のデータは復元に必要ないため、このトピックでは、次のようなこうした種類のデータのバックアップ手順は説明していません。

  - エンドポイントとサブスクリプション、アクティブな会議サーバー、一時的な会議状態など、一時的なユーザー データ (データベース: RtcDyn.mdf)

  - アドレス帳のデータ (データベース: Rtcab.mdf および Rtcab1.mdf)。アドレス帳のデータベースは Active Directory ドメイン サービス から自動的に再生成されます。

  - コール パーク アプリケーション用の動的な情報 (データベース: CpsDyn.mdf)

  - エージェントのサインイン状態、通話待機の情報など、一時的な応答グループ データ (データベース: RgsDyn.mdf)

  - 常設チャットのコンプライアンス データベース (データベース: MgcComp.mdf)。常設チャットのコンプライアンスを有効にしている場合は、データベースから情報を読み取って別の形式に変換するように構成されているアダプターがある限り、常設チャットのコンプライアンス データベース内の情報は一時的です。したがって、常設チャットのコンプライアンス データベースは一時的と見なされます。
    
    Lync Server 2013 常設チャット サーバーには XML アダプターが付属しています。また、このデータを取得して、Exchange Hosted Archive など、他のソースに移行するカスタム アダプターをインストールすることもできます。

## ファイル ストア データの要件

Enterprise Edition の展開では、Lync Server ファイル ストアは、通常、ファイル サーバーに配置されます。Standard Edition の展開では、Lync Server ファイル ストアは既定では Standard Edition サーバーに配置されます。通常、サイト向けに共有されている Lync Server ファイル ストアが 1 つあります。常設チャット ファイル ストアは、Lync Server ファイル ストアと同じファイル共有を使用しています。

## 追加のバックアップの要件

障害が発生した時点の状態に Lync Server サービスを復元できるようにするには、Lync Server そのものには含まれていない必要なコンポーネントをバックアップしておく必要があります。次のコンポーネントは、このドキュメントで説明する Lync Server のバックアップと復元の手順ではバックアップおよび復元されません。

  - **Active Directory ドメイン サービス**   Lync Server のバックアップと同じタイミングで、Active Directory のツールを使用して AD DS をバックアップする必要があります。AD DS を Lync Server と同期しておくことが重要です。連絡先オブジェクトが AD DS のものと一致しないと Lync Server が見なした場合に発生する問題を回避するためです。AD DS には、Lync Server で使用される次の設定が格納されます。
    
      - ユーザーの SIP URI と他のユーザー設定。
    
      - 応答グループ、会議アテンダントなど、アプリケーションの連絡先オブジェクト。
    
      - 中央管理ストアへのポインター。
    
      - Kerberos 認証のアカウント (オプションのコンピューター オブジェクト) および Lync Server のセキュリティ グループ。
    
    Windows Server 2008 での AD DS のバックアップと復元の詳細については、「ステップ バイ ステップ ガイド - AD DS のバックアップと回復」(<http://go.microsoft.com/fwlink/?linkid=209105>) を参照してください。

  - **証明機関と証明書**   証明機関 (CA) と証明書のバックアップに関する組織のポリシーを使用します。エクスポート可能な秘密キーを使用している場合は、証明書と秘密キーをバックアップでき、このドキュメントの手順を使用して Lync Server を復元する場合にエクスポートできます。内部 CA を使用している場合は、Lync Server の復元が必要なときには再登録できます。秘密キーは、コンピューターで障害が発生した場合に使用できる安全な場所に保持しておくことが重要です。

  - **System Center Operations Manager**   Microsoft System Center Operations Manager (以前の Microsoft Operations Manager) を使用して Lync Server の展開を監視している場合、Lync Server の監視中に作成されるデータをバックアップすることもできます。SQL Server の標準的なバックアップ手順に従って System Center Operations Manager のファイルをバックアップします。これらのファイルは復旧の際に復元されません。

  - **公衆交換電話網 (PSTN) ゲートウェイの構成**   エンタープライズ VoIP または存続可能ブランチ アプライアンスを使用している場合には、PSTN ゲートウェイの構成をバックアップする必要があります。PSTN ゲートウェイ構成のバックアップと復元の詳細については、ベンダーに確認してください。

  - **Lync Server または Office Communications Server のバージョンとの共存**   Lync Server 2013 展開が Lync Server 2010 または Office Communications Server の以前のバージョンと共存する場合、このドキュメントの手順で以前のバージョンのバックアップと復元は行えません。代わりに、該当のバージョンのドキュメントにあるバックアップと復元の手順に従う必要があります。Lync Server 2010 のバックアップと復元の詳細については、<http://go.microsoft.com/fwlink/?linkid=265417> を参照してください。Microsoft Office Communications Server 2007 R2 のバックアップと復元の詳細については、<http://go.microsoft.com/fwlink/?linkid=168162> を参照してください。

  - **インフラストラクチャの情報**   ファイアウォールの構成、負荷分散の構成、インターネット インフォメーション サービス (IIS) の構成、ドメイン ネーム システム (DNS) のレコードと IP アドレス、動的ホスト構成プロトコル (DHCP) の構成など、インフラストラクチャについての情報をバックアップする必要があります。これらのコンポーネントのバックアップの詳細については、各ベンダーに確認してください。

  - **Microsoft Exchange および Exchange ユニファイド メッセージング (UM)**   Microsoft Exchange のドキュメントの説明に従って、Microsoft Exchange および Exchange UM のバックアップと復元を行います。Exchange Server 2013 のバックアップと復元の詳細については、<http://go.microsoft.com/fwlink/?linkid=285384> を参照してください。Exchange Server 2010 のバックアップと復元の詳細については、<http://go.microsoft.com/fwlink/?linkid=209179> を参照してください。
    
    Lync Server 2013 では、ユーザーの連絡先の一覧、高解像度のユーザーの写真、およびアーカイブ データを Exchange 2013 に保存しておくことができるようになりました。これらの種類のデータをバックアップする方法については、以下の一覧を参照してください。
    
      - **高解像度の写真**は Exchange Server バックアップの一環としてバックアップされます。
    
      - Lync Server 2013 には**統合連絡先ストア**が導入されています。統合連絡先ストアにより、ユーザーはすべての連絡先情報を Exchange 2013 に保存できます。
        
        ユーザーの連絡先を統合連絡先ストアに保存するか、Lync バックエンド サーバーに保存するかにかかわらず、バックアップがユーザーにとって確実に最新となるようにする必要があります。次のシナリオで、ユーザーの連絡先を統合連絡先ストアに移行するとバックアップと復元の処理で問題が発生する可能性があることを示します。
        
        **シナリオ 1:** ユーザーの連絡先が統合連絡先ストアに移行され、ユーザーの連絡先を移行する前に取得された Lync Server のバックアップから復元が実行されます。このシナリオでは、Lync Server の移行タスクによりユーザーの連絡先を Exchange へ移行する処理が開始されるまで、ユーザーの連絡先の状態は最大 1 日古くなります (ユーザーの連絡先は事前に統合連絡先ストアに移行されたので、Exchange の連絡先情報が使用されます)。このシナリオでは、管理者の介入は必要ありません。
        
        **シナリオ 2:** ユーザーの連絡先は事前に統合連絡先ストアに保存されましたが、ロールバックされました。ユーザーの連絡先が統合連絡先ストアに保存されたときに取得された Lync Server のバックアップから復元が実行されます。このシナリオでは、クライアントまたは Lyss サーバー ログ内の `Error: Incorrect Exchange Version` というエラー メッセージにより、これは問題であると見なされることがあります。ユーザーは Exchange から Lync 2013 内の連絡先の一覧に直接アクセスできますが、クライアントの状態は Lync Server の状態と一致しません。この問題を解決するには、管理者が、影響を受けるユーザーに対して **Invoke-CsUCSRollback** コマンドレットを実行する必要があります。
    
      - **アーカイブ データ**は Exchange 2013 に保存できます。このデータは Lync Server にとっては重大ではありませんが、規制上組織にとっては重大になることがあります。アーカイブ データが Exchange に保存され、組織にとって重大である場合は、Exchange のバックアップと復元の手順を実行してください。Exchange に保存されているバックアップ データを Lync Server に戻すことはできません。また、Lync アーカイブ データベースに既に保存されているデータを Exchange に移行することもできません。
