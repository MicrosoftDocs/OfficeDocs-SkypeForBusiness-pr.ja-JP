# ポートの概要 - Lync Server 2013 のリバース プロキシ

 

_**トピックの最終更新日:** 2015-03-09_

リバース プロキシには、ファイアウォールとポート/プロトコルに関して最小限の要件があります。

  - 外部ファイアウォールの要件は HTTPS/TCP/443 と HTTP/TCP/80 (オプション) です。HTTPS は、リバース プロキシ経由のセキュリティで保護された SSL および TLS 通信で使用されます。HTTP は、証明書を変更するのが困難なとき、またはコスト的に見合わないときに、自動検出サービスにアクセスできるようにする場合に使用されます。

  - クライアントは HTTPS で Office Web Apps サーバー に接続します。Office Web Apps サーバー は、HTTPS/TCP/443 の内部クライアントからの通信を想定します。推奨構成として、リバース プロキシから Office Web Apps サーバー への HTTPS/TCP/443 を許可します。

  - ポート 8080 は、リバース プロキシの内部インターフェイスから フロント エンド サーバー、フロント エンド プールの仮想 IP (VIP)、またはオプションの ディレクターまたは ディレクター プールの VIP にトラフィックをルーティングするために使用されます。外部 Web サービス公開ルールの証明書の変更が望ましくない場合には (多数の SIP ドメインがある場合など)、Lync を実行するモバイル デバイスが自動検出サービスを検索するために、TCP ポート 8080 が必須となります。必要な SAN エントリを含む新しい証明書を取得する場合は、TCP ポート 8080 は不要であるため、省略可能です。

  - ポート 4443。リバース プロキシの内部インターフェイスから フロント エンド サーバー、フロント エンド プールの仮想 IP (VIP)、またはオプションの ディレクターまたは ディレクター プール VIP へのトラフィックに使用されます。
    
    ![リバース プロキシおよび外部 Web サービス](images/JJ204932.13142405-d5c9-45b7-a8b7-a8c89f09c97c(OCS.15).jpg "リバース プロキシおよび外部  Web サービス")  
    

    > [!TIP]
    > リバース プロキシから内部展開への TCP 4443 と、Standard Edition サーバーまたは 中央管理ストアの役割を管理する フロント エンド プールからの TCP トラフィックのポート 4443 を混同しないでください。
