# Lync Server 2013 での自動検出サービスの DNS レコードの作成

 

_**トピックの最終更新日:** 2014-06-20_

Lync Mobile ユーザーは自動検出を有効にすることが必要です。その一部として、ドメイン ネーム システム (DNS) レコードの作成が含まれます。ニーズによって以下が必要です。

  - 組織のネットワーク内から接続するモバイル ユーザーをサポートするための内部 DNS レコード。

  - インターネットから接続するモバイル ユーザーをサポートするための外部 (パブリック) DNS レコード。

両方が必要な理由は、内部 DNS レコードと外部 DNS レコードは SIP ドメインごとに作成する必要があるためです。

作成する DNS レコードには A (ホスト) レコードまたは CNAME レコードを使用できます。ご参考までに、これらの内部および外部の DNS レコードの作成方法を以下で説明します。モバイル ユーザーの DNS 要件の詳細については、「[Lync Server 2013 でのモビリティの技術要件](lync-server-2013-technical-requirements-for-mobility.md)」を参照してください。

## 内部 DNS CNAME レコードの作成

1.  内部 DNS レコードを作成するには、Domain Admins グループか DnsAdmins グループのメンバーであるドメイン アカウントで、ネットワーク内の DNS サーバーにログオンする必要があります。

2.  DNS 管理スナップインを開きます。\[**スタート**\] ボタンをクリックし、\[**管理ツール**\]、\[**DNS**\] の順にクリックします。

3.  DNS サーバーのコンソール ツリーで、Lync Server 2013 のディレクター プールとフロント エンド プール がインストールされている Active Directory ドメイン (contoso.local など) の \[**前方参照ゾーン**\] を展開します。

4.  内部 DNS レコードで、ディレクター プールのホスト A (IPv6 の場合は AAAA) レコードが存在するかどうかを確認します。ホスト A レコードは、ディレクター プールの内部 Web サービスの完全修飾ドメイン名 (FQDN) (lyncwebdir01.contoso.local など) には必ず存在します。ホスト A レコードが存在しない場合は、ディレクター プールを使用していない可能性があり、フロント エンド プールの FQDN を使用するか、設定によっては単一サーバーの FQDN を使用する必要があります。

5.  内部 DNS レコードのフロント エンド プールのホスト A (IPv6 の場合は AAAA) レコードがあるかどうかを必ず確認してください。ホスト A (または AAAA) レコードは、フロント エンド プールの内部 Web サービスの FQDN (lyncwebpool01.contoso.local など) には必ず存在します。存在しない場合は、フロント エンド サーバーまたは Standard Edition サーバーの FQDN をメモしておく必要があります。

6.  DNS サーバーのコンソール ツリーに存在するホスト A (または AAAA) レコードを確認したら、SIP ドメイン (contoso.com など) の \[**前方参照ゾーン**\] を展開します。

7.  SIP ドメイン名を右クリックして、\[**新しいエイリアス (CNAME)**\] をクリックします。

8.  \[**エイリアス名**\] に、内部自動検出サービス URL のホスト名として、「lyncdiscoverinternal」と入力します。

9.  \[**ターゲット ホスト用の完全修飾ドメイン名 (FQDN)**\] で、ディレクター プールの内部 Web サービスの FQDN (lyncwebdir01.contoso.local など) を入力または参照し、\[**OK**\] をクリックします。

10. Lync Server 2013 環境でサポートする各 SIP ドメインの前方参照ゾーンで新しい自動検出 CNAME レコードを作成する必要があります。

## 外部 DNS CNAME レコードの作成

1.  外部 DNS CNAME レコードを作成するには、パブリック DNS プロバイダーに接続してから、当社の指定する手順に従う必要があります。外部 DNS の管理にはさまざまな方法があるため、ご使用の DNS プロバイダーの環境について詳しく説明することはできませんが、以下の手順を参照してください。

2.  外部 DNS プロバイダーに、その環境で DNS レコードを作成できるアカウントでログインします。

3.  この環境用の SIP ドメインを作成済みである必要があります。使用する外部 DNS インターフェイスによって、この SIP ドメインの \[**前方参照ゾーン**\] を展開または選択します。

4.  表示されるディレクター プールのホスト A (IPv6 の場合は AAAA) レコード (lyncwebexdir01.contoso.com など) を確認します。表示されない場合は、ディレクター プールを使用していない可能性があります。この場合は、フロント エンド プールの FQDN を使用する必要があります。単一サーバー、フロントエンド サーバー、Standard Edition サーバーの場合も同様です。

5.  フロント エンド プールの外部 Web サービスの完全修飾ドメイン名 (FQDN) (lyncwebextpool01.contoso.com など) のホスト A (IPv6 の場合は AAAA) レコードが存在するか、フロント エンド プールがない場合は単一サーバー FQDN の FQDN が存在することも確認する必要があります。前の手順で説明したように、ディレクター プールがない場合は、これが以下で必要です。

6.  次に、外部 DNS プロバイダーの形式に従って、**新規エイリアス (CNAME)** を作成するオプションを選択します (DNS プロバイダーの形式によって、メニュー オプションやリンクの場合があります)。

7.  内部 DNS と同様に、**エイリアス名**のテキスト ボックスにはいくつかのフォームがあります。外部自動検出サービス URL のホスト名として、「lyncdiscover」と入力します。

8.  **ターゲット ホストの完全修飾ドメイン名 (FQDN)** のテキスト ボックスにもいくつかのフォームがあります。ここに、ディレクター プールの外部 Web サービスの FQDN (lyncwebexdir01.contoso.com など) を入力して \[OK\] をクリックするか、外部 DNS で何らかの操作を実行してこのエントリの作成を許可します。上記の手順 4. で説明したとおり、ディレクター プールがない場合は、必要に応じて、設定済みのフロント エンド プールの FQDN か単一サーバーの FQDN を使用する必要があります。

9.  Lync 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出の CNAME レコードを作成する必要があります。

## 内部 DNS A レコードの作成

1.  内部 DNS レコードを作成するには、Domain Admins グループか DnsAdmins グループのメンバーであるドメイン アカウントで、ネットワーク内の DNS サーバーにログオンします。

2.  DNS 管理スナップインを開きます。\[**スタート**\] ボタンをクリックし、\[**管理ツール**\]、\[**DNS**\] の順にクリックします。

3.  内部 DNS レコードの場合、DNS サーバーのコンソール ツリーで、Lync Server 2013 のディレクター プールとフロント エンド プールがインストールされている Active Directory ドメイン (contoso.local など) の \[**前方参照ゾーン**\] を展開します。

4.  内部 DNS レコードで、ディレクター プールのホスト A (IPv6 の場合は AAAA) レコードが存在するかどうかを確認します。ホスト A レコードは、ディレクター プールの内部 Web サービスの完全修飾ドメイン名 (FQDN) (lyncwebdir01.contoso.local など) には必ず存在します。ホスト A レコードが存在しない場合は、ディレクター プールを使用していない可能性があり、フロント エンド プールの IP アドレスを使用するか、設定によっては単一サーバーの IP アドレスを使用する必要があります。

5.  内部 DNS レコードで、フロント エンド プールのホスト A (IPv6 の場合は AAAA) レコードがあるかどうかを必ず確認してください。ホスト A (または AAAA) レコードは、フロント エンド プールの内部 Web サービスの FQDN (lyncwebpool01.contoso.local など) には必ず存在します。存在しない場合は、フロント エンド サーバーまたは Standard Edition サーバーの IP アドレスをメモしておく必要があります。

6.  DNS サーバーのコンソール ツリーに存在するホスト A (または AAAA) レコードを確認したら、SIP ドメイン (contoso.com など) の \[**前方参照ゾーン**\] を展開します。

7.  SIP ドメイン名を右クリックして、\[**新しいホスト (A または AAAA)**\] をクリックします。

8.  \[**名前**\] に、内部自動検出サービス URL のホスト名として、「lyncdiscoverinternal」と入力します。

9.  \[**IP アドレス**\] に、ディレクターの内部 Web サービスの IP アドレスを入力します (または、ロード バランサーを使用する場合は、ディレクター ロード バランサーの仮想 IP (VIP) を入力します)。 上記の手順 4. で説明したとおり、ディレクターを使用していない場合は、フロント エンド サーバーか Standard Edition サーバーの IP アドレスの入力が必要な場合があります。また、ロード バランサーを使用する場合は、フロント エンド プール ロード バランサーの VIP を入力します。

10. \[**ホストの追加**\] をクリックし、\[**OK**\] をクリックします。

11. 追加の A レコードまたは AAAA レコードを作成するには、手順 8. 〜 10. を繰り返します。Lync Server 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出の A レコードまたは AAAA レコードを作成する必要があるので注意してください。

12. A (IPv6 では AAAA) レコードの作成が完了した後で、\[**完了**\] をクリックします。

## 外部 DNS A レコードの作成

1.  外部 DNS レコードを作成するには、パブリック DNS プロバイダーに接続してから、当社の指定する手順に従う必要があります。外部 DNS の管理にはさまざまな方法があるため、ご使用の DNS プロバイダーの環境について詳しく説明することはできませんが、以下の手順を参照してください。

2.  ご使用の環境で DNS レコードを作成できるアカウントでログインする必要があります。

3.  外部 DNS レコードの場合、DNS サーバーのコンソール ツリーで、使用する SIP ドメイン (contoso.com など) の \[**前方参照ゾーン**\] を展開します。

4.  ディレクター プールのホスト A (IPv6 の場合は AAAA) レコード (lyncwebexdir01.contoso.com など) が表示されるので、この表示と IP アドレスを確認します。表示されない場合は、ディレクター プールを使用していない可能性があります。この場合は、フロント エンド プールの IP アドレスを使用する必要があります。単一サーバー、フロントエンド サーバー、Standard Edition サーバーの場合も同様です。サーバーがロード バランサーの背後にあったり、リバース プロキシを使用していたりする可能性も考慮する必要があります。以下の手順を実行するため、IP アドレスもメモしておきます。

5.  フロント エンド プールの外部 Web サービスの完全修飾ドメイン名 (FQDN) (lyncwebextpool01.contoso.com など) のホスト A (IPv6 の場合は AAAA) レコード (フロント エンド プールがない場合は単一サーバーの Lync インストールの IP アドレス) が存在することも確認する必要があります。前の手順で説明したように、ディレクター プールがない場合は、これが以下で必要です。

6.  次に、外部 DNS プロバイダーの形式に従って、**新規ホスト A または AAAA** を作成するオプションを選択します (DNS プロバイダーの形式によって、メニュー オプションやリンクの場合があります)。

7.  **名前**を入力する場所に、外部自動検出サービス URL のホスト名として「lyncdiscover」と入力します。

8.  また、**IP アドレス** テキスト ボックスに ディレクター プール (lyncwebexdir01.contoso.com など) の IP かプールのロード バランサーの IP (または結果が同じリバース プロキシ IP) を入力して \[OK\] をクリックするか、外部 DNS で何らかの操作を実行してこのエントリの作成を許可します。上記の手順 4. で説明したとおり、ディレクター プールがない場合は、必要に応じて、設定済みのフロント エンド プールの IP アドレスか単一サーバーの IP アドレスを使用する必要があります。

9.  Lync 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出の A レコードまたは AAAA レコードを作成する必要があります。
