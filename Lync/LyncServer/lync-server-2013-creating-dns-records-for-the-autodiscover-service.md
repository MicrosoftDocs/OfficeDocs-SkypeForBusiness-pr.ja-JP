# <a name="creating-dns-records-for-the-autodiscover-service-in-lync-server-2013"></a>Lync Server 2013 での自動検出サービスの DNS レコードの作成

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**トピックの最終更新日:** 2014-06-20_

Lync モバイルユーザーは自動検出を有効にする必要があります。また、の一部では、いくつかのドメインネームシステム (DNS) レコードを作成する必要があります。 必要に応じて、次のものが必要になります。

  - 組織のネットワーク内から接続しているモバイルユーザーをサポートするための内部 DNS レコード。

  - インターネットから接続しているモバイルユーザーをサポートするための外部 (パブリック) DNS レコード

その理由 内部 DNS レコードと外部 DNS レコードの両方を SIP ドメインごとに作成する必要があります。

作成する DNS レコードは、(ホスト) レコードまたは CNAME レコードのいずれかになることができます。 お客様のために、以下の内部および外部の DNS レコードを作成する方法を順を追って説明します。 モバイルユーザーの DNS 要件の詳細については、「 [Lync Server 2013 でのモビリティの技術要件」](lync-server-2013-technical-requirements-for-mobility.md)を参照してください。

<div>

## <a name="creating-an-internal-dns-cname-record"></a>内部 DNS CNAME レコードの作成

1.  内部 DNS レコードを作成するには、Domain Admins グループのメンバーまたは DnsAdmins グループのメンバーであるドメインアカウントを使用して、ネットワーク内の DNS サーバーにログオンする必要があります。

2.  DNS 管理スナップインを開きます。[**スタート**] ボタンをクリックし、[**管理ツール**]、[**DNS**] の順にクリックします。

3.  DNS サーバーのコンソールツリーで、Lync Server 2013 のディレクタープールおよびフロントエンドプールがインストールされている Active Directory ドメインの [**前方参照ゾーン**] を展開します。 (たとえば、contoso.)。

4.  内部 DNS レコードのディレクタープールにホスト A (AAAA for IPv6) レコードが存在するかどうかを確認します。ホスト A レコードは、ディレクタープールの内部 Web サービス完全修飾ドメイン名 (たとえば、lyncwebdir01.contoso.local) に対して存在する必要があります。 ここで使用されていない場合は、ディレクタープールを使用していない可能性があります。また、設定している場合は、フロントエンドプールまたは単一のサーバー FQDN で FQDN を使用する必要があります。

5.  内部 DNS レコードのフロントエンドプールにホスト A (AAAA for IPv6) レコードが存在するかどうかを確認し、ホスト a (または AAAA) レコードが存在する場合は、フロントエンドプールの内部 Web サービス FQDN (例:、: lyncwebpool01.contoso.local)。 そうでない場合は、フロントエンドサーバーまたは Standard Edition サーバーの FQDN をメモしておく必要があります。

6.  ホスト A (または AAAA) レコードが存在することを確認したら、DNS サーバーのコンソールツリーで、使用している SIP ドメインの [**前方参照ゾーン**] を展開します (contoso.com など)。

7.  SIP ドメイン名を右クリックして、[**新しいエイリアス (CNAME)**] をクリックします。

8.  [**エイリアス名**] に、内部自動検出サービス URL のホスト名として「lyncdiscoverinternal」と入力します。

9.  [**ターゲットホスト用の完全修飾ドメイン名 (FQDN)**] で、ディレクタープールの内部 WEB サービス FQDN (たとえば、lyncwebdir01.contoso.local) を入力または参照し、[ **OK]** をクリックします。

10. Lync Server 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出 CNAME レコードを作成する必要があります。

## <a name="creating-an-external-dns-cname-record"></a>外部 DNS CNAME レコードを作成する

1.  外部 DNS CNAME レコードを作成するには、パブリック DNS プロバイダーに接続してから、手順を実行する必要があります。 外部 DNS を管理する方法が異なるかもしれませんが、DNS プロバイダーの環境での目的を正確に説明することはできませんが、これらの手順のヘルプをご利用いただけます。

2.  その環境で DNS レコードを作成できるアカウントを使用して、外部 DNS プロバイダーにログインします。

3.  この環境では、SIP ドメインが既に作成されている必要があります。 この SIP ドメインの**前方参照ゾーン**を展開するか、使用されている外部 DNS インターフェイスに応じて、それを選択します。

4.  ディレクタープール (lyncwebexdir01.contoso.com など) に対してホスト A (AAAA for IPv6) レコードが既に表示されているので、それを確認する必要があります。 そうでない場合は、ディレクタープールを使用していない可能性があります。 その場合は、フロントエンドプールの FQDN を使用するか、またはフロントエンドサーバーまたは Standard Edition サーバーに対してこの手順を1台のサーバーに対して実行する必要があります。

5.  また、フロントエンドプール (lyncwebextpool01.contoso.com など) に対して、外部 Web サービスの完全修飾ドメイン名 (FQDN) のレコードが存在すること、またはフロントエンドプールがない場合は、単一サーバーの FQDN の FQDN を使用していることを確認する必要があります。 前の手順で説明したように、ディレクタープールを持っていない場合は、以下のことが必要になります。

6.  次に、外部 DNS プロバイダに適した形式で、**新しいエイリアス (CNAME)** を作成するためのオプションを選択します (これは、DNS プロバイダーの形式に応じて、メニューオプションまたはリンクとなる場合があります)。

7.  内部 DNS と同様に何らかの形式の**エイリアス名**テキストボックスがある必要があります。外部自動検出サービス URL のホスト名として lyncdiscover を入力する必要があります。

8.  [**ターゲットホスト用の完全修飾ドメイン名 (FQDN)** ] テキストボックスの形式もあります。ここでは、ディレクタープールの外部 WEB サービス FQDN (lyncwebexdir01.contoso.com など) を入力し、[OK] をクリックするか、または外部 DNS のすべての操作を行ってこのエントリの作成を承諾します。 前述の手順4で説明したように、ディレクタープールを持っていない場合は、フロントエンドプールの FQDN または必要に応じて設定した単一サーバーの FQDN を使用する必要があります。

9.  Lync 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出 CNAME レコードを作成する必要があります。

## <a name="creating-an-internal-dns-a-record"></a>内部 DNS A レコードを作成する

1.  内部 DNS レコードを作成するには、Domain Admins グループまたは DnsAdmins グループのメンバーであるドメインアカウントを使用して、ネットワーク内の DNS サーバーにログオンします。

2.  DNS 管理スナップインを開きます。[**スタート**] ボタンをクリックし、[**管理ツール**]、[**DNS**] の順にクリックします。

3.  内部 DNS レコードの場合、DNS サーバーのコンソールツリーで、Lync Server 2013 のディレクタープールとフロントエンドプールがインストールされている Active Directory ドメイン (たとえば、[contoso])] の [**前方参照ゾーン**] を展開します。

4.  内部 DNS レコードのディレクタープールにホスト A (AAAA for IPv6) レコードが存在するかどうかを確認します。ホスト A レコードは、ディレクタープールの内部 Web サービス完全修飾ドメイン名 (たとえば、lyncwebdir01.contoso.local) に対して存在する必要があります。 ここで使用していない場合は、ディレクタープールを使用していない可能性があります。セットアップの場合は、フロントエンドプールまたは1つのサーバー IP アドレスで IP アドレスを使用する必要があります。

5.  内部 DNS レコードのフロントエンドプールにホスト A (AAAA for IPv6) レコードが存在するかどうかを確認し、ホスト a (または AAAA) レコードが存在する場合は、フロントエンドプールの内部 Web サービス FQDN (例:、: lyncwebpool01.contoso.local)。 そうでない場合は、フロントエンドサーバーまたは Standard Edition サーバーの IP アドレスをメモしておく必要があります。

6.  ホスト A (または AAAA) レコードが存在することを確認したら、DNS サーバーのコンソールツリーで、使用している SIP ドメインの [**前方参照ゾーン**] を展開します (contoso.com など)。

7.  SIP ドメイン名を右クリックして、[**新しいホスト (A または AAAA)**] をクリックします。

8.  [**名前**] に、内部自動検出サービス URL のホスト名として「lyncdiscoverinternal」と入力します。

9.  [ **Ip アドレス**] にディレクターの内部 WEB サービス IP アドレスを入力します (または、ロードバランサーを使用している場合は、ディレクターロードバランサーの仮想 IP (VIP) を入力します)。 前述の手順4で説明したように、ディレクターを使用していない場合は、フロントエンドサーバーまたは Standard Edition サーバーの IP アドレスを入力する必要があります。または、ロードバランサーを使用する場合は、フロントエンドプールのロードバランサーの VIP を入力する必要があります。

10. [**ホストの追加**] をクリックし、[**OK**] をクリックします。

11. 追加の A または AAAA レコードを作成するには、Lync Server 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで新しい自動検出 A または AAAA レコードを作成する必要があることを考慮して、手順 8 ~ 10 を繰り返します。

12. A (IPv6 では AAAA) レコードの作成が完了した後で、[**完了**] をクリックします。

## <a name="creating-an-external-dns-a-record"></a>外部 DNS A レコードを作成する

1.  外部 DNS レコードを作成するには、パブリック DNS プロバイダーに接続し、手順に従います。 外部 DNS を管理する方法が異なるかもしれませんが、DNS プロバイダーの環境での目的を正確に説明することはできませんが、これらの手順のヘルプをご利用いただけます。

2.  この環境で DNS レコードを作成できるアカウントとしてログインする必要があります。

3.  外部 DNS レコードの場合、DNS サーバーのコンソール ツリーで、使用する SIP ドメイン (例: contoso.com) の [**前方参照ゾーン**] を展開します。 外部 DNS レコードの場合、DNS サーバーのコンソール ツリーで、使用する SIP ドメイン (例: contoso.com) の [**前方参照ゾーン**] を展開します。

4.  ディレクタープール (lyncwebexdir01.contoso.com など) に対してホスト A (AAAA for IPv6) レコードが既に表示されているため、それが存在することと、IP アドレスが何であるかを確認する必要があります。 そうでない場合は、ディレクタープールを使用していない可能性があります。 その場合は、フロントエンドプールの IP アドレスを使用するか、またはフロントエンドサーバーまたは Standard Edition サーバーに対してこの操作を1台のサーバーに対して行う必要があります。 サーバーが負荷分散の背後にある場合や、リバースプロキシを使用している場合もあることに注意してください。 次の手順に従って、IP アドレスを書き留めておいてください。

5.  また、フロントエンドプール (lyncwebextpool01.contoso.com など) の外部 Web サービスの完全修飾ドメイン名 (FQDN)、または単一サーバーの Lync インストール用の IP アドレスがあるかどうかを確認する必要があります (例、フロントエンドプールがありません。 前の手順で説明したように、ディレクタープールを持っていない場合は、以下のことが必要になります。

6.  次に、外部 DNS プロバイダに適した形式で、**新しいホスト a または AAAA**を作成するためのオプションを選択します (これは、DNS プロバイダーの形式に応じて、メニューオプションまたはリンクの場合があります)。

7.  **名前**を入力する場所があります。外部自動検出サービス URL のホスト名として「lyncdiscover」と入力します。

8.  また、[ **Ip アドレス**] テキストボックスがあります。ここでは、ディレクタープールの ip を入力します (たとえば、lyncwebexdir01.contoso.com)。または、プールのロードバランサーの ip (または同じにつながるリバースプロキシ ip) を入力して、このエントリの作成を受け入れるように外部 DNS の任意のアクションを実行します。 前述の手順4で説明したように、ディレクタープールを持っていない場合は、フロントエンドプールの IP アドレスを使用するか、または必要に応じてセットアップした単一サーバーの IP アドレスを使用する必要があります。

9.  Lync 2013 環境でサポートされている各 SIP ドメインの前方参照ゾーンで、新しい自動検出 (A または AAAA) レコードを作成する必要があります。
