# 電話会議、アプリケーションおよび仲介サーバーのポート範囲の構成

 

_**トピックの最終更新日:** 2015-04-30_

サービスの品質 (QoS) を確保するには、会議サーバー、アプリケーション サーバー、および仲介サーバーのオーディオ、ビデオ、アプリケーション共有に同一のポート範囲を設定する必要があります。さらにこれらのポート範囲は、どのような場合でも重複しないように設定する必要があります。わかりやすい例として、会議サーバーでのビデオ用にポート 10000 からポート 10999 を使用するとします。つまりアプリケーション サーバーと仲介サーバーのビデオ用にもポート 10000 からポート 10999 を予約しておく必要があります。予約しない場合は、QoS は期待どおりに機能しません。

同様に、ビデオ用にポート 10000 からポート 10999 を予約し、オーディオ用にポート 10500 からポート 11999 を予約するとします。これによりポート範囲が重複するため、QoS に関する問題が発生する可能性があります。QoS では、各モダリティのポートのセットは一意でなければなりません。そのためビデオ用にポート 10000 からポート 10999 を使用する場合は、別の範囲を使用する必要があります (たとえばオーディオ用にはポート 11000 からポート 11999 を使用するなど)。

既定では、Microsoft Lync Server 2013 におけるオーディオとビデオのポート範囲は重複しないようになっています。しかしアプリケーション共有に割り当てられているポート範囲は、オーディオとビデオ両方のポート範囲と重複します (つまり、これらの範囲はどれも一意ではありません)。次の 3 つのコマンドを Lync Server 2013 管理シェルから実行すると、会議サーバー、アプリケーション サーバー、および仲介サーバーの既存のポート範囲を確認できます。

    Get-CsService -ConferencingServer | Select-Object Identity, AudioPortStart, AudioPortCount, VideoPortStart, VideoPortCount, AppSharingPortStart, AppSharingPortCount
    
    Get-CsService -ApplicationServer | Select-Object Identity, AudioPortStart, AudioPortCount
    
    Get-CsService -MediationServer | Select-Object Identity, AudioPortStart, AudioPortCount


> [!WARNING]
> 上記のコマンドでわかるように、各ポート タイプ (オーディオ、ビデオ、およびアプリケーション共有) には 2 つの個別のプロパティ値 (ポート開始とポート数) が割り当てられています。ポート開始は、そのモダリティで使用される最初のポートを示します。たとえばオーディオのポート開始が 50000 の場合は、オーディオ トラフィックに使用される最初のポートはポート 50000 であるということです。オーディオ ポート数が 2 (これは有効な値ではありませんが、ここではわかりやすいように使用します) の場合、オーディオ用に割り当てられているのが 2 つのポートのみであることを意味します。最初のポートがポート 50000 で、合計 2 つのポートがある場合は、2 番目のポートはポート 50001 でなければなりません (ポート範囲は連続している必要があります)。その結果、オーディオのポート範囲は、ポート 50000 から 50001 になります。<BR>アプリケーション サーバーと仲介サーバーでは、オーディオの QoS のみをサポートしていることに注意してください。アプリケーション サーバーと仲介サーバーでは、ビデオまたはアプリケーション共有のポートを変更する必要はありません。

既に述べたとおり、QoS に対応するために Lync Server ポートを構成するときは、次のことを確認する必要があります。1) 会議サーバー、アプリケーション サーバー、および仲介サーバーで、オーディオ ポート設定が同一である。2) ポート範囲が重複していない。上記の表を注意深く見ると、ポート範囲が 3 種類のサーバーで同一であることがわかります。たとえば、各サーバー タイプで開始オーディオ ポートがポート 49152 に設定され、各サーバーのオーディオ用に予約されているポートの合計数も 8348 で同じです。しかし、ポート範囲は重複しています。つまりオーディオ ポートはポート 49152 から開始し、アプリケーション共有用に確保されたポートも同じく 49152 から開始します。QoS を最適に利用するためには、アプリケーション共有が一意のポート範囲を使用するように再設定する必要があります。たとえば、アプリケーション共有がポート 40803 から開始し、8348 個のポートを使用するように構成することができます。ポートの個数が 8348 個である理由は、これらの値をすべて合算 (40803 + 8348) すると、アプリケーション共有がポート 40803 からポート 49151 を使用することになるためです。オーディオ ポートはポート 49152 より前に開始しないため、重複したポート範囲は使用されません。

アプリケーション共有に新しいポート範囲を選択したら、Set-CsConferencingServer コマンドレットを使用して変更できます。この変更は、アプリケーション サーバーまたは仲介サーバーでは必要ありません。これらのサーバーは、アプリケーション共有トラフィックを処理しないためです。オーディオ トラフィックに使用するポートを再割り当てする場合に限り、これらのサーバーのポート値を変更する必要があります。

単一の会議サーバーでアプリケーション共有用のポート値を変更するには、Lync Server 管理シェルから次のようなコマンドを実行します。

    Set-CsConferenceServer -Identity ConferencingServer:atl-cs-001.litwareinc.com -AppSharingPortStart 40803 -AppSharingPortCount 8348

すべての会議サーバーでこのような変更を行う場合は、次のコマンドを実行できます。

    Get-CsService -ConferencingServer | ForEach-Object {Set-CsConferenceServer -Identity $_.Identity -AppSharingPortStart 40803 -AppSharingPortCount 8348}

ポート設定を変更したら、変更によって影響を受ける各サービスを停止してから再実行する必要があります。

必ずしも、会議サーバー、アプリケーション サーバー、および仲介サーバーで全く同じポート範囲を共有しなければならないわけではありません。真に必要なことは、すべてのサーバー上で一意のポート範囲を確保することだけです。ただし、すべてのサーバーで同じポートのセットを使用すると、通常、管理が容易になります。
