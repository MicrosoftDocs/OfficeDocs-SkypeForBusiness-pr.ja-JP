# Lync Server 2013 の新しい障害復旧および高可用性機能

 

_**トピックの最終更新日:** 2012-09-20_

Lync Server 2010 と同様に、Lync Server 2013 の主要な高可用性 (HA) スキームは、プーリングによるサーバー冗長性に基づいています。特定のサーバーの役割を実行するサーバーで障害が発生した場合は、プール内の同じ役割を実行する他のサーバーがそのサーバーの負荷を引き受けます。同じことが、フロントエンド サーバー、エッジ サーバー、仲介サーバー、およびディレクターでも実行されます。

Lync Server 2013 では、2 つのデータ センターに配置されたフロントエンド プールをペアにすることができます。これにより、新しい障害復旧手段が追加されました。ペアの一方のプールがダウンした場合、管理者はそのプールからペアのもう一方のプールにユーザーをフェールオーバーすることでサービスを続行できます。この機能には、ストレージ ネットワークや共有ディスクなどの高価なネットワーク ソリューションまたはハードウェア ソリューションは不要です。

Lync Server 2013 では、バックエンド サーバーの高可用性も追加されました。これはオプションのトポロジであり、1 つのフロントエンド プール用に 2 つのバックエンド サーバーを展開し、バックエンド サーバー上で実行されているすべての Lync データベースに同期 SQL ミラーリングをセットアップします。ミラーの監視を展開するかどうかを選択できます。
