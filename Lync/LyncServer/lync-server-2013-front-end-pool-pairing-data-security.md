# Lync Server 2013 でのフロントエンド プールのペアリング データのセキュリティ

 

_**トピックの最終更新日:** 2016-12-08_

バックアップ サービスは、 Lync Server 2013 で導入されたデータ レプリケーション メカニズムです。障害復旧のため、2 つのデータ センターにまたがって、ペアになった 2 つの フロント エンド プール 間で継続的にユーザー データと会議コンテンツを転送します。ユーザー データには、ユーザーの SIP URI のほか、連絡先リストと設定が含まれます。会議コンテンツには、 Microsoft PowerPoint 2010 のアップロードのほか、会議で使用するホワイトボードが含まれます。ソース プールでは、ローカル ストレージからユーザー データと会議コンテンツがエクスポートされ、圧縮され、ターゲット プールに転送されます。ターゲット プールでは、それが解凍され、ローカル ストレージにインポートされます。バックアップ サービスでは、2 つのデータ センター間の通信リンクがインターネットから保護された企業ネットワーク内であることを前提とします。2 つのデータセンター間で転送されるデータは暗号化されませんし、HTTPS などのセキュアなプロトコル内でネイティブにカプセル化されることもありません。そのため、企業ネットワーク内にいる内部者による中間者 (man-in-the-middle) 攻撃が可能です。

## セキュリティ リスクの評価

複数のデータ センターにまたがって Lync Server 2013 を展開しており、障害復旧機能を使用している企業では、企業イントラネットによってデータ センター間のトラフィックを保護する必要があります。内部攻撃からの保護を求める企業は、データ センター間の通信リンクをセキュアにする必要があります。

企業のデータ センターは企業イントラネット内部で保護されているという前提は標準的です。こうしたデータ センター間で転送される企業の重要データには、ほかにも多くの種類があります。こうしたデータ センター間のリンクが保護されていなければ、企業の IT インフラストラクチャは深刻なリスクにさらされます。

企業ネットワーク内では中間者攻撃のリスクが存在する一方で、トラフィックをインターネットに露出することに比べればまだ無難です。具体的にいうと、バックアップ サービスによって露出されるユーザー データ (SIP URI など) は一般的に、その他の手段 (Exchange やその他のディレクトリ ソフトウェアによるグローバル アドレス帳など) によって、社内の従業員全員が利用できます。そのため、ペアになった 2 つのプール間でデータをコピーするためにバックアップ サービスを使用する場合は、2 つのデータ センター間の WAN を保護することに重点を置くべきです。

## セキュリティ リスクの軽減

バックアップ サービスのトラフィックに関するセキュリティ保護を強化するためには、データ センターへのアクセスの制限から、2 つのデータ センター間の WAN トランスポートの保護まで、多様な方法があります。通常、 Lync Server 2013 を展開する企業には、必要なセキュリティ インフラストラクチャがすでに存在します。ガイダンスを求める企業のために、マイクロソフトはセキュアな IT インフラストラクチャを構築する例としてソリューションを提供しています。ただし、それが唯一のソリューションということではなく、 Lync Server の推奨ソリューションということでもありません。企業のお客様には、IT セキュリティのインフラストラクチャと要件に基づいて、各社のニーズに合ったソリューションを選択することをお勧めします。マイクロソフトのソリューション例では、サーバーとドメインの分離に IPSec とグループ ポリシーを採用しています。詳細については、 [http://go.microsoft.com/fwlink/?linkid=268544\&clcid=0x411](http://go.microsoft.com/fwlink/?linkid=268544%26clcid=0x411) を参照してください。ご質問やご意見がございましたら、secwish@microsoft.com までお寄せください。

もう 1 つのソリューションは、バックアップ サービスそのものによって送信されたデータのセキュリティ保護だけに IPSec を使用するというものです。この方法を採用する場合は、プール A とプール B が 2 つのペアになった フロント エンド プール になっている次のサーバーについて、SMB プロトコルの IPSec ルールを構成してください。

  - プール A の各 フロント エンド サーバー からプール B によって使用されるファイル ストアへの SMB サービス (TCP/445)。

  - プール B の各 フロント エンド サーバー からプール A によって使用されるファイル ストアへの SMB サービス (TCP/445)。


> [!WARNING]
> IPsec は、SSL/TLS のようなアプリケーション レベルのセキュリティに代わるものではありません。IPsec を使用する利点の 1 つは、既存のアプリケーションを変更することなく、ネットワーク トラフィックのセキュリティを提供できることです。単に 2 つのデータ センター間のトランスポートを保護したい企業は、それぞれのネットワーク ハードウェア ベンダーに対し、ベンダーの機器を使用してセキュアな WAN 接続を構築する方法を相談してください。
