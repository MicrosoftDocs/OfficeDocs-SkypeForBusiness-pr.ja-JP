# Lync Server 2013 のユーザー登録レポート

 

_**トピックの最終更新日:** 2015-03-09_

ユーザー登録レポートは、ユーザー ログオン アクティビティの概要を示し、特に、指定期間内 (時間単位、日単位、週単位、月単位) 内に Microsoft Lync Server 2013 にログオンしたユーザーの数に関する情報を示します。このレポートで示されるのは、ログオンしたユーザーの数のみです。*どの*ユーザーがログオンしたかは示されません。監視レポートでは、どのユーザーが Lync Server 2013 を使用しているか (および使用していないか) に関する情報は示されません。ただし、ユーザー アクティビティ レポートを使用してユーザー情報の概算は把握できます。

ユーザー登録レポートでは、ユーザー ログオンに関する情報を示す場合に 2 つの重要な分類が行われます。まず、内部ログオンと外部ログオンという 2 つの大きなカテゴリにログオンが分類されます。内部ログオンは、組織のファイアウォール内から (企業ネットワークに接続中に) ログオンしたユーザーを表します。外部ログオンは、そのファイアウォールの外部から エッジ サーバー経由でログオンしたユーザーを表します (たとえば、インターネット カフェからログオンしたユーザーは外部ログオンと見なされます)。ファイアウォールの外部からログオンしたユーザーの数が必要な場合、ユーザー登録レポートでその情報を確認できます。

また、ユーザー登録レポートは、指定した期間内に存在した*アクティブ* ユーザーの数も示します。アクティブ ユーザーとは、インスタント メッセージング (IM) セッションに参加したユーザー、Lync 会議に参加したユーザー、通話を発信または受信したユーザー、あるいはその期間内にその他の方法で Lync Server を使用したユーザーです。ログオンしただけで実際にシステムを使用しなかったユーザーとは異なります。

## ユーザー登録レポートへのアクセス

ユーザー登録レポートには、監視レポートのホーム ページからアクセスできます。ユーザー登録レポートからは他のレポートへリンクしません。

## ユーザー登録レポートの効果的な活用方法

Lync Server を展開した後によくある質問として、この新しいテクノロジをユーザーが実際に使用しているかどうかをどうしたら確かめられるか、というものがあります。いくつか制限はありますが、ユーザー登録レポートがこの疑問の解決に役立ちます。ユーザーが Lync Server を使用しているかどうかを確認するには、2 つの作業が必要です。まず、ユーザー登録レポートで \[一意のログオン ユーザー数\] 指標の値を確認します。この値は、 Lync Server にログオンしたユーザーの人数を示します。

これに対し、\[ログオン合計数\] 指標は、ユーザーが Lync Server にログオンした回数の合計を示します。たとえば、Ken Myer が 1 日の間に Lync Server に 5 回ログオンしたとします。この場合、Ken Myer は、\[ログオン合計数\] 指標でログオン セッション回数 5 回として記録されるのに対し、\[一意のログオン ユーザー数\] 指標ではログオン ユーザー 1 人として記録されます。また、1 人のユーザーが複数のデバイスまたは複数の場所からログオンすることも珍しくありません。たとえば、同じユーザーが自分のデスクトップ コンピューターとラップトップ コンピューターからログオンするほか、 Lync Server に自動的にログオンする IP 電話も使用するとします。この場合、一意のユーザー数は 1 人、ログオン回数は 3 回になります。

この場合、ログオン合計数は 5 回ですが、ログオンしたユーザーは Ken Myer (ログオン回数 3 回) と Pilar Ackerman (ログオン回数 2 回) の 2 人だけです。ログオン合計数と一意のログオン ユーザー数はこのように異なります。

一意のログオン ユーザー数のほか、 Lync Server に対して有効になっているユーザーの合計数も把握する必要があります。この値を取得するには、 Lync Server 2013 管理シェルを開いて次の Windows PowerShell コマンドを実行します。

    (Get-CsUser).Count

このコマンドで返される値が 1,236 であり、\[一意のログオン ユーザー数\] 指標に返される値が平均 667 だとすると、 Lync に対して有効になっているユーザーのうち半分強が、毎日システムに実際にログオンしていることになります (667 を 1,236 で割ると約 54% です)。


> [!WARNING]
> ログオンの指標に記録されるのは、指定した期間内に実際にログオンしたユーザーの数である点に注意してください。その前に既にシステムにログオンしていたユーザーは追跡されません。たとえば、[一意のログオン ユーザー数] 指標が 667、ユーザー数が 1,236 の場合、約半分のユーザーがシステムにログオンしたことになります。ただし、ログオン データのチェックを開始する前に既に 300 人のユーザーがシステムにログオンしていたとします。この場合、実際には 1,000 人近いユーザーが Lync Server にログオンしており、ユーザーの 80% がログオンしたことになります。



また、\[一意のログオン ユーザー数\] の値と \[一意のアクティブなユーザー数\] 指標も比較する必要があります。\[一意のアクティブなユーザー数\] 指標は、 Lync Server を実際に使用した一意のユーザー (通話を発信したり、 Lync 会議に参加したり、IM セッションに参加したりしたユーザー) の数を示します。 Microsoft Lync 2013 はユーザーが Windows を起動すると自動的に起動するように構成することもできるので、この情報は有用です。そのような場合は、毎日 Windows にログオンするたびに自動的に Lync にログオンするものの実際には指定期間内に一度も Lync Server を使用しなかったユーザーが、多数存在する可能性があります。

\[一意のアクティブなユーザー数\] 指標は、ユーザーが 1 日の終わりに Windows をログオフしないことが多い組織ではさらに有用なデータになります。その場合、ユーザーはコンピューターをロックするだけで、 Windows と Lync は実行中のままになります。このような状態では、ユーザーは何日もログオンしたままでログオフしないので、1 日あたりのログオン数は非常に低くなる可能性があります。ただし、\[一意のアクティブなユーザー数\] は、ユーザーが実際に Lync や他の Lync Server クライアントを使用しているかどうかを示します。

## フィルター

フィルターは、細かく絞り込んだデータ セットを返したり、返されたデータをさまざま方法で表示したりする方法として利用できます。たとえば、ユーザー登録レポートでは、すべての レジストラー プールと エッジ サーバーのデータを表示したり、個別のプールのデータを表示したりできます。また、データをグループ化する方法を選択することもできます。この場合は、登録が、時間、日、週、または月を基準にグループ化されています。
