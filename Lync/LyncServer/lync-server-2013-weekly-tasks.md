# Lync Server 2013 での週次タスク

 

_**トピックの最終更新日:** 2016-12-08_

週単位の作業は一般的に、ログとレポートの収集および分析に関連しています。

## イベント ログをアーカイブする

必要に応じてイベントを上書きするようにイベント ログを構成していない場合は、イベント ログを定期的にアーカイブおよび削除する必要があります。 セキュリティ ログでは、セキュリティ侵害の試行を調査するときに必要になる可能性があるため、このアクションが特に重要です。

組織レベルで、ログのアーカイブに関するポリシーと手順を定義する必要があります。

## レポートを作成する

容量計画、SLA の確認、パフォーマンス分析を容易にするために、ステータス レポートを作成します。 イベント ログとシステム モニタから取得した日次データを使用して、ディスク、メモリ、CPU 使用率に関するレポートを作成します。System Center Operations Manager を使用して、稼働時間レポートと可用性レポートを生成します。

組織レベルで、ステータス レポートに関するポリシーと手順を定義する必要があります。

## インシデント レポート

Lync Server に関連する組織レベルのインシデント レポートに対して週単位の監査を実施します。 この監査には、次のものを含める必要があります。

  - 発生済み、解決済み、および保留中の重要なインシデント。

  - 未解決のインシデントに対するソリューション。

  - 新しいトラブル チケットを含めるためのレポートの更新。

  - サービス停止に関するトラブルシューティングのガイドや事後分析のためのドキュメント リポジトリの更新。

それぞれの組織のインシデント追跡システムは、Lync Server とは関係なく選択されるものであるため、具体的な説明や指針は提供できません。組織が選択したシステムに対応するドキュメントを参照してください。

## IIS のログとパフォーマンスを確認する

インターネット インフォメーション サービス (IIS) のログとパフォーマンスを週単位で確認します。IIS のログとパフォーマンスを監視する方法については、「[Windows Server 2003 のインターネット インフォメーション サービス (IIS) のイベント ログの概要](http://go.microsoft.com/fwlink/?linkid=36077)」を参照してください。 この確認には、次のものを含める必要があります。

  - WWW サービスのキャッシュを監視する Web Service Cache カウンター。

  - ASP として実行されているアプリケーションを監視する Active Server Pages (ASP) カウンター。

IIS のログとパフォーマンスを監視する方法については、「[Windows Server 2003 のインターネット インフォメーション サービス (IIS) のイベント ログの概要](http://go.microsoft.com/fwlink/?linkid=36077)」を参照してください。

## データベース レポートを生成する

**SQL データベースに関するレポートを生成するには**

1.  Lync Server 2013 を開きます。

2.  コンソール ツリーで、フォレスト ノードを展開し、\[**エンタープライズ プール**\] を展開してから、データベース レポートを生成するプールをクリックします。

3.  詳細ウィンドウで、\[**データベース**\] タブをクリックします。

4.  \[**データベース**\] タブでは、以下のことを実行できます。
    
    1.  データベースの名前を表示するには、\[**全般設定**\] を展開し、データベース名を表示します。
    
    2.  プールに対して現在のユーザーの概要統計情報を取得するには、\[**ユーザー概要レポート**\] を展開し、\[**実行**\] をクリックして、結果を表示します。
    
    3.  プールの単一のユーザーに対して現在のユーザー単位のデータを取得するには、\[**ユーザー単位のレポート**\] を展開し、ユーザーの SIP URI を入力して、\[**実行**\] をクリックし、結果を表示します。

プールに対して現在の電話会議の概要統計情報を取得するには、\[**電話会議の概要レポート**\] を展開し、\[**実行**\] をクリックして、結果を表示します。

## セキュリティと Lync Server の更新プログラムのチェック

すべての新しいサービス パック、修正プログラム、更新プログラムを確認します。必要な場合は、テスト ラボでこれらをテストし、変更管理手順を使用して実稼働サーバーに展開する方法を手配します。また、Lync Server コンポーネントの更新プログラムも、現在は Windows Update の一部として利用できます。Lync Server コンポーネントのすべての更新プログラムは、それらの更新プログラムの適用対象である Lync Server を実行しているすべてのサーバーで同時に更新する必要があります。

## Lync Server 2013 ベスト プラクティス アナライザーの実行

Lync Server 2013 ベスト プラクティス アナライザー ツールは、構成情報を収集し、マイクロソフトのベスト プラクティスに従って構成が設定されているかどうかを判断する診断ツールです。 このツールに関するドキュメントは、[Lync Server 2013 ベスト プラクティス アナライザー](lync-server-2013-lync-server-best-practices-analyzer.md) および [ベスト プラクティス アナライザーの実行](https://technet.microsoft.com/ja-jp/library/gg398652\(v=ocs.15\)) で利用できます。

このツールは、展開済みの構成データを、Lync Server に対応する一連の定義済みルールと比較し、潜在的な問題を報告します。 報告されたすべての問題に対して、このツールでは Lync Server 環境内の現在の構成と、推奨される構成を示します。

ネットワーク アクセスが正しく設定されている場合は、このツールは AD DS と、Lync Server 2013 を実行しているサーバーを確認して、次のことを実行することができます。

  - 事前に正常性チェックを実行して、推奨されるベスト プラクティスに従って構成が設定されていることを確認します

  - 最適化されていない構成設定や、サポートされていないオプション、推奨されていないオプションなど、問題の一覧を生成します

  - システムの全般的な正常性を判定します

  - 特定の問題に関するトラブルシューティングを支援します

  - 利用可能な更新プログラムが存在する場合は、それらのダウンロードを促す画面を表示します

  - 報告されている問題に関するオンライン ドキュメントとローカル ドキュメント、トラブルシューティングのヒントを提供します

  - 後で確認を行う目的でキャプチャできる構成情報を生成します

RTCBPA.msi がすべての Lync Server 2013 サーバーにインストールされていることを確認し、週単位の正常性チェック レポートを生成します。 結果を確認し、必要な場合は修正します。

## SLA パフォーマンスの指標を確認します

前の週の主要なパフォーマンス データをチェックします。SLA の要件に照らしてパフォーマンスを確認します。トレンドと、目標を達成していない項目を特定します。

## System Center Operations Manager 管理パックと QoE レポートを確認します

Lync Server 2013 管理パックと QoE レポートを取得し、確認します。

## エンタープライズ プールに対応するデータベース レポートの生成と表示

1.  Lync Server 2013 を開きます。

4.  \[**データベース**\] タブでは、以下のことを実行できます。
    
    1.  データベースの名前を表示するには、\[**全般設定**\] を展開し、データベース名を表示します。
    
    2.  プールに対して現在のユーザーの概要統計情報を取得するには、\[**ユーザー概要レポート**\] を展開し、\[**実行**\] をクリックして、結果を表示します。

プールに対して現在の電話会議の概要統計情報を取得するには、\[**電話会議の概要レポート**\] を展開し、\[**実行**\] をクリックして、結果を表示します。

各エンタープライズ プールに対して、管理者は \[**データベース**\] タブを使用してデータベース名を表示し、データベースから取得したレポートを表示することができます。

## Bandwidth Utilization Analyzer の実行

Bandwidth Utilization Analyzer は、エンタープライズ ネットワークの WAN リンク全体の UC エンドポイントで消費される帯域幅を、さまざまな観点から確認できるレポートを作成するツールです。 これらのレポートは、現在の帯域幅の消費パターンの把握と、帯域幅の容量計画の作成に活用できます。また、さまざまなリンクに割り当てられた帯域幅容量を反復処理します。

このツールでは、次のことを実行できます。

  - ネットワーク全体の音声の使用率に関するレポートを生成する

  - より効果的な容量計画の立案と、さまざまなリンクに割り当てられた帯域幅容量の反復処理をサポートする

Bandwidth Utilization Analyzer では、帯域幅容量のグラフィカル プロットと使用状況レポートを作成できます。その範囲と実行可能な操作は次のとおりです。

  - エンタープライズ ネットワーク内のすべての WAN リンク

  - 事前に指定した WAN リンクでフィルター処理

  - リンク容量を超過した WAN リンクでフィルター処理

  - 帯域幅の使用率が予測を下回る WAN リンクでフィルター処理

  - 重大レベル (WAN リンクの帯域幅使用率が容量の 90% を超過している) に達している WAN リンクでフィルター処理

  - WAN リンクの種類 (ネットワーク サイト リンク、地域間リンク、サイト内リンク) でフィルター処理

  - ネットワーク地域でフィルター処理

このツールに関するドキュメントについては、[Lync Server 2013 リソース キット ツール ドキュメント](https://technet.microsoft.com/ja-jp/library/jj945604\(v=ocs.15\))を参照してください。
