# Lync Server 2013 のピアツーピア アクティビティ診断レポート

 

_**トピックの最終更新日:** 2015-03-09_

ピアツーピア アクティビティ診断レポートは、ピアツーピア通信セッションの成功およびエラーに関する情報を提供します。 Microsoft Lync Server 2013 では、以下に示す各種エラーが区別されます。

  - **予期されたエラー** 。通常、予期されたエラーは、最も厳密な意味でのエラーです。たとえば、だれかに電話をかけ、その相手がオフィスにいなくて電話に出られないとします。その電話は応答がなかったので、厳密にはエラーと見なされます。しかし、これは予期されたエラーです。 Microsoft Lync Server 2013 では、ユーザーが電話に出られない場合には電話の応答がないものと想定しているからです。同様に、予期されたエラーは、オフラインのユーザーや、インスタント メッセージングをサポートしていない電話にしかログオンしていないユーザーにインスタント メッセージを送信しようとした場合にも発生します。

  - **予期しないエラー** 。予期しないエラーは、まさにその名前が示すとおりのもので、その発生を状況に基づいて予測できないエラーです。たとえば、だれかに電話をかけ、その人物が電話に出ることができるとします。しかし、 Lync Server 2013 がその通話をボイスメールにルーティングしようとすると、Exchange ユニファイド メッセージングへの接続が失われているという理由で通話はエラーになりました。これは予期しないエラーです。通話は、常にボイスメールにルーティングできるものと想定されているからです。通常は、予期しないエラーが本来のエラーです。こうしたエラーは、ユーザーの教育やこれに類似した対策では改善できない可能性が高い問題です。

成功、予期されたエラー、予期しないエラーの各指標の値を合計してもセッションの合計数の指標値に等しくならない場合があります。たとえば、先ほど説明した例では、次のような値が得られました。

2,024 + 469 + 16 を計算すると合計は 2,509 セッションになりますが、セッションの合計数の列には合計 2,521 セッションと示されています。"不足している" 12 セッションは、システムが成功にもエラーにも分類できなかったセッションです。こうした状況は、サードパーティの製品で導入された新しい診断コードが Lync Server で認識できない場合に発生することがあります。この場合、その製品を使用して行われ、該当する診断コードを通知する通話は、必ずしも成功、予期されたエラー、または予期しないエラーのいずれかに分類されない可能性があります。

## ピアツーピア アクティビティ診断レポートへのアクセス

ピアツーピア診断レポートには、監視レポートのホーム ページからアクセスできます。以下の指標のどちらかをクリックすると、 [Lync Server 2013 のエラー分布レポート](lync-server-2013-failure-distribution-report.md)にアクセスできます。

  - \[予期しないエラー ボリューム\]

  - \[予期されるエラー ボリューム\]

## ピアツーピア アクティビティ診断レポートの活用

ピアツーピア アクティビティ診断レポートをフィルターできる方法はいくつかありますが、既定ではそうしたフィルター処理オプションがビューに表示されません。フィルター処理オプションの表示を有効にするには、レポート ウィンドウの右上隅にある \[パラメーターの表示/非表示\] ボタンをクリックします。この操作により、フィルター処理オプションを使用できるようになります。

## フィルター

フィルターは、細かく絞り込んだデータ セットを返したり、返されたデータをさまざま方法で表示したりする方法として利用できます。たとえば、ピアツーピア アクティビティ診断レポートを使用すると、セッション モダリティ (インスタント メッセージング、ファイル送信、アプリケーション共有など) のようなものでフィルターできます。また、データをグループ化する方法を選択することもできます。この場合は、通話が、時間、日、週、または月を基準にグループ化されています。
