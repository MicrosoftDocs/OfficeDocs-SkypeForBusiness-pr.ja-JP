# <a name="peer-to-peer-activity-diagnostic-report-in-lync-server-2013"></a>Lync Server 2013 のピアツーピアアクティビティ診断レポート

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainBody">

<span> </span>

_**トピックの最終更新日:** 2012-10-01_

ピアツーピア アクティビティ診断レポートは、ピアツーピア通信セッションの成功およびエラーに関する情報を提供します。 Microsoft Lync Server 2013 では、さまざまな種類の障害について区別されることに注意してください。

  - **予期されたエラー**。 予期されたエラーは通常、単に技術的な意味においてのエラーです。 たとえば、だれかを呼び出すと、自分がオフィスの外にいて、電話に応答できないとします。 呼び出しが応答されなかったため、この呼び出しは技術的に失敗したと見なされます。 一方、これは予期されたエラーです。電話に応答できない場合、Microsoft Lync Server 2013 は電話に応答することを期待していません。 同様に、オフラインのユーザーにインスタントメッセージを送信しようとした場合や、インスタントメッセージングをサポートしていない電話にのみログオンしている場合は、予期されるエラーが発生します。

  - **予期しないエラー**。 予期しないエラーは名前が示すとおり、その状況下では発生が予期されないエラーです。 たとえば、ある人を呼び出すと、その人物が通話に応答できるとします。ただし、Lync Server 2013 が通話をボイスメールにルーティングしようとすると、Exchange ユニファイドメッセージングへの接続が失われたため、呼び出しが失敗します。 これは予期しないエラーです。呼び出しは常にボイスメールにルーティングされる可能性があると考えられます。 一般的なルールとして、予期しないエラーが発生する場合があります。これは、ユーザー教育や同様の手段では解決できない問題です。

成功、予期されたエラー、予期しないエラーの各指標の値を合計してもセッションの合計数の指標値に等しくならない場合があります。たとえば、先ほど説明した例では、次のような値が得られました。

## <a name="accessing-the-peer-to-peer-activity-diagnostic-report"></a>ピアツーピア アクティビティ診断レポートへのアクセス

ピアツーピア診断レポートには、監視レポートのホーム ページからアクセスできます。 次のいずれかの指標をクリックすると、 [Lync Server 2013 のエラー分布レポート](lync-server-2013-failure-distribution-report.md) にアクセスできます。

  - [予期しないエラー ボリューム]

  - [予期されるエラー ボリューム]

</div>

<div>

## <a name="making-the-best-use-of-the-peer-to-peer-activity-diagnostic-report"></a>ピアツーピア アクティビティ診断レポートの活用

ピアツーピア アクティビティ診断レポートをフィルターできる方法はいくつかありますが、既定ではそうしたフィルター処理オプションがビューに表示されません。フィルター処理オプションの表示を有効にするには、レポート ウィンドウの右上隅にある [パラメーターの表示/非表示] ボタンをクリックします。この操作により、フィルター処理オプションを使用できるようになります。

</div>
