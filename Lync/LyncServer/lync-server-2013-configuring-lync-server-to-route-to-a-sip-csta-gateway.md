# SIP/CSTA ゲートウェイにルーティングするための Lync Server 2013 の構成

 

_**トピックの最終更新日:** 2012-10-05_

SIP/CSTA ゲートウェイは、SIP と CSTA (Computer Supported Telecommunications Application) の間のゲートウェイです。SIP/CSTA ゲートウェイは、既存の構内交換機 (PBX) と Lync Server の間に、リモート通話コントロール要求を PBX にルーティングするためのインターフェイスを提供します。SIP/CSTA ゲートウェイをインストールした後は、リモート通話コントロールの構成対象である Lync Server プールごとに以下の手順を実行する必要があります。

  - [Lync Server 2013 でのリモート通話コントロールの静的ルートの構成](lync-server-2013-configure-a-static-route-for-remote-call-control.md)

  - [Lync Server 2013 でリモート通話コントロールの信頼済みアプリケーション エントリを構成する](lync-server-2013-configure-a-trusted-application-entry-for-remote-call-control.md)
